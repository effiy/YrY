## 检查项

### P0 - 必须通过

| 检查项 | 检查内容 | 验证方法 |
|---------|---------|---------|
| **用户故事表格存在** | 包含用户故事与功能需求表格 | 检查是否有"用户故事与功能需求"章节和表格 |
| **用户故事表格完整** | 用户故事表格包含 4 列：用户故事、验收标准、过程生成文档、产出智能文档 | 检查表格列数是否为 4，列顺序是否正确 |
| **优先级图标使用** | 用户故事使用 🔴🟡🟢 优先级图标 | 检查表格中是否包含优先级图标 |
| **单个用户故事要求** | 表格中只包含一个用户故事，不允许多个 | 检查表格行数是否为 1（不含表头） |
| **用户故事包含主要操作场景** | 用户故事列中包含"主要操作场景"部分，且场景分行显示 | 检查用户故事列中是否有"**主要操作场景**："，且场景使用 "- " 分行显示 |
| **功能概述章节存在** | 包含"功能概述"章节 | 检查是否有"功能概述"章节 |
| **文档规范章节存在** | 包含"文档规范"章节 | 检查是否有"文档规范"章节 |
| **用户故事与文档对应规范** | 文档规范中包含"一个用户故事对应自己的需求任务、设计文档、项目报告内容及文件" | 检查文档规范章节中是否有此规范 |
| **功能详情章节存在** | 包含"功能详情"章节 | 检查是否有"功能详情"章节 |
| **验收标准章节存在** | 包含"验收标准"章节 | 检查是否有"验收标准"章节 |
| **验收标准位置正确** | 验收标准章节在用户故事后面、功能详情前面 | 检查章节顺序是否正确 |

### P1 - 应该通过

| 检查项 | 检查内容 | 验证方法 |
|---------|---------|---------|
| **核心价值章节存在** | 功能概述中包含"核心价值"列表 | 检查功能概述中是否有核心价值 |
| **用户故事格式规范** | 用户故事采用"作为[角色]，我想要[功能]，以便[价值]"格式 | 主观判断用户故事格式是否规范 |
| **用户故事链接有效** | 用户故事表格中的需求任务、设计文档、项目报告列包含链接 | 检查这三列是否包含链接，链接格式是否正确 |
| **独立文件命名规范** | 当用户故事使用独立文件时，文件名遵循"{功能名称}-{用户故事简短描述}.md"格式 | 检查独立文件命名是否规范 |
| **验收标准分级** | 验收标准按 P0/P1/P2 分级组织 | 检查是否有 P0、P1、P2 子章节 |
| **验收标准可测试** | 验收标准描述清晰，可验证可测试 | 主观判断验收标准是否可测试 |
| **使用场景章节存在** | 包含"使用场景示例"章节 | 检查是否有"使用场景示例"章节 |
| **使用场景格式规范** | 使用场景采用"背景-操作-结果"三段式格式 | 检查使用场景是否有"**背景**："、"**操作**："、"**结果**：" |
| **功能点描述完整** | 功能详情中的功能点包含功能说明、价值、解决的痛点 | 检查功能点是否包含这三个部分 |
| **关联文档链接** | 有关联文档链接（需求任务、设计文档、使用文档） | 检查元数据中是否有关联文档 |

### P2 - 可以有

| 检查项 | 检查内容 | 验证方法 |
|---------|---------|---------|
| **功能点有子章节** | 功能详情中的功能点使用 H3 子章节 | 检查功能点是否使用 H3 标题 |
| **段落划分合理** | 段落长度适中，每段不超过 3-4 行 | 主观判断段落划分是否合理 |
| **列表使用恰当** | 并列内容使用列表格式 | 主观判断列表使用是否恰当 |
| **表格格式规范** | 表格对齐美观，表头明确 | 主观判断表格格式是否规范 |
