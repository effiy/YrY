## 检查项

### P0 - 必须通过

| 检查项 | 检查内容 | 验证方法 |
|---------|---------|---------|
| **文档头部完整** | 包含版本号、最后更新日期、维护者信息 | 检查是否有文档版本、最后更新、维护者信息 |
| **关联文档链接** | 正确链接到需求任务、使用文档、CLAUDE.md | 检查元数据区域的关联文档链接 |
| **设计概述章节** | 有"设计概述"章节，100-200字 | 检查是否有设计概述章节 |
| **架构设计章节** | 有"架构设计"章节 | 检查是否有架构设计章节 |
| **整体架构图** | 包含 Mermaid 架构图（graph TB 或 LR） | 检查是否有 Mermaid 架构图 |
| **遵循 YiWeb 架构** | 使用 Store 工厂模式、组件全局注册等 | 主观判断是否符合项目架构 |
| **文件路径正确** | 模块划分表格中的文件路径有效 | 检查文件路径格式是否正确 |

### P1 - 应该通过

| 检查项 | 检查内容 | 验证方法 |
|---------|---------|---------|
| **模块划分表格** | 有完整的模块划分表格（模块名称、职责、文件位置） | 检查是否有模块划分表格 |
| **核心流程图** | 包含 Mermaid 核心流程图（flowchart TD） | 检查是否有核心流程图 |
| **架构说明清晰** | 每个图表下方都有明确的说明文字 | 检查每个 Mermaid 图表后的说明 |
| **要点说明详细** | 每个技术实现要点都说明"做什么、怎么做、为什么" | 主观判断说明是否详细 |
| **数据流程图存在** | 有 Mermaid 数据流程图（sequenceDiagram） | 检查是否有数据流程图 |

### P2 - 可以有

| 检查项 | 检查内容 | 验证方法 |
|---------|---------|---------|
| **设计原则列表** | 设计概述中包含设计原则列表（带图标） | 检查是否有设计原则列表 |
| **Mermaid 样式规范** | 图表节点色值符合项目视觉规范 | 检查是否使用标准色值 |
| **测试验证方法** | 说明如何验证修复是否有效 | 检查是否有测试验证说明 |
| **备选方案讨论** | 如果有多个方案，说明选择理由 | 检查是否有备选方案讨论 |
| **边界情况说明** | 说明边界情况的处理方式 | 检查是否有边界情况说明 |
