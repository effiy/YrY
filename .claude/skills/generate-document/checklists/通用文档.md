## 检查项

### P0 - 必须通过

| 检查项 | 检查内容 | 验证方法 |
|---------|---------|---------|
| **文档头部结构完整** | 严格按照规范包含所有头部元素 | 检查文档头部是否按顺序包含：H1标题、空行、元数据行1、空行、关联文档行、空行、章节导航（可选）、分隔线 |
| **元数据完整** | 文档包含版本号、最后更新日期、维护者 | 检查文档开头是否有 `> **文档版本**:`, `> **最后更新**:`, `> **维护者**: ` |
| **版本号格式正确** | 版本号格式为 `v{主版本}.{次版本}` | 检查版本号是否匹配 `v\d+\.\d+` 格式 |
| **日期格式正确** | 最后更新日期格式为 `YYYY-MM-DD` | 检查日期是否匹配 `\d{4}-\d{2}-\d{2}` 格式 |
| **关联文档行存在** | 有关联文档行，即使无关联文档也保留该行 | 检查是否有 `> **关联文档**: ` 行 |
| **头部与正文有分隔线** | 文档头部与正文之间有 `---` 分隔线 | 检查在头部区域后是否有 `---` 分隔线 |
| **H1 唯一** | 文档有且仅有一个 H1 标题 | 统计 `# ` 开头的行数，必须等于 1 |
| **标题层级不超过 H3** | 文档仅使用 H1-H3 三级标题 | 检查是否有 `#### ` 或更深层级的标题 |
| **Mermaid 图表有说明** | 每个 Mermaid 图表下方都有明确的说明文字 | 检查每个 ```mermaid``` 块后是否有 "**...说明**：" |
| **链接使用相对路径** | 所有文档链接使用相对路径，不使用绝对路径 | 检查链接是否包含 `http://`、`https://` 或以 `/` 开头 |
| **技术断言可溯源** | 对仓库结构、文件路径、公开 API/导出、行为与版本的断定须能在已引用上游文档或代码路径中核对 | 抽查关键句：无 `路径`/`行号`/`docs/…§` 支撑则 P0 不通过 |
| **无虚构路径与依赖** | 不引用不存在的 `src/` 路径、包名、未在仓库或上游出现的配置项 | 对文档中出现的 `路径` 用 Glob/Grep 或清单声明「未验证」并列入待补充 |
| **待补充显式化** | 无来源的段落使用 `> 待补充（原因：…）` 或 `待读者确认`，不得写 plausible 细节凑满 | 扫「确定/必须/已存在」类强断言是否均有来源或已标 C 类 |
| **开放问题可收敛** | 需人类决策的项集中在列表或明确小节，不与事实陈述混为一段 | 检查是否存在带「假设/可能/待确认」却写成既定事实的句子 |

### P1 - 应该通过

| 检查项 | 检查内容 | 验证方法 |
|---------|---------|---------|
| **段落之间有空行** | 段落之间使用空行分隔 | 检查是否有连续的非空行（代码块、表格除外） |
| **段落长度适中** | 段落不超过 3-4 行 | 检查是否有过长的段落（建议不超过 4 行） |
| **列表使用恰当** | 并列内容使用列表，步骤使用有序列表 | 检查是否有应该使用列表但未使用的情况 |
| **表格格式规范** | 对比性内容使用表格，表头明确 | 检查是否有应该使用表格但未使用的情况 |
| **代码块标注语言** | 代码块标注了正确的语言类型 | 检查 ``` 开头是否有语言标识（javascript、mermaid、markdown 等） |
| **关联文档存在** | 文档顶部有相关联的文档链接 | 检查是否有关联文档链接区域 |
| **标题命名清晰** | 标题简洁明了，准确反映章节内容 | 主观判断标题是否清晰 |

### P2 - 可以有

| 检查项 | 检查内容 | 验证方法 |
|---------|---------|---------|
| **Mermaid 节点样式** | 节点色值符合规范要求 | 检查是否使用了 #ccffcc、#ffcccc、#e1f5ff 等标准色值 |
| **图标使用统一** | 使用 🎯⚡📖🔴🟡🟢 等标准图标 | 检查图标使用是否一致和规范 |
| **章节导航** | 长文档有顶部章节导航链接 | 检查文档开头是否有章节导航链接 |
| **重要信息使用引用块** | 重要提示、说明使用引用块 | 检查重要信息是否使用 `>` 引用块 |
