# 代码结构规范

## 目录组织

### 整体架构

```
YiWeb/
├── .claude/        # Claude Code 配置和技能
├── cdn/            # 共享库（供多项目复用）
├── docs/           # 项目文档
├── src/            # 源代码
│   ├── core/       # 核心模块
│   └── views/      # 视图应用
├── CLAUDE.md       # 项目说明
└── README.md       # 项目总览
```

### 共享模块规范

**重要**：所有可复用的共享模块必须统一放在 `cdn/` 目录下，**不得**放在 `src/shared/`。

| 模块类型 | 存放位置 |
|---------|---------|
| 通用组件 | `cdn/components/common/` |
| 业务组件 | `cdn/components/business/` |
| 工具函数 | `cdn/utils/` |
| 样式文件 | `cdn/styles/` |
| Markdown 渲染 | `cdn/markdown/` |
| Mermaid 渲染 | `cdn/mermaid/` |

### 组件目录结构

每个 Vue 组件目录包含：
```
ComponentName/
├── index.js        # 组件逻辑和注册
├── template.html   # 组件模板
└── index.css       # 组件样式（可选）
```

## 核心应用流程

### 应用初始化

```
createBaseView() → createStore() → useComputed() → useMethods() → 挂载
```

### 入口文件

应用入口在 `src/views/{app}/index.js`，使用 `createBaseView` 工厂初始化。

## 状态管理模式

### Store 工厂

Store 使用工厂模式组合多个操作模块：

```javascript
// storeFactory.js
export const createStore = () => {
    // 1. 创建基础状态
    const { state, internals } = createAicrStoreState(vueRef);
    
    // 2. 创建各操作模块
    const sessionsOps = createAicrStoreSessionsOps(deps, state);
    const fileContentOps = createAicrStoreFileContentOps(deps, state, internals);
    
    // 3. 返回组合后的 store
    return {
        ...state,
        ...fileContentOps,
        ...sessionsOps
    };
};
```

### 状态定义

使用 Vue ref 创建响应式状态：

```javascript
// storeState.js
export function createAicrStoreState(vueRef) {
    const fileTree = vueRef([]);
    const selectedKey = vueRef(null);
    
    return {
        state: { fileTree, selectedKey },
        internals: { /* 内部非响应式状态 */ }
    };
}
```

### 操作模块

采用依赖注入模式：

```javascript
export const createAicrStoreSessionsOps = (deps, state) => {
    const { safeExecuteAsync } = deps;
    
    return {
        async loadSessions() {
            return safeExecuteAsync(async () => {
                // 使用 state 和 deps 实现逻辑
                state.sessions.value = await getData();
            }, '加载会话列表');
        }
    };
};
```

## BaseView 工厂

`createBaseView` 是应用初始化的核心工厂函数（位于 `cdn/utils/view/baseView.js`）：

```javascript
async function createBaseView(config = {}) {
    const {
        createStore,      // 必填：创建状态 store
        useComputed,      // 必填：定义计算属性
        useMethods,       // 必填：组合业务方法
        components = [],  // 组件名称列表
        componentModules = [], // 组件模块路径
        onMounted = null  // 挂载后回调
    } = config;
    
    // 创建 setup 函数并挂载应用
    // ...
}
```

## 方法组合模式

### useMethods 组合器

方法通过 `useMethods` 组合器组织：

```javascript
export const useMethods = (store) => {
    // 创建各方法模块
    const sessionMethods = createSessionMethods({ store, safeExecute });
    const searchMethods = createSearchMethods({ store });
    
    return {
        ...sessionMethods,
        ...searchMethods
    };
};
```

## 计算属性模式

### useComputed 组合器

计算属性通过 `useComputed` 组合器定义：

```javascript
export const useComputed = (store) => {
    return {
        filteredSessions: Vue.computed(() => {
            return (store.sessions?.value || []).filter(session => {
                // 过滤逻辑
            });
        })
    };
};
```

### 安全执行模式

```javascript
// cdn/utils/core/error.js
export const safeExecute = (fn, operationName) => {
    try {
        return fn();
    } catch (error) {
        logError(`[${operationName}] 执行失败:`, error);
        return null;
    }
};
```

## 模块导入规范

### 绝对路径导入

项目使用**绝对路径**导入：

```javascript
// ✅ 推荐：绝对路径
import { createStore } from '/src/views/aicr/hooks/store.js';
import { logInfo } from '/cdn/utils/core/log.js';

// ❌ 避免：相对路径
import { createStore } from '../hooks/store.js';
```

### 全局对象访问

对于 Vue 等全局库，直接从 `window` 访问：

```javascript
const count = Vue.ref(0);
const doubled = Vue.computed(() => count.value * 2);
```

## 检查清单

开发新模块时，请确认：

- 功能相关文件放在同一目录下
- 目录深度不超过 5 层
- 共享模块放在 `cdn/` 目录
- 组件包含 index.js/template.html（可选 index.css）
- 使用绝对路径导入模块
- Store 使用工厂模式组合操作模块
- 状态通过 Vue ref 定义，用 .value 访问
- 操作模块使用依赖注入模式
- 方法通过 useMethods 组合器组织
- 组件全局注册并触发 Loaded 事件
