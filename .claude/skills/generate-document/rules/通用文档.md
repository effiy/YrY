# 通用文档规范

## 文档头部规范（严格遵循）

### 必须包含内容

所有通用文档必须严格按照以下顺序包含文档头部：

```markdown
# {文档标题}

> **文档版本**: v{版本号} | **最后更新**: {YYYY-MM-DD} | **维护者**: {大模型名称} | **工具**: {Claude Code / Cursor}
>
> **关联文档**: [关联文档1](../目录/{文件名}.md) | [关联文档2](../目录/{文件名}.md)
>

[章节1](#章节1) | [章节2](#章节2) | [章节3](#章节3)

---
```

### 头部各部分说明

| 部分 | 必须包含 | 说明 |
|------|---------|------|
| **H1 标题** | ✅ 必须 | 文档主标题，使用 `#` 标记 |
| **空行** | ✅ 必须 | H1 标题与元数据之间的空行 |
| **元数据行1** | ✅ 必须 | 文档版本、最后更新、维护者 |
| **空行** | ✅ 必须 | 元数据行1与行2之间的空行 |
| **关联文档行** | ✅ 必须 | 关联文档链接，无关联时留空 |
| **空行** | ✅ 必须 | 关联文档行与章节导航之间的空行 |
| **章节导航** | 🟡 推荐 | 长文档推荐添加，短文档可选 |
| **分隔线** | ✅ 必须 | 使用 `---` 分隔头部与正文 |

### 元数据规范

- 所有文档开头必须包含元数据，使用块引用（`>`）格式
- 文档版本：格式 `v{主版本}.{次版本}`，新文档从 v1.0 开始
- 最后更新：格式 `YYYY-MM-DD`，使用当前日期
- 维护者：填写当前会话使用的大模型名称（如 `Claude Sonnet 4.6`、`GPT-4o` 等）
- 工具：填写实际使用的 AI 工具（`Claude Code` 或 `Cursor`）
- 关联文档：使用相对路径，多个文档用 `|` 分隔
- 章节导航：长文档推荐添加，列出主要章节

## 标题层级规范

- **H1**（`#`）：文档主标题，每个文档唯一，有且仅有一个
- **H2**（`##`）：主要章节，文档的一级分区，建议 3-8 个
- **H3**（`###`）：子章节，主要章节下的细分，按需使用
- **H4 及以下**：禁止使用

## Mermaid 图表规范

### 图表类型选择

- **自顶向下架构图**：`graph TB` - 系统架构、功能分解、模块划分
- **从左到右关系图**：`graph LR` - 组件关系、流程步骤、依赖关系
- **带判断的流程图**：`flowchart TD` - 用户流程、业务流程、核心流程
- **时序交互图**：`sequenceDiagram` - 时序交互、数据流向、调用链

### 节点样式色值

- **正向/新增/核心功能**：`fill:#ccffcc`
- **反向/问题**：`fill:#ffcccc`
- **保持/中性/规范/模板**：`fill:#e1f5ff`

### 图表说明

- 每个图表必须在图表下方添加明确的说明文字

## 格式排版规范

- **段落长度**：每段不超过 3-4 行
- **段落分隔**：段落之间使用空行分隔
- **段落内容**：每段只表达一个中心思想
- **无序列表**（`-`）：用于并列的项目、特点、要点等
- **有序列表**（`1.`、`2.`）：用于步骤、流程、优先级等
- **表格**：对比性、结构化内容使用表格
- **代码块**：代码示例使用代码块并标注语言类型
- **引用块**（`>`）：重要信息、提示、说明使用引用块
- **粗体**（`**文字**`）：用于关键术语、重要概念
- **斜体**（`*文字*`）：用于补充说明、次要信息
- **行内代码**（`` `文字` ``）：用于文件名、路径、变量名、命令

## 链接规范

- 所有文档链接必须使用相对路径，避免绝对路径
- 同级目录：`[文档名](./文档名.md)`
- 上级目录：`[文档名](../目录/文档名.md)`
- 多级上级：`[文档名](../../目录/文档名.md)`
- 链接格式统一为：`[显示文本](相对路径)`

## 质量检查清单

### P0 - 必须通过

- **元数据完整**：包含文档版本、最后更新日期、维护者
- **H1 唯一**：每个文档有且仅有一个 H1 标题
- **标题层级**：仅使用 H1-H3 三级，不使用 H4 及以下
- **Mermaid 图表说明**：每个 Mermaid 图表都有明确的说明文字
- **链接使用相对路径**：所有文档链接使用相对路径，不使用绝对路径

### P1 - 应该通过

- **段落划分合理**：段落长度适中，不超过 3-4 行
- **列表使用恰当**：并列内容使用列表，步骤使用有序列表
- **表格格式规范**：对比性内容使用表格，表头明确
- **代码块标注语言**：代码块标注了正确的语言类型
- **关联文档完整**：文档顶部有相关联的文档链接

### P2 - 可以有

- **Mermaid 节点样式**：节点色值符合规范要求
- **图标使用统一**：使用 🎯⚡📖🔴🟡🟢 等标准图标
- **章节导航**：长文档有顶部章节导航链接
