# verification-loop

## 核心理念

**不依赖重试，依赖准备。** 在执行任何验证命令前，先把所有可能导致失败的问题找出来并修复；执行只是最终确认，不是调试入口。

## 输入

- **验证目标**：如 `npm run build`、`Vite 打包`、`CDN 资源集成`（必填）
- **上下文文档**：设计文档路径 / 需求任务路径（用于理解已知约束）
- **环境信息**：Node 版本、包管理器（可选，自动从 `package.json` / `.nvmrc` 读取）

## 前置：MCP 可用性探针

> 在执行任何阶段之前，先检查本次验证所需的 MCP 工具是否可用，给出明确状态，**禁止静默降级**。

### 探针步骤

**1. 识别本次验证所需的 MCP 工具**

| 验证场景 | 所需 MCP 工具 |
|---------|-------------|
| UI 交互 / E2E 冒烟测试 | `playwright` |
| 本地文件读写 / 下载验证 | `filesystem` |
| 纯构建 / Lint / 类型检查 | 无需 MCP（Shell 命令即可） |

**2. 执行探针**

```
playwright 探针（UI 验证场景时执行）：
  操作：browser_navigate → about:blank
  结果：✅ 可用 / ❌ 不可用
```

**3. 根据探针结果决策**

| 工具 | 状态 | 行动 |
|------|------|------|
| `playwright` | ✅ 可用 | 阶段 4 的 UI 验证项使用 MCP 驱动，全自动执行 |
| `playwright` | ❌ 不可用 | **明确提示**（见下方），降级为 `npx playwright test`，结果标注 ⚠️ |
| 其他必要工具 | ❌ 不可用且无降级 | ⛔ 停止，提示用户配置后重试 |

**playwright 不可用时的明确提示（禁止静默跳过）**：

```
⚠️ Playwright-MCP 未配置，本次 UI 验证将降级为 npx playwright test（需人工确认截图）。

若要启用全自动 E2E 验证，请按以下步骤配置 Playwright-MCP：

  方式一：Cursor MCP 配置（推荐）
    在 ~/.cursor/mcp.json 中添加：
    {
      "mcpServers": {
        "playwright": {
          "command": "npx",
          "args": ["@playwright/mcp@latest"]
        }
      }
    }
    配置后重启 Cursor，重新运行本技能即可启用全自动验证。

  方式二：项目级配置
    在项目根目录 .cursor/mcp.json 中添加相同配置。

  参考文档：https://playwright.dev/docs/mcp
```

### 探针输出格式

```
=== MCP 可用性探针 ===
验证场景：<UI 验证 / 构建验证 / 混合>
playwright : ✅ 可用（全自动 MCP 驱动）
             ⚠️ 不可用（降级：npx playwright test，需人工确认）
filesystem : ✅ 可用 / ⚠️ 不可用（降级：Read/Write 工具）
结论：继续执行阶段 1 / ⛔ 停止（<工具名> 无降级方案）
```

## 工作流程（4 个阶段，顺序执行）

### 阶段 1：读取环境与配置（不执行任何命令）

在运行任何命令前，先静态读取：

1. `package.json`：确认脚本名、依赖版本、engines 字段
2. `vite.config.*` / `tsconfig.json` / `.eslintrc.*`：确认构建配置
3. `.nvmrc` / `.node-version`：确认 Node 版本要求
4. 设计文档 / 需求任务：提取已知约束与特殊配置

**输出**：环境快照（Node 要求 / 包管理器 / 构建命令 / 关键配置项）

**阻断条件**：若 `package.json` 不存在或目标脚本不存在 → **停止，报告缺失，不继续执行**

### 阶段 2：全量静态预检（不执行构建）

在执行构建前，逐项静态排查所有已知失败来源：

| 预检项 | 检查方式 | 常见失败原因 |
|--------|---------|------------|
| 依赖完整性 | 对比 `package.json` 与 `node_modules` | 缺包、版本不匹配 |
| TypeScript 类型 | 读取 `tsconfig.json`，扫描 `.ts` 文件的明显类型错误 | 类型不兼容、缺少声明文件 |
| Import 路径 | 扫描 `import` 语句，校验相对路径文件是否存在 | 路径拼写错误、文件被删除 |
| 环境变量 | 扫描 `import.meta.env.*` / `process.env.*`，对比 `.env` 文件 | 缺少必要变量 |
| Lint 规则 | 静态读取 `.eslintrc.*` 与规范文件，识别明显违规 | 格式错误、未使用变量 |
| 已知约束 | 对照设计文档提取的约束逐项检查 | 约定未落实 |

**输出**：预检报告（通过项 / 风险项 / 阻断项）

**阻断条件**：发现任何"阻断项"（如 import 路径不存在、必要环境变量缺失）→ **先修复阻断项，再进入阶段 3**

### 阶段 3：环境对齐确认

确认执行环境与配置要求一致：

- 当前 Node 版本是否满足 `engines.node` 或 `.nvmrc`
- 包管理器与 `packageManager` 字段一致
- 若有 Docker / CI 环境要求，确认环境变量已注入

**输出**：环境对齐状态（对齐 / 需调整）

**阻断条件**：环境不对齐 → 给出对齐命令（如 `nvm use`），**等待执行后再继续**

### 阶段 4：一次执行 + 结果断言

全部预检通过后，执行验证命令，**只执行一次**。UI / E2E 验证项的执行方式由前置探针结果决定：

| 验证项 | 执行方式 | 通过条件 |
|--------|---------|--------|
| 依赖安装 | `npm install` 或 `pnpm install` | exit code 0，无 peer dep 错误 |
| 类型检查 | `tsc --noEmit`（若存在） | exit code 0 |
| Lint | lint 脚本（若存在） | exit code 0 |
| 构建 | build 脚本 | exit code 0，产物目录非空 |
| P0 单元测试 | test 脚本（若存在） | exit code 0 |
| **UI / E2E 验证**（playwright ✅ 可用） | MCP `browser_navigate` + `browser_click` + `browser_snapshot` 全自动执行 | 所有 P0 检查项通过 |
| **UI / E2E 验证**（playwright ⚠️ 降级） | `npx playwright test tests/e2e/<功能名>/` | exit code 0；截图需人工确认，结果标注 ⚠️ 降级 |

若执行失败：
- **不重试**
- 解析错误输出，给出精确的根因定位（文件:行号 + 原因分类）
- 输出"一次性修复清单"（按优先级排序的具体操作）
- 说明修复后需重新从哪个阶段开始

## 输出格式

```
=== MCP 可用性探针 ===
playwright : ✅ 可用（全自动 MCP 驱动）/ ⚠️ 不可用（降级：npx playwright test）
filesystem : ✅ 可用 / ⚠️ 不可用（降级：Read/Write 工具）
结论：继续执行 / ⛔ 停止（<工具名> 无降级方案）

=== 阶段 1：环境快照 ===
Node 要求：<版本> | 当前：<版本> | 状态：对齐 / 不对齐
构建命令：<命令>
关键配置：<摘要>

=== 阶段 2：静态预检 ===
通过：<N> 项
风险项（P1）：
  - <描述>（建议：<操作>）
阻断项（P0）：
  - <描述>（必须修复：<操作>）

=== 阶段 3：环境对齐 ===
状态：对齐 / 已调整（执行：<命令>）

=== 阶段 4：验证结果 ===
结果：全部通过 / 失败
UI/E2E 验证方式：MCP 全自动 / npx playwright test ⚠️ 降级

失败项（若有）：
  - <文件>:<行> — <错误描述>
    根因：<分类：配置 / 依赖 / 类型 / 代码>
    修复：<具体操作>

一次性修复清单（按执行顺序）：
  1. <操作1>
  2. <操作2>
修复后重入：阶段 <N>
```

## 使用规则

- **前置探针有阻断项时禁止进入阶段 1**，必要 MCP 工具不可用且无降级方案时停止。
- **阶段 1-3 有阻断项时禁止进入阶段 4**，阻断项未解决不执行构建命令。
- 验证命令名称必须取自 `package.json scripts`，不得假设或硬编码命令。
- 命令不存在时，输出"脚本 <名称> 不存在于 package.json，跳过该项"。
- 阶段 4 失败后，不自动重试，给出"一次性修复清单"交由调用方决策。
- playwright 降级时，必须在输出中标注 ⚠️，不得以全通过方式呈现降级结果。
