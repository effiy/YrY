# Security Rules

## 注入条件

故事涉及以下任一项时介入，否则跳过：
- 用户输入、外部 API、认证/授权、数据持久化、第三方集成

## 威胁优先级

| 类别 | 默认等级 |
|------|---------|
| 注入（SQL/NoSQL/命令） | P0 |
| 认证绕过（会话固定、token 泄露） | P0 |
| 数据泄露（未脱敏、错误消息泄露） | P0 |
| 传输安全（明文、证书缺失） | P0 |
| 授权缺陷（越权、IDOR） | P0 |
| 资源滥用（无速率限制） | P1 |
| 配置缺陷（调试端点、CORS 过宽） | P1 |

## 审查自检

- [ ] 所有用户输入点已消毒
- [ ] 外部 API 传输安全已验证
- [ ] 认证/会话无常见漏洞
- [ ] 敏感数据不在日志/错误中暴露
- [ ] 第三方资源有 integrity hash
- [ ] 错误消息不区分用户存在/不存在

## 硬编码检查

- 第三方 CDN URL 无 integrity → P0
- 内联 script/style → P1（除非有 nonce/hash）
- 硬编码 token/secret/key → P0
