# Delivery Rules

## 交付管线

每次 rui 完成/阻断/门禁失败后按序执行：

| Step | 操作 | 失败处理 |
|------|------|---------|
| 1 | self-improve-loop | 不阻断（H11 降级） |
| 2 | import-docs --workspace | H9: token 缺失时跳过 |
| 3 | wework-bot 通知 | 不可跳过 |

## 通知消息格式

纯文本，emoji 前缀，`———` 分隔摘要段和明细段：

### 完成必含字段
🎯 结论 📝 描述 📌 范围 👉 下一步 🌐 影响 📎 证据 ⏱️ 会话

### 阻断必含字段
🎯 结论 📝 描述 📌 范围 ❌ 原因 🧭 恢复点 🌐 影响 📎 证据 ⏱️ 会话

### 门禁失败必含字段
🎯 结论 📝 描述 📌 范围 🔍 门禁 📊 结果 🌐 影响 📎 证据 ⏱️ 会话

## 格式约束

- 纯文本，禁用 markdown 语法（`#` `**` `-` `>` 等）
- 分隔线仅用 `———`（至多一条）
- 每行一个字段，emoji 后用 `:` 分隔
- 数字须来自执行结果，禁止占位符
- 全文 ≤2000 字

## 阻断处置

- `rui-state.js save --blocked` → `next-step` → 持久化 → 同步（H9/H11 跳过）→ 通知

## 环境变量

- `API_X_TOKEN`：import-docs 和 wework-bot 凭据，仅从环境变量读取
- 不得将 token 写入仓库、日志或文档
