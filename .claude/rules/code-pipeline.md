# code-pipeline

> **口诀：分支隔、测先行、模块清、闭环验。** 源码改动只走 `/rui code`。

## 适用

YrY 源码改动（元项目(插件/配置)）。

## 规则

### 分支隔

1. 功能分支从 main/master 创建，命名 `feat/YrY-<name>`
2. 改动源码前必须已切到该分支（`no-checkout`）
3. 禁止自动合并到主干（`auto-merge`）
4. 源码修改唯一入口是 `/rui code`

### 测先行（Gate A）

5. 04-测试用例评审.md 不存在，不得编码（`skip-gate-a`）
6. 例外：元项目可跳过 Gate A（配置变更）
7. 测试方案与原型未就绪即视为未通过

### 模块清

8. 逐模块编码：每模块完成后审查 P0/P1/P2，**P0 不清零不进下一模块**
9. 影响链未闭合不声称闭合（`chain-broken`）
10. 不创建设计文档外的文件

### 闭环验（Gate B）

11. 五步：环境快照 → 静态预检 → 设计/实现对齐 → 单次执行 → 报告（05-后端实施/06-前端实施/07-测试报告)
12. 报告交叉引用闭合，评审清单全 ✅ 方过
13. 修复 ≤ 2 轮，超过阻断（`gate-b-limit`）
14. 自改进必须产出 08

## 阻断标识

| 标识 | 触发 |
|------|------|
| `bad-branch` | 分支非从 main 创建 |
| `no-checkout` | 未切换故事分支即改源码 |
| `auto-merge` | 功能分支被自动合并 |
| `skip-gate-a` | Gate A 未通过即编码 |
| `chain-broken` | 影响链未闭合 |
| `gate-b-limit` | Gate B > 2 轮 |
