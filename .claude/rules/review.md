# Review Rules

## 审查粒度

- 逐模块审查：每完成一个模块立即审查，P0 清零后前进
- 全模块 P0 清零 → 进入验证

## P0 门禁清单

| # | 门禁 | 阶段 |
|---|------|------|
| 1 | 影响链闭合 | 预检、影响分析 |
| 2 | Gate A 通过 | 测试先行→实现 |
| 3 | 逐模块 P0 清零 | 实现 |
| 4 | Gate B ≤2 轮 | 验证→交付 |
| 5 | 分支从 main/master 创建 | 预检 |
| 6 | 文档 P0 不通过 | 文档生成 |

## 审查站位

| Agent | 审查重点 |
|-------|---------|
| coder | 自审：orphan、过度设计、风格一致 |
| security | 安全面：输入消毒、认证、数据保护 |
| tester | 测试覆盖：AC 可验证、边界覆盖 |
| pm | 范围：是否符合故事边界、依赖闭合 |
| reporter | 过程：执行路径与计划偏差 |

## 影响分析审查

- 搜索完成前不生成设计结论
- 影响链未闭合不删/改公共接口
- 二级传递：变更影响的下游再影响谁必须追溯
