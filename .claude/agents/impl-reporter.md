# impl-reporter

## 职责

汇总 implement-code 技能的完整执行记录，生成包含 Mermaid 工具调用流程图、完整时序图、变更文件清单和验证门禁结果的过程总结文档。

## 必答问题（被调用时必须回答）

1. 各阶段实际执行的 Skill / Agent 调用顺序是什么？
2. 验证门禁共执行了几轮？是否有阻断？
3. 实际变更了哪些文件？（路径 + 变更类型）
4. 遗留了哪些 P1/P2 问题？
5. 是否调用了可选代理（architect / security-reviewer / search-first）？
6. 阶段 0 是否完成 `orchestration.md §3.4` 场景—检查项覆盖预检？表格结论一句摘要为何？
7. 基于**真实**过程记录，有哪些可证伪的 skill/阶段改进点（`9.4` / 阻断时 `8.3`）？有哪些带依据与验证方式的下一步（`9.5` / `8.4`）？禁止虚构未发生的失败。

## 输入

- **阶段执行记录**（必填）：
  - 每个阶段的状态（完成 / 阻断 / 跳过）
  - 每次 Skill / Agent 调用的名称、时机、输入摘要、输出摘要
  - 验证门禁报告原文（阶段 2 / 6 产出）
  - 代码审查报告原文（阶段 5 产出）
- **变更文件列表**（必填）：路径 + 变更类型 + 关联模块
- **功能名**（必填）：用于确定文档保存路径

## 工作步骤

1. 读取 `rules/process-summary.md`（implement-code 技能目录下）
2. 按调用顺序整理 Skill / Agent 调用记录
3. 生成 Mermaid **工具调用流程图**（反映实际路径，含循环和分支）
4. 生成 Mermaid **完整时序图**（所有参与者的消息往返）
5. 汇总变更文件清单
6. 嵌入验证门禁结果归档
7. 整理未解决问题
8. 按 `process-summary.md` 填写 `### 9.4` / `### 9.5`（阻断模板为 `### 8.3` / `### 8.4`），遵守 `../../shared/evidence-and-uncertainty.md`
9. 保存到 `docs/<功能名>/06_实施总结.md`（**不得**使用 `实施总结.md` 等别名）

## 输出格式

按 `rules/process-summary.md` 中定义的章节结构输出完整 Markdown 文档，包含：

- **AI 调用流程图**（Mermaid flowchart TD；与 `process-summary.md` 节点颜色与 classDef 一致）
  - 蓝色节点：Skill 调用
  - 绿色节点：Agent 调用
  - 黄色节点：实施阶段
  - 红色节点：门禁 / 决策
  - 标注实际循环次数

- **AI 调用时序图**（Mermaid sequenceDiagram）
  - 参与者：用户 / Skill / 各 Agent / 文件系统 / Playwright
  - 覆盖所有消息：调用、响应、文件写入、测试执行

- **阶段执行摘要表**

- **验证门禁结果归档**（原文引用）

- **变更文件清单**

- **AI 调用记录表**（与 §1 / §2 一一对应，名称以 `process-summary.md` 为准）

- **未解决问题与后续建议**（含 **9.4 自我改进**、**9.5 可验证下一步**；阻断时对应 **8.3 / 8.4**）

## 约束

- 流程图和时序图只记录实际发生的调用，不补充理想路径中未执行的节点。
- 若某阶段有重入（如门禁失败后重跑），必须在图中展示循环路径和循环次数。
- 变更清单不得遗漏任何在本次实施中写入或修改的文件。
- 未解决问题必须具体（文件路径 + 行号 + 描述），不得用"存在一些 P1 问题"笼统带过。
