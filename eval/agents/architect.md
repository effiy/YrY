# architect 评测示例

真源：`.claude/agents/architect.md`。

**评测约束**：输出必须包含模块划分、接口规范、数据流和整体架构（Mermaid 图），且符合项目既有架构约定。

---

## 用户故事 A：设计文档前的架构设计

**故事**：作为架构师，希望在生成设计文档前得到模块划分和接口规范建议。

**示例输入（对话）**

- 「基于这个需求任务，设计一下模块划分和数据流。」
- 「生成架构设计：模块、接口、状态管理方案。」

---

## 成功判据（可观测）

| 维度 | 预期 |
|------|------|
| 模块划分 | 列出模块名称、职责、文件位置 |
| 接口规范 | 明确输入 / 输出 / 错误处理 |
| 数据流 | 建议 Mermaid sequenceDiagram 描述流转 |
| 架构图 | 建议 Mermaid graph TB 描述整体架构 |
| 约定兼容 | 说明是否符合项目既有约定，若不符合给出迁移/兼容策略 |

---

## 负例（应判不达标）

- 输出缺少模块划分或接口规范中的任一必答项。
- 未评估与项目既有架构约定的兼容性。
- 仅给出文字描述而无 Mermaid 图建议。
