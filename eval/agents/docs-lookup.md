# docs-lookup 评测示例

真源：`.claude/agents/docs-lookup.md`。

**评测约束**：准确检索和定位项目文档，提供相关文档路径和关键内容摘要。

---

## 用户故事 A：Grounding / 上游定位

**故事**：作为开发者，希望快速找到与当前任务相关的上游文档。

**示例输入（对话）**

- 「找一下跟登录功能相关的设计文档和需求任务。」
- 「定位 `src/auth.ts` 对应的上游设计文档。」

---

## 成功判据（可观测）

| 维度 | 预期 |
|------|------|
| 准确性 | 返回的文档路径真实存在且与查询相关 |
| 摘要 | 提供关键内容摘要，帮助判断是否需要全文阅读 |
| 覆盖 | 不遗漏明显相关的文档 |
| 边界 | 不编造不存在的文档路径 |

---

## 负例（应判不达标）

- 返回不存在的文档路径。
- 遗漏明显相关的文档（如只返回需求文档而忽略设计文档）。
- 摘要与文档实际内容不符。
