# doc-reviewer 评测示例

真源：`.claude/agents/doc-reviewer.md`。

**评测约束**：审查文档的完整性、一致性和可执行性，确保文档能支撑下游实施。

---

## 用户故事 A：文档保存后审查

**故事**：作为文档质量保证者，希望在文档生成后检查其质量。

**示例输入（对话）**

- 「审查 `docs/login/03_设计文档.md` 的完整性和一致性。」
- 「检查这份需求任务是否可执行，有没有遗漏的边界条件。」

---

## 成功判据（可观测）

| 维度 | 预期 |
|------|------|
| 完整性 | 检查文档是否包含所有必要章节和字段 |
| 一致性 | 检查文档内部及与上游文档之间是否存在矛盾 |
| 可执行性 | 检查需求任务是否明确到可实施的程度 |
| 必答问题 | 回答被调用时要求的必答问题 |

---

## 负例（应判不达标）

- 仅做拼写检查而未审查内容完整性和一致性。
- 未指出文档与上游需求之间的矛盾。
