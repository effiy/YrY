# planner 评测示例

真源：`.claude/agents/planner.md`。

**评测约束**：阅读需求任务文档，识别技术风险，输出实施策略与里程碑。

---

## 成功判据（可观测）

| 维度 | 预期 |
|------|------|
| 场景提取 | 逐条列出需求任务中的主要操作场景 |
| 风险识别 | 明确识别技术风险点或不确定性 |
| 里程碑 | 给出推荐的实施顺序和关键里程碑 |
| 依赖关系 | 列出需要提前处理的依赖关系 |
| 必答问题 | 回答所有被调用时要求的必答问题 |

---

## 负例（应判不达标）

- 遗漏需求任务中的主要操作场景。
- 未识别明显的技术风险（如第三方依赖冲突、性能瓶颈）。
- 未给出分阶段实施建议或里程碑。
