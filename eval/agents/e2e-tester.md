# e2e-tester 评测示例

真源：`.claude/agents/e2e-tester.md`。

**评测约束**：与 `e2e-testing` skill 的分工须符合 `shared/agent-skill-boundaries.md`；e2e-tester 负责并行专家角色与必答问题，skill 负责测试方案方法和产出格式。

---

## 用户故事 A：专家级测试设计

**故事**：作为测试专家，希望并行调用代理对复杂 UI 场景进行深度测试设计。

**示例输入（对话）**

- 「并行调用 e2e-tester 设计登录流程的边界测试场景。」
- 「对购物车功能做专家级 E2E 测试分析。」

---

## 成功判据（可观测）

| 维度 | 预期 |
|------|------|
| 测试先行 | 在代码实施前介入，确保场景验收标准先于代码产出 |
| 必答问题 | 回答所有被调用时要求的必答问题 |
| 专业性 | 提供比 skill 更深入的测试策略（如异常路径、性能边界） |
| 并行性 | 作为并行代理组合中的一员，不隐含顺序依赖 |
| 边界 | 不替代 skill 输出格式，而是提供深度分析输入 |

---

## 负例（应判不达标）

- 混淆 Skill 与 Agent 职责，直接输出 skill 的用例骨架格式而不提供专家视角。
- 未回答必答问题即结束。
