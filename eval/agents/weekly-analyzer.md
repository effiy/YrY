# weekly-analyzer 评测示例

真源：`.claude/agents/weekly-analyzer.md`。

**评测约束**：扫描 `docs/` 下全文档集和 `.claude/` 记忆文件，推断 KPI、复盘根因与后期规划；所有数据必须来自实际文件读取。

---

## 用户故事 A：周报数据分析

**故事**：作为周报生成者，希望从项目文档和 agent 记忆中提取结构化数据用于周报。

**示例输入（对话）**

- 「分析本周 `docs/` 下的功能目录，生成 KPI 候选数据。」
- 「从 agent 记忆和项目文件中推断复盘结论和后期规划。」

---

## 成功判据（可观测）

| 维度 | 预期 |
|------|------|
| 数据来源 | 所有数据来自实际文件读取，不凭印象推断 |
| KPI 覆盖 | 覆盖交付完成率、P0 通过率、防幻觉率、修复轮次、规则覆盖率 |
| 复盘 | 提炼亮点和问题根因，每条附证据路径 |
| 规划 | 后期规划五项排序，每步绑定量化目标、验证方式与风险 |
| 自改进 | 分别给出系统自改进（skills/agents/rules/shared）和项目自改进（文档/架构/代码）候选 |

---

## 负例（应判不达标）

- 统计数据无法追溯到具体文件路径。
- 无功能目录时未如实说明，而是编造用户故事案例。
- 后期规划未绑定量化目标或验证方式。
