# find-agents 评测示例

真源：`.claude/skills/find-agents/SKILL.md`。

**评测约束**：只返回 `.claude/agents/` 根目录下真实存在的代理文件名，禁止编造；返回列表默认设计为可并行调用。

---

## 成功判据（可观测）

| 维度 | 预期 |
|------|------|
| 真实性 | 返回的代理名均为 `.claude/agents/` 下真实存在的 `.md` 文件名（不含 `.md`），无编造 |
| 并行性 | 返回组合默认可并行调用，无隐含顺序依赖 |
| 必答问题 | 为每个代理生成明确的"必答问题" |
| 匹配度 | 推荐与文档类型和任务目标语义匹配，置信度合理 |
| 边界 | 明确区分 Skill 与 Agent 职责，不越界替代 Skill 功能 |

---

## 负例（应判不达标）

- 返回不存在的代理名（如 `test-expert`、`auto-fix` 等编造名称）。
- 推荐的代理组合隐含强顺序依赖，却未说明执行顺序。
- 未为代理生成必答问题或问题过于笼统。

---

## 与 eval 的关联

- Skill / Agent 边界真源：`../../shared/agent-skill-boundaries.md`
- 各代理定义见 `../../agents/<name>.md`。
