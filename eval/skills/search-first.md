# search-first 评测示例

真源：`.claude/skills/search-first/SKILL.md`。

**评测约束**：所有推荐必须有可验证来源（URL / 包名@版本）；搜索与需求不符时须明确说明"未找到满足约束的方案"，不猜测。

---

## 用户故事 A：技术选型搜索

**故事**：作为技术决策者，希望在引入新依赖前先搜索并评估候选方案。

**示例输入（对话）**

- 「前端需要一个大表格组件库，支持虚拟滚动，搜索一下 npm 上合适的方案。」
- 「对比 `zod` 和 `yup`，从活跃度、体积和社区规模评估。」

---

## 用户故事 B：MCP 工具搜索

**故事**：作为工具使用者，希望找到可用的 MCP 工具来解决特定问题。

**示例输入（对话）**

- 「搜索一下有哪些 MCP 工具可以操作浏览器。」
- 「MCP 里有没有支持文件系统操作的 server？」

---

## 成功判据（可观测）

| 维度 | 预期 |
|------|------|
| 来源可验 | 每项推荐均附有 URL、包名@版本或等价可验证来源 |
| 评估矩阵 | 覆盖功能覆盖、维护活跃度、社区规模、许可证等维度 |
| 决策明确 | 给出采用 / 扩展现有 / 组合 / 自建的建议 |
| 诚实兜底 | 搜索与需求不符时输出"未找到满足约束的方案"；无网络时明确说明参考可能过时 |

---

## 负例（应判不达标）

- 推荐无来源验证（仅有库名无 URL 或版本信息）。
- 搜索结果不足时仍编造下载量、stars 等数据。
- 将训练记忆中的过时库冒充为最新搜索结果。
