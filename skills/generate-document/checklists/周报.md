## 检查项

### P0 - 必须通过

| 检查项 | 检查内容 | 验证方法 |
|---------|---------|---------|
| **文档头部完整** | 含版本、日期、维护者、覆盖周期、关联功能目录 | 检查是否有完整头部元数据 |
| **OKR+KPI 合并总表覆盖所有功能目录** | 每个 docs/<功能名>/ 目录都有对应的 Objective 行 | 对照 docs/ 目录逐一验证 |
| **KR 有证据链接** | 总表下方引用块列出关键证据文件路径 | 检查证据引用块 |
| **KR 达成状态有据** | ✅/🟡/❌ 状态从实际文档内容推断 | 对照 06_实施总结和 05_动态检查清单验证 |
| **KPI 复盘有量化数据** | 5 个 KPI 维度都有具体数值（百分比或轮次） | 检查总表综合行数值列 |
| **链路全景图正确** | flowchart 展示 OKR→KPI→规划因果关系 | 检查 Mermaid 语法和连线逻辑 |
| **规划与改进总表可执行** | 每项有类型标签、KPI 指标、验证方式、证据 | 检查类型列和验证方式列 |
| **优先级矩阵图正确** | quadrantChart 展示影响度×实施难度分布 | 检查 Mermaid 语法和坐标合理性 |
| **防幻觉** | 无虚构数据、无虚构案例、无 D 类陈述 | 逐一核对数据来源 |

### P1 - 应该通过

| 检查项 | 检查内容 | 验证方法 |
|---------|---------|---------|
| **OKR 达成状态准确** | ✅/🟡/❌ 与实际文档内容一致 | 对照 06_实施总结验证 |
| **链路全景图因果关系正确** | 未达成的 KR 正确连线到对应 KPI 瓶颈 | 检查 flowchart 连线 |
| **优先级矩阵坐标合理** | 影响未达标 KPI 的改进项 x 值更高 | 检查 quadrantChart 坐标 |
| **规划与改进总表优先级合理** | 第 1 项对应达成率最低的 KPI 维度 | 检查首行与复盘最低项对应 |
| **覆盖周期正确** | ISO 周编号与日期范围匹配 | 检查 W 编号计算是否正确 |

### P2 - 可以有

| 检查项 | 检查内容 | 验证方法 |
|---------|---------|---------|
| **Git 提交统计准确** | 本周提交数量与 git log 一致 | 运行 git log 核对 |
| **跨周对比** | 引用上周周报进行 KPI 趋势对比 | 检查是否引用上周 docs/周报/ 文件 |
| **改进建议可执行性高** | 每条建议有明确执行步骤和预期收益 | 检查改进措施列 |
