## 检查项

### P0 - 必须通过

| 检查项 | 检查内容 | 验证方法 |
|---------|---------|---------|
| **文档头部完整** | 含版本、日期、维护者、覆盖周期、关联功能目录 | 检查是否有完整头部元数据 |
| **KPI 总表覆盖范围** | 每个本周活跃的 `docs/<功能名>/` 各占一行，或明确声明无活跃案例 | 对照 `docs/` 目录逐一验证 |
| **KPI 量化与证据** | 五个 KPI 维度有具体数值（百分比或轮次），证据块可追溯 | 检查总表与证据引用 |
| **维度综合有据** | ✅/🟡/❌ 及一句依据与 06/05 等来源一致 | 对照实施总结与检查清单 |
| **本周复盘非空** | 含进展/亮点与问题根因等可验证条目，非仅有表格 | 阅读「本周复盘」小节 |
| **链路全景图正确** | flowchart 为 KPI→复盘根因→后期规划，无 OKR 节点 | 检查 Mermaid 语法与因果链 |
| **后期规划总表可执行** | 每项有类型标签、KPI 指标、验证方式、证据 | 检查类型列与验证方式列 |
| **优先级矩阵图正确** | quadrantChart 展示影响度×实施难度分布 | 检查 Mermaid 语法与坐标合理性 |
| **防幻觉** | 无虚构数据、无虚构案例、无 D 类陈述 | 逐一核对数据来源 |

### P1 - 应该通过

| 检查项 | 检查内容 | 验证方法 |
|---------|---------|---------|
| **链路因果关系准确** | 薄弱 KPI 正确连到根因与后期规划首项 | 检查 flowchart 连线 |
| **优先级矩阵坐标合理** | 关联未达标 KPI 或高危根因的改进 x 值更高 | 检查 quadrantChart 坐标 |
| **后期规划优先级合理** | 第 1 项对应最薄弱 KPI 或最高风险根因 | 对照 KPI 总表与复盘小节 |
| **覆盖周期正确** | 使用自然周起止日期（周一~周日）且与正文一致 | 检查起止日期是否为同一自然周 |

### P2 - 可以有

| 检查项 | 检查内容 | 验证方法 |
|---------|---------|---------|
| **Git 提交统计准确** | 本周提交数量与 git log 一致 | 运行 git log 核对 |
| **跨周对比** | 引用上周周报进行 KPI 或结论趋势对比 | 检查是否引用上周 `docs/周报/` 文件 |
| **改进建议可执行性高** | 每条建议有明确执行步骤和预期收益 | 检查改进措施列 |
