# 周报检查清单

> **关联规范**: [周报规范](../rules/周报.md) | [通用文档检查清单](./通用文档.md)

## P0 - 必须通过

- 文档头部完整（版本、日期、维护者、覆盖周期、关联功能目录）
- KPI 数据来源自动化（`collect-weekly-kpi.js --with-logs` 已执行且输出被采用）
- KPI 总表覆盖范围（每个活跃功能目录各占一行）
- KPI 量化与证据（五个维度有具体数值及路径）
- 维度综合有据（✅/🟡/❌ 及依据与 06/05 等来源一致）
- 本周复盘非空（含进展/亮点与问题根因）
- 链路全景图正确（KPI→复盘根因→后期规划，无 OKR 节点）
- 后期规划总表可执行（每项有类型标签、KPI 指标、验证方式、证据）
- 优先级矩阵图正确（quadrantChart）
- 防幻觉（无虚构数据、案例、D 类陈述）

## P1 - 应该通过

- Git 统计与脚本输出一致
- 链路因果关系准确（薄弱 KPI 连到根因与规划首项）
- 后期规划优先级合理（首条对应最薄弱 KPI）
- 覆盖周期正确（自然周起止）
- 步骤 6 已执行
- doc-mermaid-expert 已调用

## P2 - 可以有

- 跨周对比
- 改进建议可执行性高
