# 需求文档检查清单

> **关联规范**: [需求文档规范](../rules/需求文档.md) | [通用文档检查清单](./通用文档.md)

## P0 - 必须通过

- 用户故事表格存在且完整（4 列：用户故事、验收标准、过程生成文档、产出智能文档）
- 用户故事使用 🔴🟡🟢 优先级图标
- 表格只包含一个用户故事，且含"主要操作场景"部分
- 功能概述章节存在
- 文档规范章节含"一个用户故事对应自己的编号集"
- 验收标准章节存在且位置正确（用户故事后、功能详情前）
- 功能详情章节存在

## P1 - 应该通过

- 功能概述含核心价值列表
- 用户故事格式规范（作为[角色]，我想要[功能]，以便[价值]）
- 过程生成文档链接指向同目录编号集文件
- 验收标准按 P0/P1/P2 分级且可测试
- 使用场景章节存在且采用"背景-操作-结果"格式
- 功能点含功能说明、价值、解决的痛点

## P2 - 可以有

- 功能点使用 H3 子章节
- 段落划分合理、列表使用恰当、表格格式规范
