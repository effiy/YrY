# 项目报告检查清单

> **关联规范**: [项目报告规范](../rules/项目报告.md) | [通用文档检查清单](./通用文档.md)

## P0 - 必须通过

- 文档头部完整
- 报告范围明确（纳入/排除/不确定项均有说明）
- 交付摘要可追溯（交付目标、核心结果、变更规模、验证结论均有来源）
- 变更文件列表完整（与 git diff 或用户提供的清单一致）
- 每个变更文件有前后对比
- 变更前内容真实（来自旧版本，非推断）
- 路径真实有效
- 汇总表完整（每个变更文件出现且含验证覆盖状态）
- 验证结果真实（已执行/未执行/失败如实记录）
- 自我改进章节存在且证据化

## P1 - 应该通过

- 变更概览清晰（能从变更域理解整体变化）
- 影响评估完整（覆盖 5 个影响面）
- 风险与遗留项明确
- 影响评估合理（无/低/中/高与实际改动范围一致）
- 关键截取得当

## P2 - 可以有

- 读者分层友好
- 变更规模数量与列表一致
