# 通用文档检查清单

> **关联规范**: [通用文档规范](../rules/通用文档.md) | [证据与不确定性](../../../shared/evidence-and-uncertainty.md)

## P0 - 必须通过

- 文档头部结构完整（按通用文档规范顺序包含所有元素）
- 元数据完整（版本号、最后更新日期、维护者）
- 版本号格式正确（`v{主版本}.{次版本}`）
- 日期格式正确（`YYYY-MM-DD`）
- 关联文档行、Git 分支行、文档时间行均存在
- 头部与正文有 `---` 分隔线
- H1 唯一，标题层级不超过 H3
- Mermaid 图表有说明
- 链接使用相对路径
- 技术断言可溯源
- 无虚构路径与依赖
- 待补充显式化（无来源段落用 `> 待补充（原因：…）`）
- 开放问题可收敛（需人类决策的项不与事实陈述混为一段）

## P1 - 应该通过

- 段落间有空行、长度适中（不超过 3-4 行）
- 列表使用恰当
- 表格格式规范
- 代码块标注语言
- 关联文档存在
- 标题命名清晰

## P2 - 可以有

- Mermaid 节点样式规范
- 图标使用统一
- 章节导航
- 重要信息使用引用块
