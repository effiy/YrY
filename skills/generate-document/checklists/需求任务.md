# 需求任务检查清单

> **关联规范**: [需求任务规范](../rules/需求任务.md) | [通用文档检查清单](./通用文档.md)

## P0 - 必须通过

- 用户故事表格存在且完整（4 列）
- 优先级图标使用（🔴🟡🟢）
- 主要操作场景：P0 用户故事至少 2 个，P1 至少 1 个
- 功能分解图、用户流程图、功能流程图均存在
- 影响分析章节存在，含搜索词与改动点清单、影响链、依赖闭合摘要、未覆盖风险四张子表
- 影响链基于实际搜索（路径和行号来自真实搜索，不得估算）
- 依赖关系已闭合
- 改动范围汇总存在

## P1 - 应该通过

- 完整时序图存在
- 功能详情、验收标准（按 P0/P1/P2 分级且可测试）、使用场景章节存在
- 使用场景格式规范
- 操作场景含前置条件、操作步骤、预期结果、验证关注点

## P2 - 可以有

- 设计文档关联明确
