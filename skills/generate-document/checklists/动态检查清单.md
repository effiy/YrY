# 动态检查清单 · 自检清单

> **关联规范**: [动态检查清单规范](../rules/动态检查清单.md) | [通用文档检查清单](./通用文档.md)

## P0 - 必须通过

- 前置条件已满足（02 需求任务、03 设计文档存在且含主要操作场景/实现章节）
- 文档头部完整
- 必需章节齐全：通用检查 / 主要操作场景验证 / 代码质量检查 / 测试检查 / 检查总结
- 场景数 = 需求任务场景数，每场景含前置条件/操作步骤/预期结果三张表
- 每个场景都链接到需求任务与设计文档对应锚点
- 代码路径真实存在
- 无虚构技能（验证工具只含 find-skills 返回值或"人工复核"）
- 通用检查四行固定
- 未使用模板

## P1 - 应该通过

- 验证方法具体（非泛化表述）
- 优先级判定合理
- 代码质量、测试检查固定行完整
- 链接可解析

## P2 - 可以有

- 检查总结三小节完整
- 图标使用一致
