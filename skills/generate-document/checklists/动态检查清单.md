## 检查项

### P0 - 必须通过

| 检查项 | 检查内容 | 验证方法 |
|--------|---------|---------|
| **前置条件已满足** | 需求任务、设计文档存在且含主要操作场景 | 检查 `docs/<功能名>/02_需求任务.md` 与 `docs/<功能名>/03_设计文档.md` 是否存在并含对应章节 |
| **文档头部完整** | 含版本、日期、维护者、关联文档 | 对照 `rules/动态检查清单.md` 第 1 节 |
| **必需章节齐全** | 通用检查 / 主要操作场景验证 / 代码质量检查 / 测试检查 / 检查总结 | 逐节核对 |
| **场景数一致** | 场景数 = 需求任务场景数 | 对比两份文档的场景数量 |
| **每场景三张表齐全** | 前置条件验证 / 操作步骤验证 / 预期结果验证 均存在 | 逐场景检查子表 |
| **来源锚点存在** | 每个场景都链接到需求任务与设计文档对应锚点 | 点击或文本搜索锚点 |
| **代码路径真实** | 所引用的文件路径在仓库中真实存在 | 文件搜索校验 |
| **无虚构技能** | "验证工具建议"只含 find-skills 返回值或"人工复核" | 逐条核对技能名 |
| **通用检查四行固定** | 完整包含四条固定通用检查项（P0） | 对照规范第 2 节 |
| **无事实-来源缺失** | 没有上游无出处的"虚构步骤/条件/结果" | 抽样核对表格条目 |

### P1 - 应该通过

| 检查项 | 检查内容 | 验证方法 |
|--------|---------|---------|
| **验证方法具体** | 每条检查项的"验证方法"列可执行、与场景相关 | 抽样 5 条检查是否为泛化表述 |
| **优先级判定合理** | P0/P1/P2 分配符合规范"优先级判定" | 随机抽查 |
| **代码质量四行完整** | 含代码风格 / 命名 / 性能 / 安全 四条 | 对照规范第 5 节 |
| **测试检查四行完整** | 含单元测试 / E2E / P0 测试通过 / 测试报告 | 对照规范第 6 节 |
| **链接可解析** | 所有锚点与相对路径可跳转 | 抽查 3-5 条链接 |

### P2 - 可以有

| 检查项 | 检查内容 | 验证方法 |
|--------|---------|---------|
| **检查总结三小节** | 含总体进度 / 待完成项 / 结论 | 对照规范第 7 节 |
| **图标使用一致** | 🏃📋🔍🟢🟡🔴⏳ 使用统一 | 目视检查 |
| **初始状态统一** | 表格"状态"列统一为 ⏳ | 抽查 |
