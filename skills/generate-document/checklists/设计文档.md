# 设计文档检查清单

> **关联规范**: [设计文档规范](../rules/设计文档.md) | [通用文档检查清单](./通用文档.md)

## P0 - 必须通过

- 设计概述、架构设计章节存在
- 整体架构图存在（Mermaid graph TB/LR）
- 影响分析章节存在，含搜索词与改动点清单、影响链、依赖闭合摘要、未覆盖风险
- 影响链基于实际搜索（路径和行号不得估算）
- 依赖关系已闭合
- 改动范围汇总存在
- 修复/变更内容、实现细节、主要操作场景实现章节存在（全文档生成时）

## P1 - 应该通过

- 影响链维度与处置方式标注完整
- 模块划分表格、核心流程图存在
- Mermaid 图表有说明
- 修复前后对比清晰
- 关键代码说明存在
- 场景实现含关键代码路径和验证要点

## P2 - 可以有

- 设计原则列表存在
- Mermaid 节点样式规范
- 测试考虑完整
