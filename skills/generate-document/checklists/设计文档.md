>
> **关联规范**: [设计文档规范](../rules/设计文档.md) | [通用文档规范](../rules/通用文档.md)

> **注**: 通用文档检查项请参考 [通用文档检查清单](./通用文档.md)
>
> **动态适配**: 检查项根据项目类型（前端/后端/全栈/桌面/扩展/移动端）动态组合，不使用固定章节结构假设

## 检查说明

- 检查结果：✅ 通过 / ⚠️ 需改进 / ❌ 不通过
- 所有 **P0 项必须 ✅ 通过**
- **P1 项建议 ✅ 通过**
- **P2 项可选通过**
- 项目类型从 `CLAUDE.md`、`README.md`、`docs/architecture.md` 动态推断

## 检查项

### P0 - 必须通过（所有项目类型通用）

| 检查项 | 检查内容 | 验证方法 |
|---------|---------|---------|
| **文档头部完整** | 包含版本号、最后更新日期、维护者信息 | 检查是否有文档版本、最后更新、维护者信息 |
| **关联文档链接** | 正确链接到需求任务、使用文档、CLAUDE.md | 检查元数据区域的关联文档链接 |
| **设计概述章节** | 有"设计概述"章节，100-200字 | 检查是否有设计概述章节 |
| **架构设计章节** | 有"架构设计"章节 | 检查是否有架构设计章节 |
| **整体架构图** | 包含 Mermaid 架构图（图类型从项目类型动态推断） | 检查是否有 Mermaid 架构图；后端项目可能用 classDiagram/erDiagram，前端项目可能用 graph TB |
| **章节结构适配** | 章节结构根据项目类型动态组合（后端有 API/ER 章节，前端有模块/组件章节） | 检查章节是否与项目类型推断一致 |
| **遵循项目架构** | 从 `docs/architecture.md` 动态推断项目架构模式 | 主观判断是否符合项目架构 |
| **影响分析章节** | 有"影响分析"章节，含搜索词清单、影响链、闭合摘要、未覆盖风险 | 检查影响分析章节完整性 |
| **影响链基于实际搜索** | 每条记录来自实际搜索，不得估算 | 检查文件路径和行号是否真实存在 |

### P0 - 必须通过（按项目类型追加）

| 项目类型推断信号 | 追加检查项 | 验证方法 |
|------------------|-----------|---------|
| 有 API/Controller/Route 代码 | **API 接口设计章节存在** | 检查是否有 API 接口设计章节 |
| 有 ORM/Schema/Database 代码 | **数据模型设计章节存在**（含 ER 图或 DB schema） | 检查是否有 erDiagram 或数据表定义 |
| 有 class/extends/interface | **类图或业务逻辑设计章节存在** | 检查是否有 classDiagram 或流程图 |
| 有 Tauri/Electron 代码 | **IPC 接口设计章节存在** | 检查是否有 IPC 方法签名 |
| 有浏览器扩展 manifest | **manifest/权限设计章节存在** | 检查是否有 manifest.json 配置设计 |
| `docs/auth.md` 存在且非空 | **鉴权流程设计章节存在** | 检查是否有鉴权流程章节 |
| `docs/security.md` 存在且非空 | **安全设计章节存在** | 检查是否有安全设计章节 |

### P1 - 应该通过

| 检查项 | 检查内容 | 验证方法 |
|---------|---------|---------|
| **模块划分表格** | 有完整的模块划分表格（列名从项目架构约定动态推断） | 检查是否有模块划分表格 |
| **核心流程图** | 包含 Mermaid 核心流程图（图类型从项目类型动态推断） | 检查是否有核心流程图 |
| **架构说明清晰** | 每个图表下方都有明确的说明文字 | 检查每个 Mermaid 图表后的说明 |
| **要点说明详细** | 每个技术实现要点都说明"做什么、怎么做、为什么" | 主观判断说明是否详细 |
| **数据流程图存在** | 有 Mermaid 数据流程图（图类型从项目类型动态推断） | 检查是否有数据流程图 |
| **接口规范完整** | 接口规范格式从项目类型推断（前端 props/emits / 后端 API endpoint / 桌面 IPC） | 检查接口规范是否与项目类型匹配 |
| **场景验证要点适配** | 验证要点格式从项目类型推断（前端 UI 交互 / 后端 API 请求响应 / 数据一致性） | 检查验证要点是否与项目类型匹配 |

### P2 - 可以有

| 检查项 | 检查内容 | 验证方法 |
|---------|---------|---------|
| **设计原则列表** | 设计概述中包含设计原则列表（带图标） | 检查是否有设计原则列表 |
| **Mermaid 样式规范** | 图表节点色值符合项目视觉规范 | 检查是否使用标准色值 |
| **测试验证方法** | 说明如何验证修复是否有效（格式从项目类型推断） | 检查是否有测试验证说明 |
| **备选方案讨论** | 如果有多个方案，说明选择理由 | 检查是否有备选方案讨论 |
| **边界情况说明** | 说明边界情况的处理方式 | 检查是否有边界情况说明 |
