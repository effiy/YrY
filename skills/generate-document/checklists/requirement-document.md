# Requirement Document Checklist

> **Related specs**: [Requirement Document Spec](../rules/requirement-document.md) | [General Document Checklist](./general-document.md)

## P0 — Must Pass

- User story table exists and is complete (4 columns: user story, acceptance criteria, process-generated docs, output smart docs)
- User stories use 🔴🟡🟢 priority icons
- Table contains only one user story, with a "main operation scenarios" section
- Feature overview chapter exists
- Document spec chapter contains "one user story maps to its own numbered set"
- Acceptance criteria chapter exists and is in the correct position (after user story, before feature details)
- Feature details chapter exists

## P1 — Should Pass

- Feature overview contains a core-value list
- User story format is standard (As [role], I want [feature], so that [value])
- Process-generated document links point to same-directory numbered-set files
- Acceptance criteria are graded by P0/P1/P2 and are testable
- Usage scenarios chapter exists and uses "background-operation-result" format
- Feature points contain description, value, and pain point solved

## P2 — Nice to Have

- Feature points use H3 sub-chapters
- Paragraph division is reasonable, list usage is appropriate, table format is standard
