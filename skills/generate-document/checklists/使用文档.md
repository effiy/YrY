# 使用文档检查清单

> **关联规范**: [使用文档规范](../rules/使用文档.md) | [通用文档检查清单](./通用文档.md)

## P0 - 必须通过

- 功能介绍完整（100-200 字，含用途、价值、适用人群）
- 快速开始完整（前置条件 + 3-5 步上手流程）
- 操作场景完整（至少 3 个，每个含步骤和预期结果）
- 常见问题完整（覆盖基础/高级/故障排查三类）
- 技术断言与代码对齐（命令、UI 路径、功能名称可追溯，无虚构）

## P1 - 应该通过

- 场景结构统一（含适用情况、操作步骤、预期结果、注意事项）
- 问题分类清晰
- 技巧提示存在（至少 3 条）
- 与设计文档对齐

## P2 - 可以有

- 快捷键表格、术语表、命令速查表
