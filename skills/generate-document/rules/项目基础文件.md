# 项目基础文件规范

> 约束 `/generate-document init` 生成的项目基础文件和 `docs/项目初始化/` 全文档编号集。

## 可重复运行更新策略（P0）

1. **优先更新事实**：技术栈、目录、命令、入口等可推断内容必须刷新
2. **保留人工补充**：团队约定/经验沉淀，未与事实冲突则保留
3. **冲突标注**：`> 待确认（原因：…；建议以代码为准）` + 证据路径
4. **可观测更新**：更新头部时间，`06_实施总结.md` 记录本轮摘要
5. **哨兵块安全重写**：`<!-- AUTO-GENERATED:BEGIN -->...<!-- AUTO-GENERATED:END -->` 区段可重写，块外保留

## 通用生成规则（P0）

1. **防幻觉**：内容必须来自代码扫描和文件读取，不得虚构
2. **不确定标注**：`> 待补充（原因：…）`
3. **版本信息真实**：维护者填大模型名称，工具填 Claude Code / Cursor
4. **相对路径**
5. **01-07 按 5 步工作流**：功能名 = "项目初始化"
