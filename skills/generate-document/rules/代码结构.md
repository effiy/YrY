# 代码结构规范

## 目录组织

```
<仓库根>/
├── .claude/        # Claude Code 配置和技能
├── <shared>/       # 共享模块（按项目现状命名）
│   ├── components/ # 共享组件（可选）
│   ├── scripts/    # 脚本（import-docs 等，可选）
│   ├── styles/     # 全局样式（可选）
│   └── utils/      # 工具函数（可选）
├── docs/           # 项目文档
├── src/            # 应用代码
│   ├── core/       # 核心配置与服务
│   └── views/      # 各应用视图
├── index.html      # SPA 入口
└── CLAUDE.md       # 项目说明
```

## 核心架构模式

### 入口初始化模式

项目通常有统一入口初始化方式（`initApp` / `createApp` / 工厂函数等），需遵循项目现状。

### Hooks 工厂模式

每个应用视图使用三文件 hooks 模式：

| 文件 | 内容 | 调用方 |
|------|------|--------|
| `hooks/store.js` | `createStore()` — 返回 `{ key: Vue.ref() }` | createBaseView |
| `hooks/useComputed.js` | `useComputed(store)` — 返回 `{ key: Vue.computed() }` | createBaseView |
| `hooks/useMethods.js` | `useMethods(store)` — 返回 `{ key: function }` | createBaseView |

Store 内部可拆分操作模块。

### 模块导入规范

使用**绝对路径**导入：`/src/views/appA/hooks/store.js`，避免相对路径。

## 应用视图结构

```
src/views/<app>/
├── components/     # 应用特有组件
├── constants/      # 常量
├── hooks/          # hooks 工厂
├── styles/         # 应用样式
├── utils/          # 应用工具
├── index.html      # 应用 HTML 入口
└── index.js        # 应用 JS 入口（createBaseView）
```

## 检查清单

- 共享层与应用层放置规则已在项目文档中明确
- hooks 三文件模式完整（store.js / useComputed.js / useMethods.js）
- 使用绝对路径导入模块
- Store 使用 `Vue.ref` 工厂，不直接 reactive
- 操作模块使用依赖注入模式
- 方法通过 useMethods 组合器组织
- 组件全局注册并触发 Loaded 事件
