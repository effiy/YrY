# 代码结构规范


## 核心架构模式

### 入口初始化模式

项目通常会有统一的入口初始化方式（例如 `initApp` / `createApp` / 工厂函数等）。架构与实现需遵循项目现状，避免引入与仓库不一致的“默认入口模式”。

### Hooks 工厂模式

每个应用视图使用三文件 hooks 模式：

| 文件 | 内容 | 调用方 |
|------|------|--------|
| `hooks/store.js` | `createStore()` — 返回 `{ key: Vue.ref() }` | createBaseView |
| `hooks/useComputed.js` | `useComputed(store)` — 返回 `{ key: Vue.computed() }` | createBaseView |
| `hooks/useMethods.js` | `useMethods(store)` — 返回 `{ key: function }` | createBaseView |

Store 内部可进一步拆分操作模块：

```
hooks/
├── store.js              # 主 Store 入口
├── storeSessionsOps.js   # 会话操作模块
├── storeFileTreeOps.js   # 文件树操作模块
├── storeUiOps.js         # UI 状态操作模块
├── useComputed.js        # 计算属性组合器
├── useMethods.js         # 方法组合器
└── methods/              # 方法子模块
```

### 共享组件模式（如存在）

若项目存在“共享组件层”，其放置路径、导出入口与注册方式应以仓库现状为准（例如集中导出入口、或由框架/构建系统按需引入）。

### 模块导入规范

项目使用**绝对路径**导入：

```javascript
// ✅ 推荐：绝对路径
import { createStore } from '/src/views/appA/hooks/store.js';

// ❌ 避免：相对路径
import { createStore } from '../hooks/store.js';
```

### 安全执行模式

```javascript
const result = safeExecute(() => {
    return someOperation();
}, '操作名称');
```

## 应用视图结构

每个应用视图遵循统一结构：

```
src/views/<app>/
├── components/     # 应用特有组件
├── constants/      # 常量
├── hooks/          # hooks 工厂
├── styles/         # 应用样式
├── utils/          # 应用工具
├── index.html      # 应用 HTML 入口
└── index.js        # 应用 JS 入口（createBaseView）
```

## 检查清单

开发新模块时，请确认：

- 共享层与应用层的放置规则已在项目文档中明确，并在实现中保持一致
- hooks 三文件模式完整（store.js / useComputed.js / useMethods.js）
- 使用绝对路径导入模块
- Store 使用 `Vue.ref` 工厂，不直接 reactive
- 操作模块使用依赖注入模式
- 方法通过 useMethods 组合器组织
- 组件全局注册并触发 Loaded 事件
