## 核心架构模式

### createBaseView 工厂

`createBaseView` 是应用初始化的核心工厂函数（位于 `cdn/utils/view/baseView.js`）：

```javascript
import { createStore } from '/src/views/aicr/hooks/store.js';
import { useComputed } from '/src/views/aicr/hooks/useComputed.js';
import { useMethods } from '/src/views/aicr/hooks/useMethods.js';
import { createBaseView } from '/cdn/utils/view/baseView.js';

const app = await createBaseView({
    createStore,
    useComputed,
    useMethods,
    components: ['AicrPage', 'AicrHeader', 'YiModal'],
    onMounted: null
});
```

### Hooks 工厂模式

每个应用视图使用三文件 hooks 模式：

| 文件 | 内容 | 调用方 |
|------|------|--------|
| `hooks/store.js` | `createStore()` — 返回 `{ key: Vue.ref() }` | createBaseView |
| `hooks/useComputed.js` | `useComputed(store)` — 返回 `{ key: Vue.computed() }` | createBaseView |
| `hooks/useMethods.js` | `useMethods(store)` — 返回 `{ key: function }` | createBaseView |

Store 内部可进一步拆分操作模块：

```
hooks/
├── store.js              # 主 Store 入口
├── storeSessionsOps.js   # 会话操作模块
├── storeFileTreeOps.js   # 文件树操作模块
├── storeUiOps.js         # UI 状态操作模块
├── useComputed.js        # 计算属性组合器
├── useMethods.js         # 方法组合器
└── methods/              # 方法子模块
```

### CDN 组件模式

共享组件在 `cdn/components/` 下开发，使用 HTML + JS + CSS 三文件结构：

```
cdn/components/common/buttons/YiButton/
├── index.js       # 组件逻辑（Options API 或 registerGlobalComponent）
├── template.html  # 组件模板
└── index.css      # 组件样式（可选）
```

组件通过 `cdn/components/index.js` barrel export：

```javascript
export { default as YiButton } from './common/buttons/YiButton/index.js';
export { default as YiModal } from './common/modals/YiModal/index.js';
```

### registerGlobalComponent

组件注册使用 `cdn/utils/view/componentLoader.js`：

```javascript
import { registerGlobalComponent } from '/cdn/utils/view/componentLoader.js';

registerGlobalComponent({
    name: 'YiButton',
    html: '/cdn/components/common/buttons/YiButton/template.html',
    css: '/cdn/components/common/buttons/YiButton/index.css',
    props: { variant: String },
    emits: ['click'],
    computed: { ... },
    methods: { ... }
});
```

### 模块导入规范

项目使用**绝对路径**导入：

```javascript
// ✅ 推荐：绝对路径
import { createStore } from '/src/views/aicr/hooks/store.js';
import { logInfo } from '/cdn/utils/core/log.js';

// ❌ 避免：相对路径
import { createStore } from '../hooks/store.js';
```

### 安全执行模式

```javascript
import { safeExecute } from '/cdn/utils/core/error.js';

const result = safeExecute(() => {
    return someOperation();
}, '操作名称');
```

## 应用视图结构

每个应用视图遵循统一结构：

```
src/views/<app>/
├── components/     # 应用特有组件
├── constants/      # 常量
├── hooks/          # hooks 工厂
├── styles/         # 应用样式
├── utils/          # 应用工具
├── index.html      # 应用 HTML 入口
└── index.js        # 应用 JS 入口（createBaseView）
```

## 检查清单

开发新模块时，请确认：

- 共享组件放在 `cdn/components/` 下，并在 `cdn/components/index.js` 中 export
- 应用特有组件放在 `src/views/<app>/components/` 下
- hooks 三文件模式完整（store.js / useComputed.js / useMethods.js）
- 使用绝对路径导入模块
- Store 使用 `Vue.ref` 工厂，不直接 reactive
- 操作模块使用依赖注入模式
- 方法通过 useMethods 组合器组织
- 组件全局注册并触发 Loaded 事件
