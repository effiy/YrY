# 通用文档规范

## 文档头部（严格遵循）

```markdown
# {文档标题}

> **文档版本**: v{版本号} | **最后更新**: {YYYY-MM-DD} | **维护者**: {大模型名称} | **工具**: {Claude Code / Cursor}
>
> **关联文档**: [关联文档1](../目录/{文件名}.md) | [关联文档2](../目录/{文件名}.md)
>
> **Git 分支**: {branch-name}
>
> **文档开始时间**: {HH:mm:ss} | **文档最后更新时间**: {HH:mm:ss}
>

[章节1](#章节1) | [章节2](#章节2) | [章节3](#章节3)

---
```

头部各部分（按顺序）：H1 标题 → 空行 → 元数据行1（版本/日期/维护者） → 空行 → 关联文档行 → 空行 → Git 分支行 → 空行 → 文档时间行 → 空行 → 章节导航（推荐） → 分隔线 `---`。

**元数据规范**：
- 版本号：`v{主版本}.{次版本}`，新文档从 `v1.0` 开始
- 最后更新：`YYYY-MM-DD`
- 维护者：大模型名称（如 `Claude Sonnet 4.6`）
- Git 分支：生成时所在分支名
- 文档时间：`HH:mm:ss`（24 小时制）；无法获取开始时间写"未知（未记录）"

## 标题层级

- **H1**（`#`）：文档主标题，唯一
- **H2**（`##`）：主要章节
- **H3**（`###`）：子章节
- **H4 及以下**：禁止使用

## Mermaid 图表规范

### 图表类型选择

- 架构图/功能分解/模块划分：`graph TB`
- 组件关系/流程步骤/依赖：`graph LR`
- 带判断的流程：`flowchart TD`
- 时序交互/数据流向/调用链：`sequenceDiagram`

### 节点样式色值

- 正向/新增/核心：`fill:#ccffcc`
- 反向/问题：`fill:#ffcccc`
- 保持/中性/规范：`fill:#e1f5ff`

### 中文与特殊字符节点（强制）

- 节点文本含中文、空格、括号、冒号等特殊字符时，必须用 **双引号** 包裹
- 正确：`A["用户登录"] --> B["验证失败（密码错误）"]`
- 错误：`A[用户登录] --> B[验证失败（密码错误）]`
- 禁止在节点文本中使用全角括号 `（）`、全角冒号 `：` 等未转义字符

每个图表下方必须有说明文字。

## 格式排版

- 段落不超过 3-4 行，段间空行，每段一个中心思想
- 并列项用无序列表（`-`），步骤/流程用有序列表（`1.`）
- 对比性内容用表格，代码示例用代码块标注语言类型
- 关键术语用**粗体**，文件名/路径/变量名用 `行内代码`
- 文档链接统一使用相对路径
