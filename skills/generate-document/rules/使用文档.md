# 使用文档规范

> 面向最终用户，以操作场景为主、常见问题为辅。上游：03/02/01。

## 文档结构

### 1. 头部

标准头部 + 导航锚点（功能介绍 | 快速开始 | 操作场景 | 常见问题 | 技巧提示）。

### 2. 功能介绍

100-200 字 + 3 核心特性(🎯⚡🔧) + 适用人群

### 3. 快速开始

前置条件(检查清单) + 30秒上手(3-5步)

### 4. 操作场景（核心）

3-5 个场景：适用情况 → 操作步骤 → 预期结果 → 注意事项(✅/❌)

### 5. 常见问题

5-10 个：💡基础 / ⚙️高级 / 🔧故障排查

### 6. 技巧与提示

💡实用技巧(3-5) / ⌨️快捷键(如适用) / 📚最佳实践(2-3)

### 7. 附录（可选）

术语表 / 命令速查 / 相关资源

## 写作原则

用户视角 / 场景驱动 / 步骤清晰 / 问题导向
