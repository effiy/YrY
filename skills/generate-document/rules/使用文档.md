# 使用文档规范

## 文档概述

- 使用文档基于设计文档、需求任务和需求文档，面向最终用户
- 以操作场景为主，常见问题为辅，帮助用户快速上手使用功能
- 上游文档：设计文档、需求任务文档、需求文档
- 下游文档：无（终端用户文档）

## 文档结构（严格遵循）

### 1. 文档头部（必须包含）

```markdown
# {功能名称}使用指南

> **文档版本**: v1.0 | **最后更新**: {日期} | **维护者**: {大模型名称} | **工具**: {Claude Code / Cursor}
>
> **关联文档**: [需求文档](./01_需求文档.md) | [需求任务](./02_需求任务.md) | [设计文档](./03_设计文档.md)
>

[功能介绍](#功能介绍) | [快速开始](#快速开始) | [操作场景](#操作场景) | [常见问题](#常见问题) | [技巧提示](#技巧与提示)

---
```

### 2. 功能介绍

- 简要介绍功能的用途和价值，100-200字
- 列出 3 个核心特性，使用 🎯⚡🔧 图标
- 说明谁应该使用这个功能

### 3. 快速开始

#### 前置条件

- 使用检查清单格式列出前置条件

#### 30秒上手

- 3-5 个简洁明了的步骤让用户快速上手
- 最后添加完成提示 🎉

### 4. 操作场景（核心部分）

3-5 个主要场景，每个场景包含：

- **适用情况**：什么情况下使用这个场景，1-2句话
- **操作步骤**：使用表格或编号列表，清晰易跟随
- **预期结果**：描述操作完成后的结果，用户能看到什么或能做什么
- **注意事项**：✅正面提示、❌负面警告

使用 📋🎨 等图标标识不同场景。

### 5. 常见问题

5-10 个问题，分三类组织：

- **💡 基础问题**：用户最可能问的基础问题
- **⚙️ 高级问题**：涉及高级用法的问题
- **🔧 故障排查**：常见错误或故障的解决方案

### 6. 技巧与提示

- **💡 实用技巧**：3-5 个实用技巧
- **⌨️ 快捷键（如适用）**：快捷键表格
- **📚 最佳实践**：2-3 个最佳实践建议

### 7. 附录（可选）

- **术语表**：关键术语的解释
- **命令速查表**：命令行操作速查表
- **相关资源**：相关链接和资源

## 写作原则

- **用户视角**：站在用户角度思考，避免技术术语堆砌
- **场景驱动**：以实际操作场景为主，每个场景都有明确的"适用情况"
- **步骤清晰**：操作步骤使用编号列表或表格，易于跟随
- **问题导向**：常见问题要覆盖用户最可能遇到的困惑
- **视觉友好**：使用图标、表格、列表提升可读性

## 保存位置

`docs/<功能名>/04_使用文档.md`

## 质量检查清单

### P0 - 必须通过

- **功能介绍**：100-200字，说明用途和价值
- **快速开始**：有前置条件和3-5步上手步骤
- **操作场景**：3-5个场景，每个有明确步骤
- **常见问题**：5-10个用户可能遇到的问题

### P1 - 应该通过

- **场景完整性**：每个场景都有"适用情况"、"操作步骤"、"预期结果"、"注意事项"
- **问题分类**：常见问题按基础/高级/故障排查分类组织
- **技巧提示**：3-5个实用技巧
- **视觉友好**：适当使用表格、列表、图标提升可读性

### P2 - 可以有

- **快捷键**：如有适用的快捷键，提供快捷键表格
- **术语表**：包含关键术语的解释
- **命令速查表**：如有命令行操作，提供速查表
- **最佳实践**：提供2-3个最佳实践建议
