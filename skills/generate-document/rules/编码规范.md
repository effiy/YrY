# 编码规范

## 语法规范

- ES6+，优先 `const`，需重新赋值用 `let`，避免 `var`
- 4 空格缩进，语句末尾分号
- 单引号 `'`，拼接用模板字符串 `${}`
- 对象字面量简写、展开运算符、解构赋值
- 优先箭头函数，异步用 `async/await`

## 命名规范

- 变量/函数：camelCase；布尔变量：`is`/`has`/`can`/`should` 前缀
- 函数前缀：`get`/`set`/`load`/`save`/`create`/`update`/`delete`/`handle`/`toggle`/`init`/`reset`
- 组件名/组件文件：PascalCase
- 常量：UPPER_SNAKE_CASE
- 工具文件：camelCase；样式文件：kebab-case

## 注释与文件组织

- 文件顶部含用途注释；公共函数含 JSDoc
- 内联注释解释"为什么"而非"是什么"
- 标记：`TODO`/`FIXME`/`HACK`/`NOTE`/`DEPRECATED`
- 导入顺序：外部库 → 核心工具 → 项目工具 → 核心服务 → 本地模块
- 优先命名导出；单文件建议不超过 500 行

## Vue 3 组合式 API

- 使用 Composition API，状态用 `ref` 定义（不用 `reactive`）
- 状态通过工厂函数创建，始终通过 `.value` 访问
- 不要解构 ref；使用 `Vue.computed` 创建计算属性

## 状态管理

- 使用工厂模式创建 Store
- 操作模块使用工厂函数，接收依赖项作为参数
- 在 `useMethods.js` 中组合所有方法模块

## 组件开发

- 三文件：`index.js`/`index.css`/`template.html`
- 使用 `registerGlobalComponent` 注册组件
- Props 必须定义类型；使用 `emits` 声明事件
- 组件在 `window` 上注册并触发 Loaded 事件

## 错误处理

- 可能抛异常的代码必须用 `safeExecute`/`safeExecuteAsync` 包装
- 使用 `createError` 创建标准化错误对象
- 使用 `showErrorMessage`/`showSuccessMessage` 展示消息
- 使用 `safeGet`/`safeGetPath` 安全访问对象属性

## 模块设计

- 单一职责；使用工厂函数模式便于依赖注入和测试
- 相关方法组织在一起，使用共同前缀
- 谨慎使用 `window` 暴露，仅用于需全局访问的功能，且有 try-catch 保护
