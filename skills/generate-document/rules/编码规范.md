# 编码规范

## 核心原则

- 思考先于编码：明确假设，不确定时询问
- 简单性优先：最小代码解决问题，不添加推测性功能
- 精准修改：只触碰必须的内容，不"改进"不相关代码
- 目标驱动：定义可验证的成功标准

## 语法规范

- 使用 ES6+ 语法，优先 `const`，需要重新赋值时用 `let`
- 避免使用 `var`
- 使用 4 空格缩进
- 语句末尾使用分号
- 字符串使用单引号 `'`，拼接用模板字符串 `${}`
- 使用对象字面量简写、展开运算符 `...`、解构赋值
- 优先使用箭头函数，异步函数用 `async/await`

## 命名规范

- 变量/函数：camelCase（小驼峰）
- 布尔变量：用 `is`/`has`/`can`/`should` 前缀
- 函数前缀：`get`/`set`/`load`/`save`/`create`/`update`/`delete`/`handle`/`toggle`/`init`/`reset`
- 组件名：PascalCase（大驼峰）
- 组件文件：组件名作为目录名，包含 `index.js`/`index.css`/`template.html`
- 常量：UPPER_SNAKE_CASE（大写下划线分隔）
- 工具文件：camelCase
- 样式文件：kebab-case（短横线分隔）

## 注释规范

- 文件顶部包含文件头注释，描述用途和作者
- 公共函数必须包含 JSDoc 注释
- 内联注释解释「为什么」而不是「是什么」
- 使用标准标记：`TODO`/`FIXME`/`HACK`/`NOTE`/`DEPRECATED`

## 文件组织规范

- 导入顺序：外部库 → 核心工具 → CDN 工具 → 核心服务 → 本地模块
- 优先使用命名导出，默认导出仅用于主函数/组件
- 单个文件建议不超过 500 行

## Vue 3 组合式 API 规范

- 使用 Composition API，状态用 `ref` 定义（不用 `reactive`）
- 状态通过工厂函数创建
- 始终通过 `.value` 访问和修改 ref 值
- 不要解构 ref（会失去响应性）
- 使用 `Vue.computed` 创建计算属性
- 使用 `onMounted`/`onUnmounted` 等生命周期钩子

## 状态管理规范

- 使用工厂模式创建 store
- 操作模块使用工厂函数，接收依赖项作为参数
- 通过参数传递依赖，而不是硬编码导入
- 在 `useMethods.js` 中组合所有方法模块

## 组件开发规范

- 每个组件包含三个文件：`index.js`/`index.css`/`template.html`
- 使用 `registerGlobalComponent` 注册组件
- Props 必须定义类型，必填项不设默认值，可选项设置默认值
- 使用 `emits` 声明事件，事件名用 camelCase
- 组件在 `window` 上注册并触发加载事件：`window.{ComponentName}` + `{ComponentName}Loaded` 事件

## 错误处理规范

- 所有可能抛出异常的代码必须使用 `safeExecute` 或 `safeExecuteAsync` 包装
- 使用 `createError` 创建标准化错误对象
- 使用 `showErrorMessage`/`showSuccessMessage` 展示消息，避免直接用 `alert`
- 使用 `safeGet`/`safeGetPath` 安全访问对象属性
- 启用浏览器扩展错误过滤器

## 模块设计规范

- 单一职责：每个模块/函数只做一件事
- 使用工厂函数模式，便于依赖注入和测试
- 相关方法组织在一起，使用共同前缀
- 谨慎使用 `window` 暴露，仅用于需要全局访问的功能，且有 try-catch 保护
