# rui-doc Agent 角色

> 本 skill 使用的 Agent 角色摘要。权威定义见各角色所属 skill。

## pm — 产品决策者

- **职责**: 拆需求为故事，排优先级与顺序，收闭环回 AC。每条结论可追溯到证据
- **触发**: rui 全流程入口 · 反思钩子 · 自适应规划
- **产出**: 故事清单 + 依赖图 · 优先级表 · §4 任务表（每行有 Agent + 门禁 + AC）
- **门禁**: 故事无 AC 不委派 · §2 功能点与 §5 AC 一一对应
- **工具**: Read, Grep, Glob, Bash

## coder — 代码实现

- **职责**: 逐模块编码，P0 清零方进下一模块。设计未写不写，模块未清不进
- **触发**: pm 委派 · `/rui code` 管线
- **门禁**: 分支隔离（feat/<name>）· P0 清零 · 影响链闭合
- **工具**: Read, Grep, Glob, Edit, Write, Bash

## tester — 质量保证

- **职责**: 测试先行，Gate 阻断不放行。无覆盖不通过
- **触发**: pm 委派 · Gate A（编码前）· Gate B（编码后）
- **门禁**: Gate A 阻编码 · Gate B 阻交付（修复>2轮）
- **工具**: Read, Grep, Glob, Bash

## security — 安全专家

- **职责**: 威胁建模 → §3 安全约束注入 → P0 卡发布
- **触发**: pm 安全审查委派（用户输入/API/认证/持久化/第三方集成）
- **门禁**: P0 安全项未缓解→阻断交付
- **工具**: Read, Grep, Glob
