# rui-doc 文档管线

> 需求到文档基线的生成规则，独立于实现细节。

## 三种模式

| 模式 | 输入 | 行为 | 适用场景 |
|------|------|------|---------|
| 默认 | 文本需求描述 | pm 拆需求为故事 → coder 补齐设计文档 | 从零开始 |
| --from-code | 源码目录 | 从现有代码反向生成文档基线 | 遗留代码文档化 |
| --from-local | 本地文档 | 从已有本地文档提取并标准化 | 迁移已有文档 |

## 主流程

```
pm 解析需求 → 拆分故事 → 逐故事串行:
  分支隔离门禁 → 场景文档 → 架构图 → 知识图谱 → 安全审计 → P0 检查 → 完成
```

### 详细步骤

| 步骤 | 操作 | Agent | 输入 | 输出 | 阻断条件 |
|------|------|-------|------|------|---------|
| 1 | 解析需求 | pm | 用户输入 | 结构化需求描述 | 输入为空 |
| 2 | 拆分故事 | pm | 需求描述 | 故事列表 | 需求不可拆分 |
| 3 | 分支隔离 | — | 故事名 | feat/<name> 分支 | 非 feat 分支 |
| 4 | 场景文档 | pm + tester | 故事需求 | 场景-N-<slug>.md | AC 不完整 |
| 5 | 架构图 | coder | 场景文档 | 架构图 HTML | 无架构信息 |
| 6 | 知识图谱 | pm → coder | 场景文档 | 知识图谱 | 节点不完整 |
| 7 | 安全审计 | security | 场景文档 | §3 安全约束 | P0 安全项 |
| 8 | P0 检查 | — | 全部文档 | P0 清单 | 任何 P0 未清 |

## 文档基线产出

| 文件 | 阶段 | Agent | 必选 | 公式 |
|------|------|-------|:---:|------|
| 故事任务.md | 文档生成 | pm | ✓ | F.story.task |
| 场景-N-<slug>.md (§0 + §1) | 文档生成 | pm + tester | ✓ | F.story.scene |
| 架构图 | 文档生成 | coder | — | — |
| 知识图谱 | 文档生成 | pm → coder | — | — |
| 安全审计 | 文档生成 | security | — | — |

## 多故事串行

多故事时逐故事串行执行，前一故事 P0 清零后再进下一故事。

**串行规则**:
1. 故事按拆分顺序依次执行
2. 故事 N 的 P0 全部清零后，故事 N+1 才能开始
3. 任一故事 P0 未清零，后续故事全部阻塞
4. 超时策略: 单故事 doc 阶段 ≤ 30 分钟

## 质量门禁

| 门禁 | 检查方式 | 通过条件 | 不通过处置 |
|------|---------|---------|-----------|
| 分支隔离 | `node lib/branch-check.mjs` | 当前在 `feat/<name>` 分支 | 创建/切换分支 |
| P0 检查 | 检查清单逐项对照 | 全部 P0 通过 | 自修复 ≤ 2 轮 |
| 文档公式 | 对照 F.story.task / F.story.scene | 必填字段完整 | 补字段 |
| 安全审计 | security agent 独立审查 | 无 P0 安全项 | 注入安全任务 |
| 术语一致性 | 领域语言 Avoid 对照 | 无禁用别名 | 替换为标准术语 |

## 降级策略

| 情况 | 降级行为 | 恢复方式 |
|------|---------|---------|
| 需求不可拆分 | 单故事执行 | 手动拆分后重跑 |
| 架构信息不足 | 跳过架构图，标注 `no-arch` | 补充架构信息后重跑 |
| 安全审查无 security agent | pm 代行基础安全审查 | 后续补充完整审查 |
| P0 自修复超 2 轮 | `doc-p0` 阻断，报告给用户 | 人工修复后重跑 |
| 分支创建失败 | 提示用户手动创建 | 手动创建后重试 |

## 核心规则

| # | 规则 | 设计理由 | 违反后果 |
|---|------|---------|---------|
| 1 | 写故事文档也走分支隔离 | 变更隔离 — 文档和代码同等对待 | 文档变更污染 main |
| 2 | 全程只读源码 | 文档阶段不修改代码 | 文档阶段引入代码变更 |
| 3 | 多故事串行，前一 P0 清零再进下一 | 清先于进 — 铁律 | 质量问题累积 |
| 4 | P0 自修复 ≤ 2 轮 | 防止无限循环 — 超限阻断 | 无限循环消耗资源 |
| 5 | 安全审计不可跳过 | 安全基线 — 安全不可绕过 | 安全漏洞遗漏 |
| 6 | 文档公式必填字段完整 | 文档质量 — 基线标准化 | 文档不完整 |
