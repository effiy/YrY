# rui-bundle-analyze 分析规则

> 项目体积与依赖结构分析的方法论和规则，独立于实现细节。

## 设计原则

1. **零配置** — 无需 webpack 插件、无需 source-map、无需构建，直接对源码目录运行
2. **只读安全** — 不修改任何源码文件，不执行任何项目代码
3. **自包含输出** — HTML 报告仅依赖 D3 CDN，其余全部内联；JSON 输出可被管线消费
4. **渐进增强** — 基础模式即时可用，高级特性（趋势对比、基线管理）按需启用

## 分析方法论

### 文件体积分析

- 递归目录遍历 → 文件 stat 取 size → 构建嵌套层级 → D3 squarified treemap 布局
- 复杂度 O(n)，n = 项目文件数
- Treemap 采用 Bruls-Huizing-van Wijk squarified 算法，确保矩形尽量接近正方形
- 面积严格与文件字节数成比例

### 依赖关系分析

- 正则匹配 import/require 语句
- 路径解析为项目相对路径
- 构建有向图 G = (V, E)，V = 文件节点，E = import 边
- 循环依赖检测: Tarjan SCC 算法，O(V+E)

### 颜色编码策略

使用 Tableau10 色板，按文件扩展名分配颜色。相同类型文件颜色一致，便于跨目录识别同类文件分布。

### 排除规则

| 类别 | 目录/扩展名 | 原因 |
|------|-----------|------|
| 硬排除目录 | `node_modules/`, `.git/`, `.claude/`, `.memory/`, `dist/`, `build/`, `.next/`, `.nuxt/`, `coverage/`, `__pycache__/`, `.turbo/`, `.cache/`, `.yarn/`, `.pnpm/` | 非源码、构建产物或缓存 |
| 软排除目录 | `.vercel/`, `.netlify/`, `.serverless/`, `cdk.out/`, `terraform/` | 部署产物，默认跳过但可通过配置纳入 |
| 二进制跳过 | `.png`, `.jpg`, `.jpeg`, `.gif`, `.webp`, `.ico`, `.bmp`, `.woff`, `.woff2`, `.ttf`, `.eot`, `.otf`, `.mp3`, `.mp4`, `.wav`, `.ogg`, `.webm`, `.map`, `.lock`, `.sum` | 不可解析的二进制文件 |

## 输出格式

### HTML 报告

自包含 HTML，D3 CDN 引用，其余内联：
- **Treemap 视图**: 文件体积可视化，按目录层级嵌套
- **依赖图视图**: 力导向图，节点大小 = fan-in + fan-out
- **统计摘要**: 总文件数 · 总体积 · 平均文件大小 · 最大文件
- **热点列表**: Top-10 最大文件 · Top-10 最多依赖文件

### JSON 输出

```json
{
  "summary": { "totalFiles": 0, "totalSize": 0, "avgSize": 0, "maxFile": "" },
  "treemap": { "name": "root", "children": [] },
  "dependencies": { "nodes": [], "edges": [] },
  "cycles": [],
  "hotspots": { "bySize": [], "byDeps": [] }
}
```

## 基线对比

| 功能 | 说明 |
|------|------|
| 基线保存 | 保存当前分析结果为基线 (`.memory/bundle-baseline.json`) |
| 趋势对比 | 对比当前 vs 基线，标注增长/缩减 |
| 告警阈值 | 总体积增长 > 20% · 单文件增长 > 50% · 新增循环依赖 |

## 核心规则

| # | 规则 | 设计理由 | 违反后果 |
|---|------|---------|---------|
| 1 | 零配置，无需构建即可运行 | 独立于构建工具链，适用任何项目 | 增加使用门槛 |
| 2 | 只读操作，不修改源码 | 分析安全，不引入副作用 | 破坏源码 |
| 3 | 自包含 HTML 输出 | 可独立查看，不依赖本地文件 | 报告无法分享 |
| 4 | 面积严格与文件字节数成比例 | 数据准确，视觉不误导 | 误判文件重要性 |
| 5 | 相同类型文件颜色一致 | 跨目录可识别同类文件分布 | 颜色混乱，无法对比 |
| 6 | 循环依赖检测使用 Tarjan SCC | 算法正确性保证 O(V+E) | 漏检循环依赖 |

## 性能约束

| 约束 | 值 | 说明 |
|------|:---:|------|
| 最大文件数 | 10000 | 超过时分批处理 |
| 最大分析时间 | 30s | 超时返回部分结果 |
| 内存上限 | 512MB | 超出时降级为仅统计 |
| HTML 报告大小上限 | 10MB | 超出时压缩或分页 |

## 边界场景

| 场景 | 处置 |
|------|------|
| 空项目目录 | 输出空报告，标注 `empty-project` |
| 超大文件 (>100MB 单文件) | 仅记录大小，不解析依赖 |
| 循环依赖群组 | 标注所有成员，突出最严重节点 |
| 无 import 语句的文件 | 标记为孤立节点 (leaf) |
| 混合语言项目 | 按语言分别着色，标注语言 |
| 符号链接目录 | 跟随，标注 `symlink`，去重 |
