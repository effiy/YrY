# rui-import 同步规则

> 文档同步到远端 API 的规则和约束，独立于实现细节。

## 同步流程

```
扫描 → 过滤 → 解析远端路径 → 拉取已有 sessions → 逐文件上传 → 新建 session → 汇总
```

### 详细步骤

| 阶段 | 动作 | 说明 | 失败处理 |
|------|------|------|---------|
| ① 扫描 | 从项目根递归遍历 | 不受 `.gitignore` 限制 | 目录不可读 → 跳过并记录 |
| ② 过滤 | 排除 `.git` / `node_modules` / `dist` / `.claude-plugin` 与显式 `--exclude` | 命中即跳过整个子树 | — |
| ③ 解析 | 计算本地→远端路径映射 | 路径分隔符统一为 `/`，空格替换为 `_` | 路径非法 → 跳过并记录 |
| ④ 拉取 | 远端 query sessions | 用于区分 `created` / `overwritten` | API 不可达 → 全部视为 created |
| ⑤ 上传 | 逐文件 POST | 并发上限 4，存在覆盖、不存在新建 | 单文件失败 → 记录，继续 |
| ⑥ 新建 | 追加 `create_document` session | 仅对新增路径执行 | session 创建失败 → 记录 |
| ⑦ 汇总 | 统计 created / overwritten / failed | 输出结构化结果 | — |

## 项目根定位

```
从 cwd 起逐级向上查找，遇到以下任一目录即视为项目根:
  - .git/
  - .claude/
找不到时回退为 cwd。
```

## 扫描规则

- 从项目根递归遍历，不受 `.gitignore` 限制
- 全量纳入，无扩展名限制
- 符号链接: 跟随（解析为实际路径）
- 空目录: 跳过

## 过滤规则

| 排除目录 | 原因 | 影响 |
|---------|------|------|
| `.git/` | 版本控制 — 内部文件无同步意义 | 跳过整个子树 |
| `node_modules/` | 依赖 — 体积大，远端自有 | 跳过整个子树 |
| `dist/` | 构建产物 — 可从源码构建 | 跳过整个子树 |
| `.claude-plugin/` | 插件元数据 — 本地专属 | 跳过整个子树 |
| 显式 `--exclude` | 用户指定 — 灵活排除 | 跳过匹配路径 |

## 路径映射

| 本地路径 | 远端路径 | 说明 |
|---------|---------|------|
| `docs/故事任务面板/<name>/*` | `故事任务面板/<name>/*` | 故事文档 |
| `skills/*/SKILL.md` | `workspace/skills/<name>/SKILL.md` | 技能规约 |
| `lib/*.mjs` | `workspace/lib/<name>.mjs` | 共享库 |
| `CLAUDE.md` | `workspace/CLAUDE.md` | 项目文档 |
| 其他文件 | `workspace/<relative-path>` | 通用映射 |

**路径转换规则**:
- 分隔符统一为 `/`
- 空格替换为 `_`
- 中文保留原样
- 特殊字符 URL 编码

## 上传策略

| 条件 | 行为 | 说明 |
|------|------|------|
| 远端不存在 | 新建 (created) | POST 创建 |
| 远端已存在 | 覆盖 (overwritten) | PUT 更新 |
| 并发上限 | 4 | 避免 API 限流 |
| 单文件失败 | 记录错误，继续处理 | 部分成功 > 全部失败 |
| 大文件 (> 1MB) | 分块上传 | 避免超时 |
| 超时 | 30s 超时，重试 1 次 | 网络抖动容错 |

## 错误模型

| 错误类型 | 严重度 | 处置 |
|---------|:---:|------|
| 文件不可读 | Warning | 跳过，记录路径 |
| API 不可达 | Critical | 终止，提示检查网络 |
| API 限流 (429) | Warning | 等待 Retry-After，重试 |
| 单文件上传失败 | Warning | 记录，继续处理 |
| session 创建失败 | Warning | 记录，不影响上传 |
| 路径非法 | Info | 跳过，记录路径 |

## 核心规则

| # | 规则 | 设计理由 | 违反后果 |
|---|------|---------|---------|
| 1 | 手动触发，不走 cron | 同步需人为确认 — 不可自动覆盖远端 | 远端数据意外覆盖 |
| 2 | 单文件失败不阻断 | 部分成功优于全部失败 | 一个文件失败全盘阻塞 |
| 3 | 并发 ≤ 4 | 避免 API 限流 — 4 并发平衡速度与安全 | 触发 429 限流 |
| 4 | 路径空格统一替换为 `_` | 远端兼容性 — 空格在 URL 中需编码 | 远端路径不一致 |
| 5 | 项目根定位基于 `.git`/`.claude` | 多项目环境 — 自动识别边界 | 扫描范围错误 |
| 6 | 不受 `.gitignore` 限制 | 全量同步 — gitignore 是版本控制，不是同步控制 | 必要文件遗漏 |
