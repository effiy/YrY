# WeChat Work Bot Message Contract

## 消息结构

两层结构：摘要段 + 明细段，以分隔线 `---` 分隔。

### 摘要段（分隔线之上，≤600 字）

必含字段：

| Field | Required | Description |
|-------|----------|-------------|
| 🎯 结论 | Always | 一句话结论 |
| 📝 描述 | Always | ≤100 字概述 |
| 📌 范围 | Always | 涉及的子项目/模块 |
| 👉 下一步 | Always | 后续操作或恢复点 |

完成/阻断/门禁类追加：

| Field | Required | Description |
|-------|----------|-------------|
| 🌐 影响 | 完成/阻断/门禁 | 受影响的文件/模块/用户 |
| 📎 证据 | 完成/阻断/门禁 | 验证命令或路径 |
| ⏱️ 会话 | 完成/阻断/门禁 | 合并耗时+用量 |

阻断类追加：

| Field | Required | Description |
|-------|----------|-------------|
| ❌ 原因 | 阻断 | ≤2 条阻断原因 |
| 🧭 恢复点 | 阻断 | 从何处恢复 |

门禁类追加：

| Field | Required | Description |
|-------|----------|-------------|
| 🔍 门禁 | 门禁失败 | 门禁名称 |
| 📊 结果 | 门禁失败 | 实际结果 |

### 明细段（分隔线之下）

详细上下文：变更文件列表、代码片段、完整错误日志等。

## 格式约束

- 分隔线至多两条
- 数字须来自执行结果，禁止占位符（如 N、X）
- 全文 ≤2000 字
- 正文不得出现字面量 `\n`（使用 `--content-file` 或脚本 `normalizeMessageText`）

## API 契约

```
POST <WEWORK_BOT_API_URL>
Headers: X-Token: <API_X_TOKEN>
Body: { "webhook_url": "<from config>", "content": "<message>" }
```

- `API_X_TOKEN` 仅从环境变量读取，禁止 CLI 参数传入
- `WEWORK_BOT_WEBHOOK_URL` 仅从环境变量读取

## 安全

- 不得提交 token、webhook URL 或 key 到仓库
- 日志和回复必须脱敏
- 完成通知为强制步骤；其他场景默认不自动发送

## 强制通知场景

| Scenario | Required Fields |
|----------|----------------|
| rui 完成 | 结论 + 描述 + 范围 + 下一步 + 影响 + 证据 + 会话 |
| 阻断 | 结论 + 描述 + 范围 + 原因 + 恢复点 + 影响 + 证据 + 会话 |
| 门禁失败 | 结论 + 描述 + 范围 + 门禁 + 结果 + 影响 + 证据 + 会话 |
| 会话中断 | 流程/阶段 + 中断原因 + 影响范围 + 证据 + 恢复点 + 会话 |
