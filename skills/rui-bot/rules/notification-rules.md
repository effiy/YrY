# rui-bot 通知规则

> 企业微信机器人通知的规则和约束，独立于实现细节。

## 调用形态

| 形态 | 输入 | 行为 | 使用场景 |
|------|------|------|---------|
| 追加日志 | `send --no-send` | 写入消息通知列表，不发 HTTP | 仅记录，不打扰 |
| 发送通知 | `send` | POST 企微 webhook | 主动推送消息 |
| 健康检查 | `send.mjs health` | 委托 rui-health 诊断，生成报告 | 定时健康巡检 |
| 失败重试 | `send.mjs flush` | 重试 `send-failed.json` 中的失败条目 | 恢复失败通知 |
| 预览 | `send.mjs --dry-run` | 展示将发送的内容，不实际发送 | 发送前验证 |

## 自动触发

| 触发方式 | 间隔 | 动作 | 失败处理 |
|---------|------|------|---------|
| 通知队列轮询 | 每小时 :07 | 扫描待发通知 → 批量推送 → 更新发送状态 | 入失败队列 |
| 健康报告 | 每小时 :17 | `health --html` → `docs/健康报告/` + 更新索引 | 下次重试 |
| 失败队列重试 | 每 30 分钟 | `flush` 重试 `send-failed.json` | 3 次后标记 dead |
| 健康趋势持久化 | 每次检查后 | 追加 `.memory/health-trend.jsonl` | 仅记录警告 |

## 消息格式

### Rich 格式 (默认)

用于通知面板展示：
- 标题 (必填): 消息主题，≤ 50 字符
- 摘要 (必填): 消息内容，≤ 200 字符
- 详情链接 (可选): 指向完整报告或相关页面
- 时间戳 (自动): 发送时间

### Verbose 格式

用于诊断/调试：
- 完整诊断数据
- 修复建议
- 相关文件路径
- 严重级别标注

## 通知场景

| 场景 | 触发条件 | 格式 | 优先级 |
|------|---------|------|:---:|
| 健康告警 | 综合分 < 60 或单维度归零 | Rich + Verbose | Critical |
| 管线完成 | 故事交付完成 | Rich | Info |
| 诊断触发 | D0-D8 诊断触发 | Verbose | Warning |
| 失败通知 | 通知发送失败 | Rich | Error |
| 自循环报告 | 定时汇总 | Rich | Info |

## 安全

| 规则 | 说明 | 违反后果 |
|------|------|---------|
| Webhook URL 不落盘 | 通过环境变量 `API_X_TOKEN` 传入 | Token 泄露 |
| 消息内容不泄露密钥 | 发送前过滤敏感信息 (Token/密码/密钥) | 密钥泄露 |
| 失败队列落盘 | `send-failed.json` 不含敏感信息 | 失败消息丢失 |
| Dry-Run 预览 | 发送前可预览，实际发送需确认 | 误发消息 |

## 失败队列

| 状态 | 含义 | 处置 |
|------|------|------|
| pending | 等待重试 | 下次 flush 重试 |
| retrying | 正在重试 | 等待结果 |
| dead | 3 次重试均失败 | 人工介入 |
| sent | 重试成功 | 移出队列 |

## 核心规则

| # | 规则 | 设计理由 | 违反后果 |
|---|------|---------|---------|
| 1 | 不直接做健康诊断，委托 rui-health | SRP — bot 管推送，health 管诊断 | 职责混淆 |
| 2 | 仅管消息推送和日志追加 | 单一职责 — 不做诊断/分析/报告 | 功能膨胀 |
| 3 | 失败不丢失，入失败队列 | 可靠性 — 通知不可丢失 | 漏通知 |
| 4 | Dry-Run 预览先于实际发送 | 安全检查 — 发送前验证 | 误发消息 |
| 5 | 密钥不落盘 | 安全基线 — 凭据保护 | Token 泄露 |
| 6 | 批量推送，不逐条发送 | 性能 — 减少 HTTP 请求 | 触发限流 |
| 7 | 3 次重试后标记 dead | 避免无限重试 — 资源保护 | 队列积压 |

## 消息模板

### 健康告警模板

```
【{项目名} 健康告警】
等级: {A/B/C/D}
综合分: {分数} ({趋势箭头})
触发维度: {维度列表}
详情: {报告链接}
时间: {ISO时间戳}
```

### 管线完成模板

```
【{项目名} 管线完成】
故事: {故事名}
版本: {版本号}
耗时: {耗时}
P0 状态: 清零
详情: {故事目录链接}
```

## 边界场景

| 场景 | 处置 |
|------|------|
| Webhook URL 不可达 | 入失败队列，下次重试 |
| 消息内容超长 (>4096) | 截断为摘要 + 详情链接 |
| 并发发送过多 | 队列化，按优先级发送 |
| 通知面板已满 | 归档旧条目，保留最近 100 条 |
| 发送频率过高 | 合并同类通知，5 分钟内去重 |
