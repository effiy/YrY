# import-docs 导入契约

## 路径生成

远端 `target_file` 按以下顺序拼接：

```
<prefix...>/<仓库目录名>/<导入目录名>/<相对文件路径>
```

规则：

- `prefix` 来自 `--prefix`，逗号分隔。
- 当前工作目录名会作为仓库目录名写入远端路径。
- 当 `--dir` 不是项目根目录（或显式指定 `--dir`）时，会额外加入导入目录名。
- 路径片段中的空格会替换为 `_`。
- 路径分隔符统一为 `/`。

## 自动检测规则

- 当前在项目根目录的 `.claude` 下 → 导入 `.claude` 目录，所有文件
- 其他情况 → 导入项目根目录，仅 `.md` 文件
- 项目根目录通过查找 `.git` 目录确定

## 文件遍历规则

- 始终忽略 `.git` 目录
- 不跟随符号链接
- **遵循 `.gitignore`**：若目标目录位于 git 仓库内，使用 `git ls-files --cached --others --exclude-standard` 获取文件列表，与 git push 规则保持一致（仅导入未被 `.gitignore` 忽略的文件）
- 若无法使用 git（如目录在仓库外或无 `.git`），回退到文件系统遍历
- `.claude` 模式：导入所有未被忽略的文件
- 根目录模式：仅导入未被忽略的 `.md` 文件

## 去重与覆盖

1. 导入前查询 `sessions` 集合中已有的 `file_path`。
2. 每个本地文件都会调用 `/write-file` 写入远端文件。
3. 若 `file_path` 已存在，仅覆盖文件内容，状态记为 `overwritten`。
4. 若 `file_path` 不存在，写入文件后创建新的 `sessions` 记录，状态记为 `created`。
5. 同一次运行中创建的路径会加入内存去重集合，避免重复创建 session。

## 安全约束

- `X-Token` **仅**来自系统环境变量 `API_X_TOKEN`（`--token` 已禁用，不从配置文件读取）。
- 不得把 token 写入仓库、文档或最终回复。
- 缺少 token 时必须停止执行，不尝试匿名导入。

## 失败处理

- 单个文件失败时记录该文件错误并继续处理后续文件。
- 最终 `failed > 0` 时脚本以非零退出码结束。
- 用户摘要中只展示失败文件相对路径和错误信息，不展示 token。
- 空目录或无匹配文件时正常退出（退出码 0）。
