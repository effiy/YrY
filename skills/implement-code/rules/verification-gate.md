# 验证门禁规范

> 面向“可交互入口”的验证决策逻辑。通过手动浏览器验证 + 入口集成检查（以项目现状为准）。

---

## 1. 门禁原则

| 编号 | 原则 |
|------|------|
| G0-1 | P0 检查项未全部通过，禁止交付 |
| G0-2 | 验证失败时必须修复代码，不得修改检查清单降级 P0 |
| G0-3 | 自修复上限 2 轮 |
| G0-4 | 人工确认类检查项不计入 P0 通过/失败判定 |
| G0-5 | 门禁未执行、被跳过或缺少报告证据，视为门禁失效 |
| G0-6 | **编码前**：未完成 [`implement-code-testing.md`](./implement-code-testing.md) Gate A（真实入口 MVP + 可追溯证据），禁止写项目代码 |
| G0-7 | **编码后**：未完成 Gate B（AI 自动主流程冒烟 + 可复核产物），禁止进入阶段 7 / 交付总结 |

---

## 2. 检查项分类

| 优先级 | 验证方式 | 通过条件 |
|--------|---------|---------|
| P0 | 手动浏览器验证 + 代码审查 | UI 可交互、data-testid 完整、入口初始化/集成方式正确 |
| P1 | 手动验证 + Jest | 错误状态展示、边界值、组件渲染 |
| P2 | 人工 | 视觉一致性、响应式布局 |

---

## 3. 验证执行方式

```
1. 启动 dev server（或 HTTP 服务器）
2. 在浏览器中按检查清单逐步操作
3. 截图记录操作结果
4. 检查 JS 控制台无错误
5. 记录：✅ 通过 / ❌ 失败
```

---

## 4. 修复循环

```
轮次 1：分析根因 → 修复代码 → 重新验证
轮次 2：同上
轮次 3：输出阻断报告，停止执行
```

---

## 5. 状态回写

验证完成后将结果回写到 `05_动态检查清单.md`：
- P0 通过 → ✅ + 验证日期
- P0 未通过 → ❌ + 失败原因

---

## 6. 禁止事项

- 将 P0 降级为 P1 以"通过"门禁
- 修改检查清单预期结果
- 修复超过 2 轮后仍继续
- 未实际验证就改状态为 ✅

---

## 7. 编码前真实场景 MVP 门禁（Gate A）

与 [`implement-code-testing.md`](./implement-code-testing.md) §2、`rules/orchestration.md` 阶段 1 退出条件一致：

1. 主场景已在 **真实入口**完成最小可用验证。
2. 证据（命令/日志/截图/checklist 路径）已落地且可追溯。
3. 测试产物位于 `tests/` 下，符合 [`artifact-contracts.md`](./artifact-contracts.md) §2。

未满足本节不得进入阶段 2（动态检查门禁），亦不得在 [`code-implementation.md`](./code-implementation.md) 下调写项目代码。

---

## 8. 编码后主流程冒烟门禁（Gate B）

与 `implement-code-testing.md` §3、`orchestration.md` 阶段 6 一致：

1. 由 **AI 执行**主路径冒烟（Playwright 或未安装时的等价可脚本化链路）。
2. P0 相关项已验证或已标注 N/A（附原因）。
3. 失败按阻断处理；修复循环遵守 §4。

---

## 9. 动态检查清单最终完成门禁

### 9.1 适用时机

阶段 7 写入实施总结前，针对 `05_动态检查清单.md` 做一次 **最终闭环**：状态与证据一致，且与 `02` 主场景对齐。

### 9.2 序列（必须按顺序）

1. **先**回写 `05` 的最终状态列 / 备注（含验证日期或证据路径）。
2. **再**复查：逐项核对 P0 是否仍为 ✅ 或可解释的 N/A。
3. **最后**将结论写入 `06_实施总结.md`（含未完成项与风险）。

### 9.3 「先回写 → 再复查 → 再写入总结」硬门禁

以下任一为假，则 **动态检查清单最终完成门禁未通过**，必须改走阻断版总结或门禁异常通知，不得对外宣称「实施完成」：

- `05` 已根据 Gate B 结果更新，且无「未验证却标 ✅」；
- 复查记录可追溯（可在 `06` §验证摘要或 `05` 备注中定位）；
- `06` 中的结论与 `05` 当前内容一致。

（关联：`artifact-contracts.md` §3 状态值、`orchestration.md` 阶段 7 退出条件。）
