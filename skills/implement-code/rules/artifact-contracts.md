# 产物与回写契约

> 本规范统一 implement-code 各阶段的文件路径、命名方式和状态字段。

---

## 1. 文档集路径

```text
docs/<功能名>/
├── 01_需求文档.md
├── 02_需求任务.md
├── 03_设计文档.md
├── 04_使用文档.md
├── 05_动态检查清单.md
├── 06_实施总结.md
└── 07_项目报告.md
```

- `06_实施总结.md`：正常完成或阻断时都必须生成
- `05_动态检查清单.md`：阶段 3 验证后回写状态列、备注列和检查总结
- `01/02/03/04/07`：阶段 4 总结后回写 `## 实施状态`

无法定位 `docs/<功能名>/` 时，写入项目日志记录文件

---

## 2. 测试产物路径

所有测试相关文件**必须**位于 `tests/` 目录，禁止落地到 `src/`、`docs/`、项目根目录等。

```text
tests/
├── e2e/<功能名>/            # spec + fixtures + pages
├── screenshots/<功能名>/    # MCP 截图归档（默认忽略）
├── snapshots/<功能名>/      # toHaveScreenshot 基准（提交）
├── downloads/<功能名>/      # 测试下载产物（默认忽略）
└── traces/<功能名>/         # trace 归档（Playwright 安装后启用，默认忽略）
```

---

## 3. 状态回写

### 05_动态检查清单状态值

| 状态 | 含义 |
|------|------|
| ⏳ | 未验证 |
| 🏃 | 原型验证通过（阶段 2） |
| ✅ | 冒烟测试通过（阶段 3） |
| ❌ | 验证失败 |
| ⚠️ | 需人工确认 |

### 01/02/03/04/07 实施状态

每份文档新增 `## 实施状态`，包含：实施状态（✅/🟡/⛔）、更新时间、实施阶段、验证结果、关联总结、下一步。

---

## 4. 命名约定

- `<功能名>`：与 `docs/<功能名>/` 目录一致
- `<场景名>`：来自 `02_需求任务.md` 操作场景
- 禁止混用简称（如 `动态检查清单.md` / `05_动态检查清单.md`）

---

## 5. 禁止事项

- 未实际验证就改状态
- 在 `tests/` 外生成或保留测试文件
- 同一文件重复新增 `## 实施状态` 小节
