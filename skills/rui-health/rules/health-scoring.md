# rui-health 评分规则

> 从 rui-health SKILL.md 提取的评分算法和诊断规则，独立于实现细节。

## 评分算法

### 综合评分公式

```
综合分 = 核心加权分 × 0.5 + 工程成熟度加权分 × 0.5

核心加权分 = Σ(核心维度得分 × 权重) / Σ(核心维度权重)
工程成熟度加权分 = Σ(工程维度得分 × 权重) / Σ(工程维度权重)
```

### 权重定义

权重定义在 `lib/constants.mjs` → `HEALTH_SCORING_DIMENSIONS`，为单一真相源。

### 评级阈值

| 等级 | 分数 | 标签 | 含义 | 建议动作 |
|------|:---:|------|------|---------|
| **A** | ≥ 90 | 优秀 | 系统健康，无需干预 | 维持现状 |
| **B** | ≥ 75 | 良好 | 基本健康，有改进空间 | 关注低分维度 |
| **C** | ≥ 60 | 一般 | 存在风险，需要关注 | 制定改进计划 |
| **D** | < 60 | 需关注 | 存在严重问题 | 立即修复 P0 项 |

## 趋势计算

基于 `.memory/health-trend.jsonl` 历史数据：

1. **线性回归**：对最近 7 个数据点拟合趋势线
2. **斜率方向**：slope > 0.5/周 → rising，slope < -0.5/周 → falling，否则 stable
3. **置信度**：R² ≥ 0.7 → high，≥ 0.4 → medium，< 0.4 → low
4. **预测**：基于趋势线外推 7 天后的评分

## 告警触发

| 条件 | 级别 | 通知 |
|------|------|------|
| 综合分 < 60 | Critical | 企微 @all |
| 综合分下降 > 10 分/周 | Critical | 企微 @all |
| 任一核心维度 = 0 | Warning | 企微通知 |
| 连续 3 次下降 | Warning | 企微通知 |
| 新增 D0-D8 诊断触发 | Info | 仅记录 |

## SLO 阈值

| 级别 | 阈值 | 标签 | 响应 |
|------|:---:|------|------|
| 健康 | ≥ 75 | ok | 无需干预 |
| 需关注 | < 75 | warning | 计划性改进 |
| 需处理 | < 60 | critical | 本周内行动 |
| SLO 违约 | < 40 | breach | 立即干预 |

## 诊断维度评分规则

### 核心维度 (9)

每个维度 0-100 分，按权重加权：

- **token** (权重 12): `API_X_TOKEN` 存在=100，不存在=0
- **config** (权重 8): config.json 存在且完整=100，缺失=0，不完整=50
- **robots** (权重 8): Webhook 配置且可达=100，仅配置=50，缺失=0
- **api** (权重 12): P95 < 2s=100，< 5s=75，< 10s=50，超时=0
- **reports** (权重 8): 24h 内有新报告=100，7d 内=50，>7d=0
- **format** (权重 8): 模板完整=100，部分缺失=50，全部缺失=0
- **diagnostics** (权重 8): 0 触发=100，1-2=75，3-4=50，5+=25
- **git** (权重 8): 干净=100，≤3 未提交=75，≤10=50，>10=0
- **security** (权重 8): 无发现=100，低危=50，高危=0

### 工程成熟度维度 (7)

- **em_testing** (权重 10): 有框架+用例=100，仅有框架=50，无=0
- **em_types** (权重 8): strict=100，tsconfig=75，JSDoc=50，无=0
- **em_linting** (权重 8): 三者齐全=100，缺一=66，缺二=33，全无=0
- **em_cicd** (权重 8): 有配置=100，无=0
- **em_docs** (权重 8): 三者齐全=100，缺一=66，缺二=33，全无=0
- **em_deps** (权重 6): 全部满足=100，缺 lockfile=50，有 CVE 扣分
- **em_git** (权重 6): 三者齐全=100，缺一=66，缺二=33，全无=0

## 约束

| # | 规则 | 反例 |
|---|------|------|
| 1 | 权重从 `lib/constants.mjs` 导入，不自定义 | 各模块各维护一份权重 |
| 2 | 诊断结果必须可追溯 | "config 看起来没问题" |
| 3 | HTML 报告自包含 | 报告依赖本地文件 |
| 4 | 不直接发送企微消息 | health.mjs 内联 HTTP POST |
| 5 | 趋势数据追加写入，不覆盖 | 每次重写整个 JSONL |
| 6 | 全维度覆盖，不可跳过 | 仅检查部分维度即报告 |
