命令族全景

search <kw>
只读
按关键词搜索 npm registry，结果按周下载量降序排列，表格化输出
install <pkg>
写入
安装 npm 包到当前项目，支持 @version、--dev、--global
update <pkg>
写入
更新指定包到兼容最新版，展示旧版→新版变更
list [--depth N]
只读
列出当前项目已安装依赖，支持依赖树深度控制
info <pkg>
只读
查看包的完整元数据：版本/许可证/维护者/依赖/GitHub
uninstall <pkg>
写入
从项目移除指定包，前置校验避免误操作
publish <path>
写入
本地文件/目录一键发布为 npm 包，自动处理 package.json
npx <pkg>
执行
不安装直接运行 npm 包，流式透传 stdout/stderr
audit
只读
审计已安装依赖的安全漏洞，按严重级别分组 + 修复建议

核心设计

统一入口
所有 npm 操作通过 node skills/rui-npm/rui-npm.mjs 或 /rui-npm 命令执行
结构化输出
结果默认表格化展示，所有只读命令支持 --json 标志输出机器可读格式
错误友好
每种失败场景有明确错误提示和恢复建议（搜索/登录/初始化引导）
安全前置
写操作前验证前置条件（package.json 存在 / npm 已登录），失败即阻断并引导

场景 1 — 搜索并安装包

搜索 react 相关包
/rui-npm search react — 按关键词搜索 npm registry，结果按周下载量降序
🔍 搜索 "react" ...
        | # | 包名 | 版本 | 周下载量 | 描述 |
        | 1 | react | 19.2.0 | 12.3M/w | React is a JavaScript library...
查看 react 详细信息
/rui-npm info react — 确认许可证、维护者和依赖健康度
📋 查询 react ...
        | 名称 | react | 许可证 | MIT | 仓库 | github.com/facebook/react
安装 react
/rui-npm install react — 安装到当前项目 dependencies
📦 安装 react ...
        ✅ react@19.2.0 安装完成

场景 2 — 本地脚本即发即用

发布本地脚本
/rui-npm publish ./my-cli.mjs --name my-util — 文件模式自动生成 package.json
👤 已登录 npm: yourname
        📝 自动生成 package.json (name: my-util) ...
        🚀 发布中 ...
        ✅ my-util@1.0.0 发布成功
           安装: npm install my-util | 运行: npx my-util
通过 npx 直接运行
/rui-npm npx my-util — 不安装即执行
▶️ npx my-util
        ...（包自身的 stdout/stderr 流式输出）...
更新后重新发布
/rui-npm publish ./my-cli.mjs --name my-util --version 1.1.0 — 指定新版本号

场景 3 — 安全审计与清理

审计依赖漏洞
/rui-npm audit — 按严重级别分组展示
🔒 审计依赖安全漏洞 ...
        | 💀 Critical | 0 | 🔴 High | 2 | 🟡 Moderate | 5 | 🟢 Low | 3 |
        ### 修复建议: npm audit fix / npm audit fix --force
更新有漏洞的包
/rui-npm update lodash — 升级到兼容最新版
⬆️  更新 lodash (当前: 4.17.20) ...
        ✅ lodash: 4.17.20 → 4.17.21
清理不再需要的包
/rui-npm uninstall moment — 从项目移除
🗑️  卸载 moment ...
        ✅ moment 已卸载
