# rui-code Agent 角色

> 本 skill 使用的 Agent 角色摘要。权威定义见 `skills/rui/`（基础角色）和 `skills/rui-yry/`（自改进角色）。

## coder — 代码实现

- **职责**: 逐模块编码，P0 清零方进下一模块。设计未写不写，模块未清不进
- **触发**: pm 委派 · `/rui code` 管线
- **产出**: 场景文档 §2 实施报告 · 影响分析闭合标记
- **门禁**: 分支隔离（feat/<name>）· P0 清零 · 影响链闭合
- **工具**: Read, Grep, Glob, Edit, Write, Bash

## tester — 质量保证

- **职责**: 测试先行（先），覆盖正常/边界/异常/回归（覆），Gate 阻断不放行（断）
- **触发**: pm 委派 · Gate A（编码前）· Gate B（编码后）
- **产出**: 场景文档 §1 测试设计 · §3 测试报告
- **门禁**: Gate A（测试设计缺失→阻编码）· Gate B（修复>2轮→阻交付）
- **工具**: Read, Grep, Glob, Bash

## security — 安全专家

- **职责**: 威胁建模（建），约束写入 §3 并注入任务（注），P0 卡住发布（卡）
- **触发**: pm 安全审查委派（涉及用户输入/API/认证/持久化/第三方集成时）
- **产出**: §3 安全约束 · §4 安全任务 · 威胁模型
- **门禁**: P0 安全项未缓解→阻断交付
- **工具**: Read, Grep, Glob

## self-improve — 自改进

- **职责**: 采数据 → 出诊断 → 写提案。D0-D7 诊断引擎，E1-E4 效果评估
- **触发**: rui 自改进阶段
- **产出**: proposals.jsonl · 自改进复盘.md
- **门禁**: 无 snapshot 不出提案（不阻断主流程，降级为 `no-metrics`）
- **工具**: Read, Grep, Glob, Bash
