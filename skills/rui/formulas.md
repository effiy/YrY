# 故事文档公式

> **口诀：公式即合约、信模型生成、不写占位符。** 故事文档的结构单一真相源——一组章节 / 表头 / 字段规约，模型据此直接产出，不依赖占位符模板。
>
> **公理推导**（[CLAUDE.md](../../CLAUDE.md)）：信模型 → 模型按公式生成结构；惜注意 → 公式 < 模板（一文件 vs 29 文件）；退化对策「先可见，后规则」→ 公式本身就在说话。
>
> **覆盖范围**：8 份主线故事文件（01–08）+ 7 类补充文档（10–19）+ 通用元素（F.meta / F.nav / F.evidence）。
>
> **配套**：参考文档公式（F.ref.\*）/ 目录与生命周期 / 数据契约见 [coder.md](./coder.md)；生成时强制约束见 [rules/doc-generation.md](../../rules/doc-generation.md)。

## 通用元素（所有故事/参考文档共用）

### F.meta — 版本头

```
> | v{version} | {YYYY-MM-DD} | {model} | {tool?} | 🌿 {branch} | ⏱️ {HH:mm}–{HH:mm} | 📎 [CLAUDE.md](../../../CLAUDE.md) |
```

- **必含**: version、date、model、branch
- **可选**: tool、time-range、philosophy 链接（仅 01-故事任务）
- **占位符必须替换**: 任何 `{...}` 留到产出即视为 P0 偏差

### F.nav — 导航块（01~04 文件首尾）

```
> **导航**: [← {prev}](./{prev-file}.md) · [↑ {index}](./00-索引.md) · [{next} →](./{next-file}.md)
```

- 00-索引: 不含导航块（自身是入口）
- 01: `[← 00-索引] · [↑ {类}文档] · [02-{section} →]`
- 04: `[← 03-{section}] · [↑ 00-索引]`（无 next）

### F.evidence — 证据等级

```
A=已验证(附路径) | B=可推导(附规则) | C=未验证(标注 `> 待补充`) | D=禁止(视为幻觉)
```

仅 A/B 直接写入；C 标注待补充；D 不得出现。详见 [agents/AGENT.md](../../agents/AGENT.md) 「证据标准」。

## 故事主线公式（docs/故事任务面板/{project}/{name}/）

### F.story.01 — 01-故事任务.md

**公式**: `meta + 角色公式速查 + Story×N + (§6 .claude改进? + §7 架构演进? + §L 自改进循环?)`

**角色公式速查**（顶部，固定文案）:
- PM: `作为 [角色] 我想要 [动作] 以便 [价值]`
- Tester: `Given [前置] When [操作] Then [预期]`
- Coder: `模块 → 接口 → 数据流`
- Security: `威胁 → 信任边界 → 缓解`

**每个 Story 章节**:

| 章节 | 负责人 | 表头/字段 |
|------|--------|----------|
| §1 Story | pm | 作为/我想要/以便/优先级(P0/P1/P2)/范围边界/依赖/子项目 + 范围外 |
| §1.1 User Operations | tester | `# \| 操作 \| 触发条件 \| 操作步骤 \| 预期结果`，UI 故事附 mermaid flowchart |
| §2 Requirements | pm | 功能点 `FP# \| 描述 \| 输入 \| 输出 \| 错误行为 \| 优先级`；业务规则 `R# \| 描述 \| 校验方式 \| 证据级别`；数据约束 `约束 \| 类型 \| 范围/格式 \| 来源` |
| §3 Design | coder + security | 设计概述（2-4 句）+ 决策表 `决策领域 \| 选定方案 \| 选择理由 \| 详见` |
| §4 Tasks | pm + 全员 | `ID \| 描述 \| 工作量(S/M/L) \| 依赖 \| 交付物 \| Agent \| 门禁 \| 交接下游` + 任务依赖 mermaid |
| §5 AC | tester | `AC# \| Given \| When \| Then \| 门禁(Gate A/B)` |
| §6 .claude 改进清单 | pm | 可选，skill/agent/rule/script/config 改进项 |
| §7 系统架构演进 | pm | 可选，近期/中期/远期 |
| §L 自改进循环 | self-improve | 可选，每次完成追加 |

**约束**: §1~§5 必填；§6/§7 按需；多故事时 `## Story 1`、`## Story 2` 顺序展开。
