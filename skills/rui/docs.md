```

├── 02-后端技术评审.md
├── 03-前端技术评审.md

├── .improvement/proposals.jsonl
```

01~08 是模板基线（最小集），补充文档按故事内容拓展。

## --from-code 自主探索模式

`/rui doc --from-code` 不传 req 时，pm 自主探索源码：

1. 扫描项目目录结构，识别可独立文档化的模块/功能区域
2. 每个候选输出：推荐名称(kebab-case)、覆盖范围、源码证据（Level A 路径列表）
3. 按优先级排序（有源码无文档 > 有文档过时 > 其他）
4. 用户选择后按选定范围生成全文档基线

`/rui doc --from-code <req>` 传 req 时跳过探索，直接按 req 限定范围反推故事。

## 故事板章节

| 章节 | 负责人 | 公式 | 内容 |
|------|--------|------|------|
| §1 Story | pm | 作为...我想要...以便... | 角色场景、价值、范围边界、依赖 |
| §1.1 User Operations | tester | Given→When→Then | 用户操作 + UI交互流程 |
| §2 Requirements | pm | 输入→输出→错误行为 | 功能点、业务规则、数据约束 |
| §3 Design | coder + security | 模块→接口→数据流 / 威胁→信任边界→缓解 | 设计概述 + 决策表 |
| §4 Tasks | pm + all | — | 任务拆解、依赖、交付物 |
| §5 Acceptance Criteria | tester | Given→When→Then | 验收标准、测试方法、预期结果 |
| §6 .claude 改进清单 | pm | 观察→诊断→改进 | skill/agent/rule/script/config 改进 |
| §7 系统架构演进任务 | pm | 观察→诊断→改进 | 近期/中期/远期演进 |
| §L 自我改进循环 | self-improve | 观察→诊断→改进 | 数据驱动改进清单(每次 rui 完成追加) |

## 补充文档规则

pm 在文档生成阶段按决策表判断：

| 条件 | 生成文档 | 负责人 |
|------|---------|--------|
| §1.1 涉及 UI 改造 | 页面设计文档 | coder |
| §2 涉及新增/修改 API | API 契约文档 | coder |
| §2 涉及数据存储变更 | 数据迁移方案 | coder |
| 第三方服务集成 | 集成方案 | coder + security |
| 新权限控制 | 权限模型 | security |
| 性能敏感路径 | 性能基准 | coder |

无匹配条件不生成补充文档。有模板优先用模板，无模板按已有模式 ad-hoc 生成。
