调用 `code-review` 技能审查代码规范性。

参数：`$ARGUMENTS`

典型触发语（用户常见说法）：
- "帮我审查这段代码 / 看看这些文件有没有问题"
- "代码规范检查 / 是否符合项目约定"
- "review 一下这个组件 / store / 工具函数"

执行要求（加强）：
- 必须基于实际读取到的代码进行审查，不得推断未看到的文件内容。
- 审查维度覆盖项目专项（入口初始化、状态管理、组件注册/导出、代码结构、编码规范）和通用质量（可读性、边界处理、安全、性能）。
- 输出必须按 P0（必须修复）/ P1（建议修复）/ P2（可选优化）分级，并给出具体文件路径和修复建议。
- 无法读取文件时，须明确说明"无法读取文件 <路径>，跳过"。

请使用 Skill 工具调用 `code-review` 技能，传入上述参数，最后返回分级审查结果。
