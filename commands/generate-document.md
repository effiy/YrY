调用 generate-document 技能生成或更新文档。

参数：$ARGUMENTS

## 命令总览

| 命令 | 作用 | 落盘路径 | 幂等性 |
|------|------|---------|--------|
| `init` | 初始化/更新项目基础文件 + `docs/项目初始化/` 全文档编号集 | 仓库根目录 10 个基础文件 + `docs/项目初始化/01-07` | **可反复执行**，自动更新事实类内容、保留人工补充 |
| `<功能名>-<用户故事简短描述>` | 生成/更新 `docs/<功能名>/` 下全文档编号集 | `docs/<功能名>/01-05, 07` | **可反复执行**，已存在则基于既有文档增量更新 |
| `weekly` | 生成/更新本周周报 | `docs/周报/<YYYY-MM-DD>~<YYYY-MM-DD>/周报.md` | **可反复执行**，同一自然周内覆盖更新 |
| `weekly <日期>` | 按指定日期所在自然周生成/更新周报 | 同上 | 同上 |
| `from-weekly <周报路径>` | 从周报「后期规划与改进」拆解为多个功能文档集 | 多个 `docs/<功能名>/` 全文档编号集 | 每个功能目录独立判断新建/更新 |

## 详细说明

### `init` — 项目初始化/再初始化

```bash
/generate-document init
```

- **首次执行**：扫描仓库生成 10 个项目基础文件（`CLAUDE.md`、`README.md`、`docs/architecture.md` 等）+ `docs/项目初始化/` 下 01-07 全文档编号集（含 `06_实施总结`）
- **反复执行**：按 `rules/项目基础文件.md` 的「可重复运行（re-init）更新策略」刷新事实类内容（技术栈、目录结构、命令、入口等），保留人工补充段落；冲突处标注 `> 待确认` 并给出证据
- **更新标记**：使用 `<!-- AUTO-GENERATED:BEGIN -->...<!-- AUTO-GENERATED:END -->` 哨兵块包裹自动生成段落，便于二次运行安全重写

### `<功能名>-<用户故事简短描述>` — 功能文档集生成/更新

```bash
/generate-document 支付模块-支持多渠道退款
```

- **首次执行**：完整执行 5 步工作流，生成 `docs/<功能名>/01_需求文档.md` … `05_动态检查清单.md`、`07_项目报告.md`
- **反复执行**：若 `docs/<功能名>/` 已存在，按以下策略更新：
  1. **读取既有文档**：先加载现有 01-07 内容作为上游事实
  2. **差异对比**：对比用户输入与既有文档的 `01_需求文档.md`，识别新增/变更的用户故事或验收标准
  3. **级联更新**：仅对受影响的下游文档执行增量重写（如 01 变更 → 02 同步 → 03 按需调整；04/05/07 按最新状态重建）
  4. **版本递增**：文档头部版本号按 `v{主版本}.{次版本+1}` 递增（如 `v1.0` → `v1.1`），并更新「最后更新时间」
  5. **变更溯源**：在 `07_项目报告.md` 或文档头部追加「更新记录」小节，记录本次更新内容与触发原因
- **命名一致性**：再次调用时必须使用**相同的 `{功能名}`** 目录名；若用户故事描述变化，仅更新 01 中描述，不重建目录

### `weekly` — 周报生成/更新

```bash
/generate-document weekly
/generate-document weekly 2026-04-29
/generate-document weekly 2026-04-27~2026-05-03
```

- **同一自然周内反复调用**：覆盖更新同一文件，版本号递增，KPI 数据重新拉取
- **跨周调用**：归入对应自然周新建文件

### `from-weekly` — 从周报拆解多组功能文档

```bash
/generate-document from-weekly docs/周报/2026-04-27~2026-05-03/周报.md
```

- 先读取周报「四、后期规划与改进优先级总表」，提取可落地条目
- 为每个条目生成/更新 `docs/<功能名>/` 全文档编号集（更新逻辑同 `<功能名>-<用户故事简短描述>`）
- 全部完成后**一次** `import-docs` + **一条**汇总 `wework-bot`

## 强制收尾（所有命令通用）

每次调用结束后必须：

1. **先执行 `import-docs`**：`node .claude/skills/import-docs/scripts/import-docs.js --dir docs --exts md`
2. **再执行 `wework-bot`**：发送完成通知，含真实 `☁️ 文档同步` 结果

两步缺一不可，顺序禁止颠倒。

请使用 Skill 工具调用 generate-document 技能，传入上述参数。
