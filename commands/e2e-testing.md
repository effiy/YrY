调用 `e2e-testing` 技能设计 E2E 测试方案。

参数：`$ARGUMENTS`

典型触发语（用户常见说法）：
- "给这个功能写 E2E 测试 / UI 测试方案"
- "设计端到端测试用例 / 冒烟测试步骤"
- "有哪些场景需要浏览器验证 / data-testid 怎么加"

执行要求（加强）：
- 测试用例只能基于已提供的场景生成，不得自行添加未在需求任务中出现的场景。
- 与 `implement-code` 联用时，产出须满足 Gate A（编码前最小可用验证）和 Gate B（编码后主流程冒烟），准入底线以 `../implement-code/rules/implement-code-testing.md` 为准。
- 若场景描述不足以推断断言条件，须明确输出"前置信息不足，需补充：<缺失内容>"，不得编造通过条件。
- 选择器策略优先级：`data-testid` > 语义标签 > 文本内容。

请使用 Skill 工具调用 `e2e-testing` 技能，传入上述参数，最后返回测试方案与用例骨架。
