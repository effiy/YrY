调用 `verification-loop` 技能执行验证循环。

参数：`$ARGUMENTS`

典型触发语（用户常见说法）：
- "验证一下能不能构建通过 / 跑一下类型检查"
- "打包看看有没有问题 / 集成验证"
- "跑 lint / 跑测试 / CI 预检"

执行要求（加强）：
- 执行前必须先进行 MCP 可用性探针，禁止静默降级；必要工具不可用且无降级方案时须停止并提示。
- 阶段 1-3 有阻断项时禁止进入阶段 4（执行阶段）；验证命令名称必须取自 `package.json scripts`，不得硬编码。
- 阶段 4 只执行一次，失败后不自动重试，须给出"一次性修复清单"并说明修复后应从哪个阶段重入。
- playwright 降级时须在输出中标注 ⚠️，不得以全通过方式呈现降级结果。

请使用 Skill 工具调用 `verification-loop` 技能，传入上述参数，最后返回验证结果或修复清单。
