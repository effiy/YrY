# Gate Rules

## Gate A — 测试先行

- 测试方案 + 原型必须就绪
- 单行 CSS 变更可跳过
- 未通过不得进入实现（H6）

## Gate B — 验证

- 环境快照 → 静态预检 → 对齐 → 单次执行
- 必须产出05-后端实施报告.md、06-前端实施报告.md、07-测试用例报告.md，缺一不可
- 三报告交叉引用闭合，各报告评审清单全部 ✅
- 修复 ≤2 轮，超过 2 轮阻断交付（H7）

## P0 审查标准

- P0=阻塞发布，必须修复才能前进
- P1=建议修复
- P2=可选优化
- P0 未清零不进下一模块
