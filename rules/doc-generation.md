# doc-generation

> **口诀：版头齐、目录清、证据足、产出聚、裁剪准。** 五条强制约束，缺一即偏差。

故事文档公式见 [skills/rui/formulas.md](../skills/rui/formulas.md)；目录生命周期 / 参考文档公式 / 数据契约见 [skills/rui/coder.md](../skills/rui/coder.md)。

## 适用

所有 `docs/` 下的 `.md` 产出，包括故事文档（`故事任务面板/`）与四类参考文档（组件/接口/页面/领域）。

## 规则

### 1. 版头齐

1. 版本行必填：`v{版本} | {YYYY-MM-DD} | {模型} | {分支}`，公式占位符 `{...}` 留到产出即 P0 偏差
2. 主体章节首尾含标准导航（← 前 · ↑ 索引 · 后 →）；索引文件不含
3. 图先文后：架构 / 流程 / 关系优先 mermaid，文字补细节

### 2. 目录清（命名唯一定义）

按项目独立子目录组织，缺项目级目录阻断（`no-project-prefix`）。

| 文档类 | 路径模式 | 用途 |
|--------|---------|------|
| 故事 | `docs/故事任务面板/<Project>/<name>/` | 执行 |
| 组件 | `docs/组件文档/<Project>/<component-name>/` | 参考（前端组件） |
| 接口 | `docs/接口文档/<Project>/<resource-name>/` | 参考（后端 API） |
| 页面 | `docs/页面文档/<Project>/<page-name>/` | 参考（页面编排） |
| 领域 | `docs/领域模型/<Project>/<domain-name>/` | 参考（领域结构） |

约束：`<Project>` 大驼峰；`<name>` kebab-case；总路径 ≤ 96 字符。

### 3. 证据足

4. 证据等级 A/B 可写入；C 标 `> 待补充`；D（无支撑）禁止（见 [agents/AGENT.md](../agents/AGENT.md)）
5. 不编造未验证的模块名 / 接口 / 路径 / 文件名
6. 故事文档引用参考文档先指向索引，按需深入具体章节

### 4. 产出聚（文件创建顺序）

| 阶段 | 创建 | 条件 |
|------|------|------|
| 文档生成 | 01 + 02/03/04 + 补充（10–19） | 01 必创建；02/03 按项目类型 |
| 验证 | 05 / 06 / 07 | 有对应技术评审时 |
| 自改进 | 08 | 必创建 |
| 交付 | 00-消息通知列表 | 自动 |

7. 文件按管线阶段创建，不可提前

### 5. 裁剪准（增量分级）

| 级别 | 范围 | 影响分析 | 架构设计 | 文档刷新 |
|------|------|---------|---------|---------|
| T1 | 措辞 / 格式 | 跳过 | 跳过 | 仅变更章节 |
| T2 | 增删故事 / 接口变更 | 裁剪 | 裁剪 | 目标 + 下游 |
| T3 | 边界变化 / 跨故事重构 | 完整重跑 | 完整重跑 | 全级联刷新 |

### 补充文档（pm 主导，按需）

按编号 `10-{领域}.md` / `1x-{专题}.md` 创建，无匹配条件不生成：

| 触发 | 文档 |
|------|------|
| UI 改造 | 页面设计 |
| API 变更 | API 契约 |
| 数据存储变更 | 数据迁移方案 |
| 第三方集成 | 集成方案 |
| 新权限控制 | 权限模型 |
| 性能敏感 | 性能基准 |

### 参考文档（索引 + 主体章节）补充约束

8. 统一结构：索引（入口）+ 分层主体（参考手册 / 架构蓝图 / 视觉规范 / 用户手册）
9. 索引含身份卡片 + 角色路径 + 文件清单，不含技术细节
10. 详细章节见 [coder.md §参考文档公式](../skills/rui/coder.md#参考文档公式)

### 策展

11. 策展阶段必须 git commit
12. `--from-code` 空输入：pm 自主扫描源码输出推荐列表，等用户选择后生成参考文档（见 [agents/pm.md](../agents/pm.md)）

## 例外

- T1 级变更可跳过影响分析与架构设计章节
- 参考文档不走故事八件套，使用对应专属骨架
