# Document Generation Rules

> **口诀：版头齐、目录清、证据足、产出聚、裁剪准。** 五条强制约束，缺一即偏差。故事文档公式见 [skills/rui/formulas.md](../skills/rui/formulas.md)，目录生命周期 / 参考文档公式 / 数据契约见 [skills/rui/coder.md](../skills/rui/coder.md)。

## 1. 版头齐（格式）

- **版本行必填**：`v{版本} | {YYYY-MM-DD} | {模型} | {分支}`，公式占位符 `{...}` 留到产出即 P0 偏差
- **导航块**：01–04 文件首尾含标准导航（← 前 · ↑ 索引 · 后 →）；00-索引不含
- **图先文后**：架构 / 流程 / 关系优先 mermaid，文字补细节（CLAUDE.md「图为首」）

## 2. 目录清（命名 — 唯一定义）

所有 rui 产出按项目独立子目录组织，缺项目级目录阻断（`no-project-prefix`）：

| 文档类 | 路径模式 | 用途 |
|--------|---------|------|
| 故事 | `docs/故事任务面板/<Project>/<name>/` | 执行（做什么 / 怎么做 / 做了什么） |
| 组件 | `docs/组件文档/<Project>/<component-name>/` | 参考（前端组件当前状态） |
| 接口 | `docs/接口文档/<Project>/<resource-name>/` | 参考（后端 API 当前状态） |
| 页面 | `docs/页面文档/<Project>/<page-name>/` | 参考（页面编排与交互） |
| 领域 | `docs/领域模型/<Project>/<domain-name>/` | 参考（领域结构与规则） |

约束：`<Project>` 大驼峰；`<name>` kebab-case；总路径 ≤ 96 字符。

## 3. 证据足（内容）

- **证据等级**：A/B 可写入，C 标注 `> 待补充`，D（无支撑）禁止（见 [agents/AGENT.md](../agents/AGENT.md)）
- **不编造**：未验证的模块名 / 接口 / 路径 / 文件名不得出现
- **引用格式**：故事文档引用参考文档先指向 00-索引，按需深入具体章节

## 4. 产出聚（边界）

- 关键产出只允许在对应目录下，不得跨目录生成
- 文件按管线阶段创建，不可提前：

| 阶段 | 创建 | 条件 |
|------|------|------|
| 文档生成 | 01 + 02/03/04 + 补充（10 / 1x） | 01 必创建；02/03 按项目类型 |
| 验证 | 05 / 06 / 07 | 有对应技术评审时 |
| 自改进 | 08 | 必创建 |
| 交付 | 00-消息通知列表 | 自动 |

## 5. 裁剪准（增量）

变更分级决定裁剪范围：

| 级别 | 范围 | 影响分析 | 架构设计 | 文档刷新 |
|------|------|---------|---------|---------|
| T1 | 措辞 / 格式 | 跳过 | 跳过 | 仅变更章节 |
| T2 | 增删故事 / 接口变更 | 裁剪 | 裁剪 | 目标 + 下游 |
| T3 | 边界变化 / 跨故事重构 | 完整重跑 | 完整重跑 | 全级联刷新 |

## 补充文档决策（pm 主导）

按需创建（编号 `10-{领域}.md` / `1x-{专题}.md`），无匹配条件不生成：

| 触发 | 文档 |
|------|------|
| 涉及 UI 改造 | 页面设计 |
| 新增 / 修改 API | API 契约 |
| 数据存储变更 | 数据迁移方案 |
| 第三方集成 | 集成方案 |
| 新权限控制 | 权限模型 |
| 性能敏感 | 性能基准 |

补充文档无固定公式，按对应主线文档章节风格 ad-hoc 生成。

## 参考文档（00–04）补充约束

- 统一结构：00-索引 + 01–04 分层
- 00-索引：阅读入口，30 秒定位。含身份卡片 + 角色路径 + 文件清单。不含技术细节
- 01–04 一句话定位：
  - 01 = 参考手册（查阅型）
  - 02 = 架构蓝图 / 数据字典（理解型）
  - 03 = 视觉规范 / 安全白皮书（审查型）
  - 04 = 用户手册 / 集成手册（验证型）
- 详细章节见 [coder.md §参考文档公式](../skills/rui/coder.md#参考文档公式) F.ref.\* 块

## 策展

- 策展阶段必须 git commit
- `--from-code` 空输入：pm 自主扫描源码输出推荐列表，等用户选择后生成参考文档（见 [agents/pm.md](../agents/pm.md)）
