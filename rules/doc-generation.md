# Document Generation Rules

> 完整文档规范见 [skills/rui/docs.md](../skills/rui/docs.md)。本文件定义生成时的强制约束。

## 格式约束

1. **版本行必填**: 所有文档开头 `v{版本号} | {YYYY-MM-DD} | {模型} | {分支}`，模板占位符不得保留到产出
2. **导航块**: 01~04 文件首尾包含标准导航（← 前驱 · ↑ 索引 · 后继 →），00-索引不需要后继导航
3. **mermaid 优先**: 架构/流程/关系用 mermaid 图表达，文字补充细节。符合 CLAUDE.md「图为首」准则

## 目录命名（唯一定义）

4. 所有 rui 产出按项目独立子目录组织，缺失项目级目录阻断（`no-project-prefix`）:

   | 文档类 | 路径模式 | 用途 |
   |--------|---------|------|
   | 故事 | `docs/故事任务面板/<Project>/<name>/` | 执行文档（做什么/怎么做/做了什么） |
   | 组件 | `docs/组件文档/<Project>/<component-name>/` | 参考文档（前端组件当前状态） |
   | 接口 | `docs/接口文档/<Project>/<resource-name>/` | 参考文档（后端 API 当前状态） |
   | 页面 | `docs/页面文档/<Project>/<page-name>/` | 参考文档（页面编排与交互） |
   | 领域 | `docs/领域模型/<Project>/<domain-name>/` | 参考文档（领域结构与规则） |

   `<Project>` 大驼峰，`<name>` kebab-case，总路径 ≤ 96 字符。

## 内容约束

5. **产出内聚**: 关键产出只允许在对应目录下，不得跨目录生成
6. **证据标准**: Level A/B 可写入，Level C 标注 `> 待补充`，Level D（无支撑）禁止
7. **不编造**: 未验证的模块名/接口/路径/文件名不得出现在文档中
8. **引用格式**: 故事文档引用参考文档时指向 00-索引，按需深入具体文件章节

## 增量裁剪

9. 变更分级决定裁剪范围:

   | 级别 | 范围 | 影响分析 | 架构设计 | 文档刷新 |
   |------|------|---------|---------|---------|
   | T1 | 措辞/格式 | 跳过 | 跳过 | 仅变更章节 |
   | T2 | 增删故事/接口变更 | 裁剪 | 裁剪 | 目标+下游 |
   | T3 | 边界变化/跨故事重构 | 完整重跑 | 完整重跑 | 全级联刷新 |

## 文件创建时序

10. 文件按管线阶段创建，不可提前:

    | 阶段 | 创建文件 | 条件 |
    |------|---------|------|
    | 文档生成 | 01 + 02/03/04 + 补充文档(10/1x) | 01 必创建，02/03 按项目类型 |
    | 验证 | 05/06/07 | 有对应技术评审时 |
    | 自改进 | 08 | 必创建 |
    | 交付 | 00-消息通知列表 | 自动 |

## 补充文档

11. pm 按决策表判断是否创建补充文档（编号 `10-{领域}.md` / `1x-{专题}.md`）:
    - 涉及 UI 改造 → 页面设计
    - 涉及新增/修改 API → API 契约
    - 涉及数据存储变更 → 数据迁移方案
    - 第三方集成 → 集成方案
    - 新权限控制 → 权限模型
    - 性能敏感 → 性能基准
    
    无匹配条件不生成。

## 策展

12. 策展阶段必须 git commit
13. `--from-code` 空输入: pm 自主扫描源码输出推荐列表，等用户选择后生成参考文档

## 参考文档（00~04）生成规则

14. 参考文档（组件/接口/页面/领域）统一结构: 00-索引 + 01~04 分层文档
15. **00-索引**: 阅读入口，30 秒定位。含身份卡片 + 角色阅读路径 + 文件清单。不含技术细节
16. **01~04**: 按「查阅→理解→审查→验证」递进。每份文件有明确的一句话定位:
    - 01 = API 参考手册（查阅型）
    - 02 = 架构蓝图/数据字典（理解型）
    - 03 = 视觉规范/安全白皮书（审查型）
    - 04 = 用户手册/集成手册（验证型）
17. 章节结构详见 [docs.md](../skills/rui/docs.md) 对应文档类型章节
