📢 rui-bot · 企微通知

企业微信 Webhook 消息推送 — 1-5 数字键切换 Tab · 悬停代码复制

🚶 演练 📊 概览 ⚡ 命令 📁 文件 🧪 测试
发送完成通知
故事完成时通过企微 Webhook 推送状态变更通知到群聊。API_X_TOKEN 从环境变量注入，消息含故事名、阶段、结论等字段。
node skills/rui-bot/send.mjs --story=user-login --status=complete📋
调试模式：仅追加日志不发送
仅追加通知日志但不发送 HTTP 请求。适用于本地测试或 token 不可用时的降级模式。
node skills/rui-bot/send.mjs --story=user-login --status=complete --no-send📋
发送阻断通知
故事遇到阻断时发送阻断通知，包含阻断标识和修复建议。支持 complete / blocked / gate-fail 三种状态。
node skills/rui-bot/send.mjs --story=user-login --status=blocked📋
健康检查
检查 token 是否存在、config.json 配置完整性、webhook URL 可达性。不发送消息，仅输出健康状态。
node skills/rui-bot/send.mjs health📋
管线集成：交付收口第三步
rui 管线 delivery-gate 的第三步 hook：追加日志 → 文档同步 → 企微通知。通知含 stage / fileStats / conclusion / nextStep 等完整字段。
node skills/rui-bot/send.mjs --story=user-login --status=complete --stage=deliver --conclusion=done📋

默认命令
send.mjs --story= --status=
发送通知到企微群（先追加日志，再 HTTP 发送）。status 支持 complete / blocked / gate-fail。
node skills/rui-bot/send.mjs --story=name --status=complete📋
调试模式
send.mjs --no-send
仅追加通知日志不发送 HTTP 请求。token 不可用时的降级模式，管线默认行为。
node skills/rui-bot/send.mjs --story=name --status=complete --no-send📋
健康检查
send.mjs health
检查 token / config.json / webhook URL / API 可达性。不发送消息。
node skills/rui-bot/send.mjs health📋
阻断通知
send.mjs --status=blocked
故事阻断时发送通知，消息含阻断标识和修复路径。
node skills/rui-bot/send.mjs --story=name --status=blocked📋
完整参数
send.mjs 含可选字段
支持 --stage / --fileStats / --conclusion / --description / --scope / --nextStep / --impact / --evidence 等扩展字段。
node skills/rui-bot/send.mjs --story=name --status=complete --stage=deliver --conclusion=done --scope=full📋
