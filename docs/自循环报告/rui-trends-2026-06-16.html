根因分析: 全源故障

🔴 根因 1: GitHub API 限速
GitHub Trending 和 Top-Starred 同时 fetch failed，且均依赖 GitHub API (search/trending)。未认证请求限额 60 req/h，连续两个源调用可能触发。
证据：两源同时失败，其他非 GitHub API 源未出现同类 fetch failed
修复：增加调用间隔至 30s + GITHUB_TOKEN 认证提升限额至 5000 req/h
🟡 根因 2: TrendShift 页面结构变更
API 返回 1 条 malformed 记录 (ID "54341", 名称 "Sponsorships//Open")，说明页面可达但 DOM 解析选择器失效。
证据：06-05 同样乱码 → 06-09/06-12 可达 → 06-16 再次乱码，模式不连续
修复：增加备用 CSS 选择器路径
🟡 根因 3: OSS Insight DOM 渐进变更
trending-repositories 端点连续两周期异常：06-12 可达但 06-16 返回空。页面 HTTP 200 但 HTML 解析未提取条目。
证据：间歇性而非持续性故障，暗示渐进式 DOM 变更
修复：更新解析选择器 + 增加 API 响应结构快照对比
🔵 排除因素
网络分区：排除 — OSS Insight 和 TrendShift 页面可达 (HTTP 200)
DNS 故障：排除 — 所有域名可解析
TLS 证书：排除 — HTTPS 握手成功
GitHub 整体宕机：排除 — github.com 正常访问
🚫 升级条件：连续 2 周期 ≥2 源不可达 → P0 自动升级。06-12 已有 1/4 源不可达 (TrendShift 06-12 实际可达，是 06-05 乱码)。本周期 4/4 不可达触发 P0。
