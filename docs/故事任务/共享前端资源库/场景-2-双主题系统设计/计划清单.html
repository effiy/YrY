📋 需求与优先级

需求优先级来源
架构图/知识图谱使用 Cat A Mono 主题P0设计规范
文档/场景页使用 Cat B System 主题P0设计规范
14 设计令牌跨主题可移植P0架构一致性
CSS 变量分层避免冲突P1可维护性

🎯 实施步骤

  1. §2.1 Cat B 默认页面引用 — shared/index.css + theme/index.css
  2. §2.2 Cat A 架构图页面引用 — shared + theme-mono + fonts (JetBrains Mono)
  3. §2.3 组件使用设计令牌 (带 fallback) — var(--yry-*, fallback) 双参数调用
  4. §2.4 令牌分层原则 — L1 Surfaces → L2 Brand → L3 Semantic → L4 Text → L5 Elevation
  5. §2.5 WCAG 对比度矩阵 — 文字令牌 ≥ 4.5:1 (AA) · 大文字 ≥ 3:1
  6. §2.6 主题运行时切换 — data-theme 属性 + localStorage 持久化
  7. §2.7 令牌命名规范 — 按用途命名，禁止按颜色命名 (--yry-blue 为反模式)

✅ 验收清单

  • Cat A 页面渲染正确 (架构图/源码/知识图谱)
  • Cat B 页面渲染正确 (文档/场景页/首页)
  • 14 令牌全部定义 (Surfaces 3 + Brand 2 + Semantic 3 + Text 3 + Elevation 3)
  • 跨主题可移植 (切换 CSS 引用组件样式自动适配)
  • 令牌 fallback 生效 (移除 theme CSS 组件仍可读)
  • WCAG AA 对比度达标
  • 组件 CSS 无硬编码颜色
  • 主题切换无 FOUC (录屏首帧检查)
  • 高对比度模式 (prefers-contrast: more)
  • 亮色主题可用 (prefers-color-scheme: light)
