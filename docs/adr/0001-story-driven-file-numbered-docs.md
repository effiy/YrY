# 0001-story-driven-pipeline-with-file-numbered-docs

决策：采用故事驱动 SDLC 管线，每故事用编号文件（01–08）记录完整生命周期。

**为什么？** 传统 AI coding 失败模式：(1) Agent 误解需求→产出偏差 (2) 没有测试先行→代码不可靠 (3) 没有核查清单→问题被遗漏 (4) 过程不可追溯→无法复盘改进。编号文件矩阵强制每个阶段产出可验证的文档，管线阻断点防止跳过关键步骤。

**拒绝的方案：** 自由文档（无编号约定）——已证明 agent 产出散乱不可追踪。全脚本驱动——过度僵化，agent 智能被浪费。

**后果：** 故事文档数量固定但内容灵活。每文档有公式规约（formulas.md）但不依赖模板文件。编号强制顺序感和完整度检查。
