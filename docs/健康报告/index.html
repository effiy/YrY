📐 评估体系 · 19 维加权模型

综合健康指数 CHI 由三大类共 19 个维度加权聚合而成，每维度评分 0–100，通过加权综合得出最终评级（A≥80/B≥60/C≥40/D<40）。

▸ 9 核心维度（基础保障层 · 默认权重 55%）
① Token 安全 — API/X_TOKEN 仅通过环境变量传入·不落盘·不硬编码·定期轮换检查
② 配置健康 — .claude/settings.json 完整性·plugin.json 版本一致性·无冗余配置
③ 机器人就绪 — robots.txt 存在且格式规范·无过度封锁·爬虫友好
④ API 可达性 — 外部依赖 URL 可达·API 响应正常·超时与重试策略完备
⑤ 报告质量 — 报告文件存在·格式规范·数据可追溯·无空字段·时间戳精确
⑥ 格式规范 — Markdown/YAML/JSON 语法正确·缩进统一·无尾随空格
⑦ 诊断引擎 — D0-D7 诊断覆盖率·误报率控制·响应时间·建议可执行率
⑧ Git 纪律 — 分支命名规范·commit message 规范·PR 审查·未提交文件数控制
⑨ 安全基线 — 无高危依赖·认证不可绕过·输入必校验·XSS/注入防护

▸ 7 工程成熟度维度（工程实践层 · 默认权重 35%）
① 测试覆盖(30%) — 单元·集成·E2E 测试覆盖率·用例通过率·边界条件覆盖
② 类型安全(15%) — TypeScript/JSDoc 类型标注覆盖·严格模式·any 使用控制
③ 代码检查(15%) — ESLint·Prettier·arch-check 配置完善·规则严格度
④ CI/CD(10%) — 自动化流水线·门禁检查·自动部署·构建时长
⑤ 文档覆盖(10%) — API 文档·README 完整性·场景文档·知识图谱完备
⑥ 依赖管理(10%) — 版本锁定·安全审计(npm audit)·许可证合规·更新频率
⑦ Git 实践(10%) — 分支策略·commit 规范·PR 审查·worktree 隔离

▸ 3 扩展维度（系统运维层 · 默认权重 10%）
① 文件体积 — lib ≤ 20 文件·编排器 ≤ 500 行·总大小受控
② 依赖分析 — 循环依赖检测·模块耦合度·import 健康度
③ 通知就绪 — 企微 Webhook 配置·消息格式规范·失败队列管理

诊断联动：维度评分 < 55 自动触发对应 D0-D7 诊断 · 连续 2 次同维度 C 级 → P1 优先级 · 连续 3 次 → 自动升级 P0
趋势持久化：每次 health 检查自动追加 .memory/health-trend.jsonl（JSONL 格式）· 输入自改进分析与趋势预测面板
生成命令：node skills/rui-bot/send.mjs health --html 生成报告 · --notify 推送企微 · --short 单行摘要 · --alert 智能告警
