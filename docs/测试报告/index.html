📐 三维加权评估模型

测试质量指数 TQI = 代码覆盖率 × 50% + 用例通过率 × 30% + 用例充分度 × 20%

① 代码覆盖率(50%) — 四维覆盖指标综合：行覆盖率≥80%满分·分支覆盖率≥70%满分·函数覆盖率≥75%·语句覆盖率≥85%。数据源：vitest --coverage（c8/istanbul 提供）。
② 用例通过率(30%) — 单元测试·集成测试·E2E 测试三项通过率均值。单元测试≥95%满分·集成测试≥90%满分·E2E≥85%满分。数据源：vitest run 结果。
③ 用例充分度(20%) — 每模块/组件平均用例数≥5满分·边界条件覆盖·异常路径覆盖·等价类划分。数据源：test manifest 统计。

联动评估：工程成熟度 7 维（测试覆盖·类型安全·代码检查·CI/CD·文档·依赖·Git 实践）作为参考维度，加权模型与健康报告共用同一数据源 docs/自我改进/summary.json。
评级标准：A≥80 优秀 · B≥60 良好 · C≥40 需改进 · D<40 严重缺陷 · 每 5min 自动刷新
