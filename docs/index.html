📐 架构合规评分 · 10 维度 由 node lib/arch-check.mjs 自动验证
架构合规检查覆盖 10 个维度，是 AQI（架构质量指数）的数据源。 评级标准：A ≥ 90% 合规 · B ≥ 75% · C ≥ 60% · D ＜ 60%。 支持 --fix 自动修复与 --append-trend 趋势持久化。
📏 内核体积
lib/ 共享库 ≤ 20 文件 · 编排器 ≤ 500 行。保持内核轻量，扩展通过 Skill 注册实现。
🚫 范式合规
禁止 class/extends · 禁止 export default · 禁止空 catch 块 · 函数式编程范式。
🔌 扩展隔离
新增 Skill 不修改编排器代码。技能通过 skills/ 目录注册，编排器自动发现。
📄 文档新鲜度
场景 HTML 7 日内更新率。规约文件与实现代码的一致性校验。退化检测自动告警。
🎯 SRP 单一职责
每个 Skill 单一目的 · 每个 lib/ 文件单一职责。函数不超过 50 行，模块不超过 500 行。
🔄 DRY 不重复
共享代码统一放 lib/，禁止 copy-paste 跨文件。各脚本从 lib/ 导入通用函数/常量。
🔓 OCP 开闭原则
对扩展开放，对修改关闭。新功能通过新增 Skill 实现，不修改已有编排逻辑。
🧩 ISP 接口隔离
Skill 之间通过规约契约通信，不依赖对方的内部实现。Agent 交接信号可被下游验证。
💡 运行 node lib/arch-check.mjs 查看完整 10 维合规报告。 运行 node lib/arch-check.mjs --fix 自动修复可修复项。 运行 node lib/arch-check.mjs --append-trend 追加趋势记录。
