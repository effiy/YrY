## §2 场景详述

### 场景 1: 新接手项目

| 角色 | 开发者 |
|------|--------|
| 触发条件 | 首次接触代码库 |
| 核心目标 | 建立全局认知，识别核心模块 |

```mermaid
flowchart LR
    A["执行基础统计"] --> B["了解文件/模块规模"]
    B --> C["发现大文件"]
    C --> D["全局认知建立"]
```

### 场景 2: 重构前健康检查

| 角色 | 开发者 |
|------|--------|
| 触发条件 | 计划重构或架构调整 |
| 核心目标 | 定位大文件、不稳定模块和组件依赖 |

```mermaid
flowchart LR
    A["定位超大文件"] --> B["识别变更热点"]
    B --> C["审计组件依赖"]
    C --> D["制定重构计划"]
```

### 场景 3: 日常迭代监控

| 角色 | 开发者 |
|------|--------|
| 触发条件 | 常规开发周期 |
| 核心目标 | 追踪修改集中区域，了解外部趋势 |

```mermaid
flowchart LR
    A["查看变更热点"] --> B["关注修改集中区"]
    B --> C["了解外部趋势"]
    C --> D["保持技术敏感度"]
```
