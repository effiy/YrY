场景 6: 第三方框架与服务自检 — 双轨架构

双轨自检 · vitest 工程化轨道 + 遗留轨道 · FP13 19 项 + FP14 6 项 · 阻断/警告分级 · 双轨独立运行 · 零风险迁移

⚙ 工程化轨道 (vitest)

  • • FP13: vitest 框架存在性 — 19 项阻断级检查
  • • FP14: 第三方服务可达性 — 6 项警告/阻断级检查
  • • package.json: vitest + @vitest/ui devDependencies
  • • vitest.config.mjs: include/exclude/isolation/timeout/coverage
  • • vitest-adapter.mjs: assert API 映射到 vitest expect

📦 遗留轨道 (custom harness)

  • • tests/skills/*.test.mjs — 7 技能测试套件
  • • tests/agents/agents.test.mjs — Agent 定义测试
  • • tests/rules/rules.test.mjs — 规则定义测试
  • • tests/integration/*.test.mjs — 3 集成测试
  • • node tests/run.mjs — 零依赖兜底运行器

✅ 双轨兼容基线

  • • vitest 和 legacy 独立运行，互不依赖
  • • 一个失效不影响另一个
  • • npm test: vitest run (工程化)
  • • npm run test:legacy: node tests/run.mjs (遗留)
  • • npm run test:all: 双轨全量
