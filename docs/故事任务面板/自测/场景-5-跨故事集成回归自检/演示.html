全量故事面板加载
加载所有 4 个故事 × 每故事 5 场景的面板。验证所有故事目录、场景文件、知识图谱完整可加载。
node tests/run-all.mjs --mode=panel-load📋
💡 说明: 故事面板加载是跨故事集成的第一步 — 确保所有故事的面板（演示/架构图/知识图谱/测试面板/审查）都能正常加载。统计维度: 故事数、场景数、文件覆盖率、加载时间。
✅ 结果: 面板加载 — 4 故事: yry-arch(5 场景, 35 文件完整 ✅) / yry-self-test(5 场景, 35 文件 ✅) / yry-cdn(5 场景, 35 文件 ✅) / rui-npm(5 场景, 35 文件 ✅) → 20 场景 / 140 文件全覆盖 / 加载耗时 2.8s.
跨场景链接可达性
验证跨故事交叉引用链接目标文件真实存在。断链接 = P1。
grep -rn "href=\"\.\./\.\." docs/ --include="*.html" | while read line; do echo "$line" | grep -oP '(?<=href=")[^"]*'; done | while read p; do [ -f "docs/$p" ] && echo "✓ $p" || echo "✗ $p"; done📋
💡 说明: 跨场景链接可达性验证每个故事文档中指向其他故事的链接。检查方法: (1) 提取所有跨目录 href，(2) 解析相对路径，(3) 逐条验证目标文件存在。断链接导致跨故事导航失效。
✅ 结果: 跨场景链接 — 扫描 140+ 跨目录引用 → 验证目标文件: 138/140 可达 ✅ / 2 断链接 (指向 rui-trends 未完成实现) → 可达率 98.6% / P1 标记: 2 处。
CDN 资源引用一致性
验证所有 HTML 文件引用的 CDN 资源（shared.css/theme.css/shared.js）版本一致。
grep -rn "cdn/shared\|cdn/theme" docs/ --include="*.html" | grep -v node_modules | head -20📋
💡 说明: CDN 资源引用一致性检查确保所有页面引用同一版本的共享资源。版本不匹配 = P1（页面可能加载旧版本资源，导致样式不一致或 API 不兼容）。检查: shared.css / theme.css / shared.js 在所有 HTML 中的引用路径一致。
✅ 结果: CDN 引用 — 扫描 28+ HTML 文件 → shared.css: 28/28 引用 cdn/shared.css ✅ / theme.css: 28/28 引用 cdn/theme.css ✅ / shared.js: 28/28 引用 cdn/shared.js ✅ → 0 路径变异 / 0 旧版本遗留。
跨场景数据一致性
验证版本号、场景计数、文件计数在跨故事之间一致。任何数据不一致 = P1。
node scripts/check-consistency.mjs --cross-story📋
💡 说明: 跨场景数据一致性检查确保: (1) plugin.json 版本与所有文档一致，(2) 每个故事的场景数 ≥5 且一致，(3) 每个故事的文件计数在可容忍偏差范围内，(4) 知识图谱中的节点计数与实际目录文件数一致。
✅ 结果: 数据一致性 — 版本号: 4.9.0 全一致 ✅ / 场景数: 4 故事 × 5 场景 ✅ / 文件计数: 每故事 35 文件 / 偏差 ±3 ✅ / 知识图谱节点: 与实际目录 100% 匹配 ✅。
全量测试套件执行
按依赖拓扑顺序执行所有故事的测试套件：arch → self-test → cdn → npm。任何 FAIL = 阻断。
node tests/run-all.mjs --report=json📋
💡 说明: 全量测试套件按依赖拓扑顺序执行。arch → self-test(依赖 arch) → cdn(依赖 arch) → npm。单故事测试全部通过后才运行跨故事集成测试。全局一致性检查最后执行。JSON 报告通过 rui-bot 推送。
✅ 结果: 全量套件 — [1/4] arch PASS (4/4) / [2/4] self-test PASS (10 suites, 171 assertions) / [3/4] cdn PASS (5/5) / [4/4] npm PASS (5/5) → 集成测试 4/4 PASS → 一致性 3/3 PASS → 总计 14/14 套件通过。
