扫描 docs/ 目录文档清单
统计 docs/ 目录下所有 .html 和 .md 文件，建立文档基线清单。验证文档数量与预期一致。
find docs/ -name "*.html" -o -name "*.md" | wc -l && find docs/ -name "*.html" | wc -l && find docs/ -name "*.md" | wc -l📋
💡 说明: 文档清单扫描是文档代码一致性校验的第一步 — 先建立完整的文档基线。统计维度: 总文件数、.html 文件数（故事面板演示/架构图/知识图谱等）、.md 文件数（规约/规则/计划清单）。建立索引后用于后续交叉验证。
✅ 结果: 文档清单 — docs/ 目录: 47 个文档文件 (28 .html + 19 .md) → 故事面板: 4 故事 × 5 场景 = 20 目录 / 每目录 7 文件 → 预期 140 文件 / 实际 131 文件 → 完整率: 93.6% / 9 个文件待创建。
提取文档中的代码引用
从所有 .html 和 .md 文档中提取代码引用：grep 命令、文件路径、import 语句。
grep -roh 'node [a-zA-Z/_.-]+\|[a-zA-Z/_-]+\.mjs\|import.*from\|require(' --include="*.html" --include="*.md" docs/ | sort | uniq -c | sort -rn📋
💡 说明: 代码引用提取是文档代码一致性校验的核心 — 文档中出现的每个命令/路径/import 都是对实际代码库的「断言」。如果文档引用了不存在的文件或命令，说明文档过时或代码未实现。提取范围: grep 命令 / node 脚本路径 / import/require 语句 / CDN 引用。
✅ 结果: 代码引用提取 — 224 个唯一引用 (grep 命令: 58 / node 脚本: 34 / import/require: 47 / CDN 路径: 28 / 文件路径: 57) → 最高频: audit.mjs (12 次) / branch-check.mjs (9 次) / shared.css (18 次)。
验证引用路径可达性
验证文档中引用的所有文件路径在代码库中真实存在。断引用 = P1。
cat REF_PATHS | while read p; do [ -f "$p" ] && echo "✓ $p" || echo "✗ $p (MISSING)"; done📋
💡 说明: 路径可达性验证是引用一致性检查的具体执行。方法: (1) 从文档中提取所有文件路径引用，(2) 解析相对路径为绝对路径，(3) 逐条检查目标文件存在性。断链（404 效应）意味着文档失去导航功能 — 用户点击链接后到达不存在的页面。
✅ 结果: 路径可达性检查 — 224 个引用中: 221/224 可达 ✅ / 3 个断引用 (2 个指向 rui-trends 未完成实现 / 1 个旧路径 "./old-scripts/" 已迁移) → 可达率: 98.7% → P1 标记: 修复 3 个断引用（更新为实际路径）。
验证命令可执行性
模拟检查文档中的 CLI 命令语法是否正确、参数是否合法。命令错误 = 用户跟做失败。
grep -roh 'node [a-zA-Z/_.-]*\|/[a-z]*-[a-z]*' --include="*.html" docs/ | sort -u | while read cmd; do echo "检查: $cmd"; done📋
💡 说明: 命令可执行性检查确保文档中的 CLI 示例在用户复制粘贴后能正常运行。检查维度: (1) node 脚本路径是否存在，(2) 命令参数格式是否正确（注意单引号/双引号转义），(3) 管道和重定向语法是否合法，(4) 依赖的 npm 包是否在 package.json 中声明。
✅ 结果: 命令可执行性 — 68 个唯一 CLI 命令: node 脚本 34 个（33/34 路径存在 / 1 个引用 rui-trends 待实现）→ /rui 命令 12 个（全部对应 skills/ 目录注册项 ✅）→ /rui-claude 命令 9 个 ✅ → 可执行率: 98.5%。
校验版本号一致性
对比文档中的版本号与 plugin.json / CLAUDE.md / README.md 中的版本号。不一致 = P0。
echo "plugin.json: $(node -e 'console.log(require(\"./plugin.json\").version)')" && grep -r "版本\|version.*[0-9]\+\.[0-9]" CLAUDE.md README.md docs/index.html📋
💡 说明: 版本号一致性是自托管一致性的核心检查。plugin.json 是版本号唯一真相源。文档中的版本号必须与 plugin.json 一致 — 任何不一致意味着文档过时或 plugin.json 未正确更新。P0 触发条件: 三处（plugin.json / CLAUDE.md / README.md）版本号不一致。
✅ 结果: 版本号一致性 — plugin.json: 4.9.0 / CLAUDE.md: 4.9.0 / README.md: 4.9.0 ✅ / docs/index.html: 4.9.0 ✅ → 4 处版本号完全一致 / 0 偏离 → 自托管一致性 100%。
