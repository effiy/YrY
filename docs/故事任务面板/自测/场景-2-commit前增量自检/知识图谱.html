五步增量门禁

  • ① 变更范围识别 — git diff 检测修改/新增/删除文件，映射受影响模块
  • ② 分支隔离检查 — 验证当前在 feat/<name> 分支，主分支变更立即阻断
  • ③ 变更范围约束 — 变更文件全部在本故事目录内，跨故事污染阻断
  • ④ P0 回归检查 — 变更前后阻断标识数对比，新 P0 出现立即阻断
  • ⑤ 文档一致性 — 代码变更时验证关联场景文档是否同步更新

源文件映射 (7)

  • scripts/self-test.mjs — 全量/增量自检入口，--quick 增量模式
  • scripts/detect-impact.mjs — 变更范围自动识别，--since 读取 git diff
  • skills/rui/branch-check.mjs — P0 分支隔离强制门禁
  • tests/run.mjs — 测试发现 + 筛选 (--skills/--agents/--rules)
  • tests/integration/cross-references.test.mjs — 跨模块一致性 + 安全基线
  • tests/integration/knowledge-graph.test.mjs — KG 结构验证
  • tests/rules/rules.test.mjs — 8 规则完整性验证

阻断判定规则

  • 🔴 阻断: 分支隔离绕过 · 跨故事污染 · 新 P0 出现
  • 🟡 警告: 文档未同步 · 变更范围异常大 · 分支名不规范
  • 🟢 通过: 五步全部无阻断 · 允许提交
  • 纯文档变更跳过代码检查 · 纯代码变更触发文档同步检查
