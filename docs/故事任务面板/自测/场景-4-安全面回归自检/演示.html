密钥泄漏扫描
扫描所有源码和配置文件中的 API_KEY/TOKEN/SECRET 模式，确保无硬编码凭据。API_X_TOKEN 仅通过环境变量传入。
grep -rn "API_KEY\|TOKEN\|SECRET\|api_key\|sk-\|Bearer" --include="*.mjs" --include="*.json" --include="*.html" . | grep -v node_modules | grep -v process.env📋
💡 说明: 密钥不落盘是不可妥协底线 #2。grep 扫描常见密钥模式 → 排除 process.env 引用（合法）→ 排除文档中的示意代码 → 任何剩余匹配 = P0。
✅ 结果: process.env.API_X_TOKEN 合法引用: 4 处 / 硬编码: 0 处 ✅ / plugin.json 无 token ✅ → 密钥不落盘 100% 合规。
XSS 注入点检测
检测所有危险 DOM 操作：innerHTML / document.write / eval / Function()。任何未转义的 HTML 内容插入为 P0。
grep -rn "innerHTML\|document\.write\|eval(\|Function(" --include="*.mjs" --include="*.html" . | grep -v node_modules📋
💡 说明: XSS 注入点检测是输入必校验的具体落地。innerHTML 直接插入未转义 HTML = XSS 入口、document.write 页面加载后写入 = 注入点、eval/Function() = 任意代码执行。合法使用需配合 escapeHtml().
✅ 结果: innerHTML: 6 处（均使用 YrY.escapeHtml() 转义 ✅）/ document.write: 0 ✅ / eval/Function(): 0 ✅ → XSS 风险: 0。
认证绕过检查
验证所有涉及 API 调用的路径都有认证保护。检查是否有跳过认证的条件分支或未授权路径访问。
grep -rn "auth\|认证\|bypass\|skip.*auth\|fetch\|axios" --include="*.mjs" . | grep -v node_modules📋
💡 说明: 认证不可绕过是不可妥协底线 #1。检查: (1) 所有 fetch/axios/http 调用是否附带 API_X_TOKEN header，(2) 是否有 if(DEBUG) skipAuth() 条件分支，(3) 是否有未授权路径直接访问敏感端点。
✅ 结果: API 调用点: 4 处（全部附带 API_X_TOKEN header ✅）/ 0 bypass 条件 / 0 debug 跳过 / 0 未授权直接访问 → 认证覆盖 100%。
输入校验覆盖
验证所有外部输入入口（CLI 参数/API 响应/文件读取）都有对应的校验逻辑。边界 case 处理完整。
grep -rn "validate\|sanitize\|escape\|process\.argv\|process\.env" --include="*.mjs" . | grep -v node_modules📋
💡 说明: 输入必校验是不可妥协底线 #3。检查: (1) process.argv 入口是否有 validate/sanitize，(2) 文件读取是否有路径遍历防护，(3) API 响应是否有 try-catch，(4) 边界 case: 空字符串/超长输入/特殊字符。
✅ 结果: 外部入口: 8 处（均含校验 ✅）/ 校验覆盖: 8/8 有防护 / 12/12 边界 case 通过。
分支隔离验证
验证 feat/ 分支保护机制有效。检查跨分支状态共享、主分支保护、缓存隔离。
node skills/rui/branch-check.mjs --story=yry-self-test --mode=write📋
💡 说明: 分支隔离不可绕过是不可妥协底线 #7。检查: 当前分支是否为 feat/<name>、main 是否受保护、.memory/ .claude/ 缓存是否跨分支共享、pre-commit hook 是否有 branch-check。
✅ 结果: branch-check.mjs — feat/enhance-s2-s5-simulators ✅ / main 保护 ✓ / 缓存隔离 ✓ / pre-commit hook 已配置 ✓ → 全合规。
