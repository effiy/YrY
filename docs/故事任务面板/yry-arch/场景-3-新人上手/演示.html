了解项目全貌
阅读 README.md 了解系统全景和领域语言（20 个术语）。CLAUDE.md 定义四大铁律和不可妥协底线。
cat README.md CLAUDE.md📋
✅ 结果: README.md: YrY 全景图 + 20 个领域语言术语定义（含 Avoid 禁用别名列）+ 6 个技能入口 + 角色拓扑概览。CLAUDE.md: 3 基础信念 + 4 铁律 + 项目约束 + 6 不可妥协底线 + 4 层退化对策。
💡 说明: 这两个文件是新人必读的「第一小时文档」。README 回答「这是什么」，CLAUDE.md 回答「怎么操作」。领域语言的 Avoid 列特别重要 — 列出了历史中曾使用但已废弃的术语别名，防止新旧概念混淆（退化对策 L2 重生机制）。
搭建开发环境
克隆仓库 → 配置 API_X_TOKEN 环境变量 → 运行 /rui init 初始化项目基线。
export API_X_TOKEN=your_token && /rui init📋
✅ 结果: API_X_TOKEN 从环境变量注入（密钥不落盘验证通过）→ /rui init 六阶段全部通过 → detect(6技能+9Agent+8规则) → generate(12个基线文件) → verify(PASS)。所有 rui marker 段已生成。
💡 说明: API_X_TOKEN 必须通过环境变量传入，禁止硬编码在源码或配置文件中（不可妥协底线 #2）。init 命令是幂等的 — 多次运行不会产生重复或冲突，marker 段每次全量重生。首次运行建议在 clone 后立即执行。
从源码反推理解
如果项目已有代码但缺少文档，用 --from-code 从源码反推故事文档基线，全程只读不修改。
/rui doc --from-code📋
✅ 结果: 只读扫描 skills/ agents/ rules/ lib/ → 识别 6 技能 + 9 Agent + 8 规则 + 4 共享库 → 生成故事任务.md + 4 场景文档 + 知识图谱.json (含源码路径引用) → 证据 Level B（源码路径可追溯）。
💡 说明: --from-code 模式对新人理解存量代码库尤其有用 — 不需要手动写文档，系统从源码自动提取结构和关系，生成人类可读的故事文档。全过程只读（EACCES 检查），不会修改任何源文件。生成文档的证据级别为 Level B（源码路径 + grep 可验证），不如 Level A（运行验证）强但适合快速建立基线。
完成首次修改
走完整管线：需求 → doc → code → test → deliver。分支隔离保护，Gate A/B 确保质量。
/rui 修复文档中的错别字📋
✅ 结果: pm 识别为 T1 措辞变更 → doc 生成场景卡片（影响范围: 仅 2 个 .md 文件）→ code 创建 feat/fix-typo 分支 → Gate A 通过 → 修改 2 处 → Gate B 1 轮通过 → deliver 合并到 main。
💡 说明: 首次修改走完整管线体验全流程。T1 级变更（措辞/格式）自动跳过深度分析，流程更快（约 2 分钟 vs T3 的 15 分钟）。分支隔离策略确保 main 分支始终可交付，feat 分支的实验不会污染主分支。这是理解管线运作的最佳方式 — 做一个真实的小修改。
理解管线纪律
阅读 code-pipeline.md 理解分支隔离策略、Gate A/B 关卡、逐模块 P0 清零流程。
cat rules/code-pipeline.md📋
✅ 结果: code-pipeline.md 定义完整管线纪律 — 分支隔离（feat/ 前缀强制 + main 保护）→ Gate A 准入（测试先行 + 安全约束注入 + 分支验证）→ 逐模块 P0 清零（Modules: N→1，Module: M→1 子任务，当前模块 P0 未清 → 阻断下一模块）→ Gate B 交付（≤2 轮修复 + 交付 hook 三步）。
💡 说明: 管线纪律是 YrY 最核心的治理机制。逐模块 P0 清零是「清先于进」铁律的直接体现 — P0 代表阻断级问题（安全漏洞/数据丢失/规约不一致），未清不得进入下个模块。不是鼓励完美主义（P1/P2 可以后续迭代），而是防止 P0 积累。
同步团队配置
rui-claude sync 从远端同步 .claude/ 配置，确保团队使用一致的 Agent 和规则。
/rui-claude sync📋
✅ 结果: 远端 .claude/ 配置 → 本地覆盖 — 同步 settings.json (权限+环境变量) / hooks.json (钩子脚本) / MCP 服务器列表 / CLAUDE.md 备份。diff 显示 3 个文件更新，0 冲突。
💡 说明: sync 确保团队成员使用统一的 Claude Code 配置（权限规则、钩子、MCP 服务器等），避免「在我机器上能跑」问题。首次入职或配置更新后必须运行。与 /rui-claude update 不同 — sync 仅同步配置不更新 YrY 插件本身，update 则先 git pull 再 sync。
