绘制依赖图
扫描 plugin.json 和 skills/ 目录，构建模块间依赖关系图。识别直接依赖和传递依赖。
✅ 结果: plugin.json 注册表: 6 技能依赖（rui → rui-story/rui-claude/rui-import/rui-bot/rui-trends, rui-npm 独立）→ 传递依赖链: rui→rui-claude→rui-import(tty.mjs) → lib/ 共享 4 模块。直接依赖 7 条，传递依赖 3 条。
💡 说明: 依赖图是影响分析的输入。plugin.json 记录了技能的注册依赖（直接依赖），但 lib/ 共享库的引用关系形成传递依赖。变更 lib/tty.mjs 会影响所有通过 rui-import 引用的技能，形成 2 层传递链。用 grep "from.*lib/" 可补充发现 plugin.json 未记录的隐式依赖。
cat plugin.json | grep -A20 skills📋
影响分析
变更一个模块时，自动计算受影响的模块集合。架构图标注依赖方向和影响范围。
✅ 结果: 影响分析报告 — 直接受影响: 2 文件 (SKILL.md + 场景-2-数据流追踪.md) / 间接影响: 4 文件 (架构图.html 引用 + 知识图谱.json 边更新 + 审查.html + 测试面板.html) / 风险等级: 🟡 中等 (T2 级变更)。
💡 说明: --impact 模式在变更前模拟影响范围，输出受影响文件列表和风险等级。这支持「溯先于修」铁律 — 在动手修改前先了解影响面。黄色风险（T2）意味着需要裁剪分析但不需要完整重跑全管线。红色风险（T3/破坏性变更）需要全量重新验证。
/rui update user-login --impact📋
破坏性变更检测
检测接口签名变更、文件重命名/删除、配置格式变更。标记为 P0 并强制阻断。
✅ 结果: 破坏性变更检测扫描 42 个文件 — 接口签名: 0 变更 / 文件重命名: 0 检测 / 配置格式: 0 变更 / 导出函数: lib/tty.mjs 2 个导出均稳定 (formatBox + colorize 签名未变) → 结果: ✅ 无 P0 阻断。
💡 说明: 破坏性变更检测是自托管一致性的前置检查。lib/ 共享库的导出函数签名变更会影响所有引用方（从 grep "from.*lib/" 检索），属于 P0 级。--breaking-check 对比基线（init 时记录的签名快照）与当前状态，签名不匹配即阻断。
/rui doc --from-code --breaking-check📋
自托管一致性验证
验证 plugin.json 版本号与 skills/agents/rules 实际内容一致。不一致时标记 P0。
✅ 结果: audit.mjs 输出 — 版本一致性: plugin.json(4.3.0) = CLAUDE.md(4.3.0) = README.md(4.3.0) ✅ / 模块计数: 6 skills = 6 目录 ✅ / 9 agents = 9 .md 文件 ✅ / 8 rules = 8 .md 文件 ✅ / lib/ 引用: 4 共享模块无循环依赖 ✅ → 综合评分 100/100。
💡 说明: audit.mjs 是自托管一致性的自动化验证脚本。每次 init 或版本变更后必须运行，确保 plugin.json（注册表）与磁盘上的实际文件一致。版本号不一致是最常见的 P0 — 例如 plugin.json 升级到 4.4.0 但 CLAUDE.md 仍写 4.3.0，audit 会标记并阻断后续管线。
node skills/rui/audit.mjs📋
增量变更范围
T1 措辞/格式（跳过分析）、T2 增删/接口变更（裁剪分析）、T3 边界/重构（完整重跑）。
✅ 结果: T2 裁剪分析 — 变更范围: 2 文件 (SKILL.md + 场景文档) → 受影响模块: rui (直接) + rui-story (间接引用) → 裁剪后仅重跑这 2 个模块的分析和测试 → 耗时 ~3 分钟 (vs T3 全量 ~15 分钟)。
💡 说明: 三级变更分级避免「改一个字跑全部」。T1 (措辞) 只做 diff review，T2 (增删) 裁剪分析受影响模块，T3 (重构) 完整重跑。--no-code 表示只更新文档不变更代码，进一步缩小范围。分级的关键是精确判断变更类型 — 低估会导致遗漏，高估会浪费资源。
/rui update user-login --no-code --tier T2📋
知识图谱依赖追踪
知识图谱以 JSON 格式记录节点间边关系。变更影响可沿边追溯，量化影响范围。
✅ 结果: 知识图谱渲染 — 23 个节点 (6 skills + 9 agents + 8 rules) / 56 条边 (depends_on 14 / invokes 18 / governed_by 16 / generates 8) → 选定 rui-npm 节点: 入度 0 (独立技能) 出度 2 (引用 lib/tty.mjs + lib/constants.mjs)。
💡 说明: 知识图谱是依赖追踪的可视化载体。JSON 格式存储节点和边，支持沿边追溯影响范围 — 选中一个节点，所有通过 depends_on 边可达的节点都是受影响范围。图谱与架构图互补 — 架构图展示静态结构，知识图谱展示动态关系。变更一个模块后必须更新图谱的边关系。
cat 知识图谱.html📋
