⚡ ⚡ 场景-4-依赖变更影响 · 交互示例

依赖变更时的全局影响分析与修复路径

🚶 操作步骤

检测变更
识别变更文件的类型（T1/T2/T3）和影响面
git diff --name-only HEAD~1📋
追踪影响链
从变更点出发，追踪所有直接和间接依赖
grep -r "changed-module" --include="*.md"📋
级联更新
按优先级刷新受影响文件，确保引用一致
/rui update 📋
回归验证
运行全量自检，确认变更不引入回归
node tests/run.mjs📋

💡 预期输出

变更类型: T1 (措辞修正)
影响面: 3 文件直接 + 7 文件间接
级联更新: 10 文件已刷新
