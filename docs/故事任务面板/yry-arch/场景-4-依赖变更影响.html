场景 4: 依赖变更影响 — 从变更点到完整影响链

变更点定位 · 依赖链追踪 · 管线阶段影响 · 约束冲突检测 · 可操作更新清单

五种变更类型

  • 能力变更 — Skill 接口/行为修改
  • 角色变更 — Agent 职责/契约调整
  • 约束变更 — Rule 适用范围/条件变更
  • 阶段变更 — Pipeline 阶段增删改
  • 交叉变更 — 多种变更同时触发

约束冲突等级

  • 阻断 P0: 不可继续，需修复
  • 降级 P1: 记录不阻断，需关注
  • 无冲突: 变更不触发任何约束
  • 每个冲突附: 触发条件 + 执行者 + 恢复方式

影响报告输出格式

  • 变更点定位 + 完整影响链
  • L1 直接消费者 + L2 二级传递
  • 受波及管线阶段列表
  • 触发约束 + 阻断标识清单
  • 可操作更新清单 (优先级 + 执行者)

闭合验证原则

  • 逐层追踪，不跳级不猜测
  • Grep 验证声明关系与实际引用
  • 隐式依赖标注 '待确认'
  • 循环依赖显式标注位置
  • 项目边界外标注 '边界外'
