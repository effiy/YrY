场景 3: 新人上手 — 从零到首次交付的 7 步路径

项目全景认知 · 管线流程理解 · 初始化验证 · 模块拓扑学习 · 数据流追踪 · 首次变更实践 · 交付收口

前置检查清单

  • Node.js >= 18 已安装
  • Git 已配置，有 GitHub 账号
  • Claude Code 已安装并可用
  • 项目已 clone 到本地
  • IDE 已配置（推荐 VS Code）

三个关键问题

  • Q1: 某个模块在哪？ → 模块目录
  • Q2: 它的上下游是谁？ → 拓扑导航图
  • Q3: 变更它会影响到谁？ → 依赖影响分析
  • 能回答这三个问题 = 上手完成

常见障碍与解决

  • /rui init 失败 → 检查 Node 版本、.claude/ 目录
  • Gate A 阻断 → 补充测试用例再提交
  • Gate B 阻断 → P0 未清零，逐模块修复
  • 分支错误 → 确认在 feat/<name> 分支

领域语言速查

  • 故事 Story — 一个独立的业务需求单元
  • 场景 Scene — 故事的交互切片
  • P0 清零 — 最高优先级缺陷数归零
  • Gate A/B — 管线中两个强制门禁
  • feat/<name> — 功能分支命名规范
