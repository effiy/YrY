场景 2: 数据流追踪 — 从指令到交付的 11 阶段管线

管线阶段编目 · 数据流序列 · 门禁矩阵 · 角色参与矩阵 · 交付收口链路

正常路径 (Happy Path)

  • 指令 → 解析 → 规划 → 分析 → 设计
  • 文档 → 分支 → Gate A 通过
  • 逐模块实现 → P0 清零
  • Gate B 通过 → 复盘 → 交付
  • 三步 Hook: 日志 → 同步 → 通知

阻断路径 (Block Conditions)

  • Gate A 阻断: 测试用例不达标
  • Gate B 阻断: 测试未通过/P0 未清零
  • no-branch-isolation: 未切 feat 分支
  • chain-broken: 影响链未闭合
  • auto-merge: 非法自动合并

关键产出物

  • 01-需求解析 → 故事定义
  • 02-自适应规划 → 执行路径
  • 03-影响分析 → 变更影响报告
  • 04-架构设计 → 技术方案文档
  • 05-09 → 故事/场景/测试/实现文档

协作角色映射

  • pm: 阶段 1-3, 5 (必须)
  • architect: 阶段 3-4 (必须)
  • coder: 阶段 5-6, 8 (必须)
  • tester: 阶段 7, 9 (必须)
  • self-improve: 阶段 10 (必须)
