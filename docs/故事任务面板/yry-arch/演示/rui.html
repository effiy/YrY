初始化项目基线
detect → explore → generate → arch → setup → verify，六步建立项目骨架。CLAUDE.md/README.md 全量重生，架构故事和自检方案自动生成。
/rui init📋
从需求生成文档基线
pm 拆需求为故事 → coder 补齐设计文档 → security 独立审计。产出：故事任务.md + 场景-N-<slug>.md + 知识图谱.json。
/rui doc 用户需要登录功能📋
从源码反推文档
存量代码库生成故事文档基线。全程只读源码，证据 Level B + 源码路径，缺口标「待补充」。
/rui doc --from-code📋
实现故事（源码改动唯一入口）
分支隔离门禁 → Gate A 测试先行 → 逐模块 P0 清零 → Gate B 验证 ≤ 2 轮 → 自改进 D0-D7 → 交付。
/rui code user-login📋
增量更新
T1 措辞/格式（跳过分析）、T2 增删/接口变更（裁剪分析）、T3 边界/重构（完整重跑）。
/rui update user-login --no-code📋
全自主自改进闭环
扫描所有故事 → D0-D7 诊断 → 选取最优改进项 → 实现 → 验证 → 版本升级 → 循环至收敛。
/rui yry --depth 3📋
版本升级与发布
自主判定下一版本号 → 更新 plugin.json/CLAUDE.md/README.md → git commit → merge main → push + tag。
/rui version --up📋

阶段: 初始化
/rui init
detect→explore→generate→arch→setup→verify。可重复运行，每次全量重生 rui 标记段。
/rui init📋
阶段: 文档
/rui doc 需求
拆需求为故事 + 生成文档基线。全程只读源码，多故事串行。
/rui doc 用户登录功能📋
阶段: 实现
/rui code name
Gate A → 逐模块 P0 清零 → Gate B ≤ 2 轮 → 自改进 → 交付。
/rui code user-login📋
阶段: 更新
/rui update name
T1/T2/T3 自动裁剪。--no-code 仅文档不改源码。
/rui update user-login --no-code📋
阶段: 自改进
/rui yry --depth N
全自主闭环：扫描→诊断→实现→验证→版本升级，循环至收敛。
/rui yry --depth 3📋
阶段: 交付
/rui version --up
自主判定版本号→更新文件→commit→merge→push+tag。
/rui version --up📋
阶段: 回退
/rui version --rollback
基于 git 版本链回退故事文档到指定历史版本。
/rui version --rollback user-login --to v1.2.0📋
阶段: 只读反推
/rui code --from-doc
从已有文档反推，只读源码补全缺失章节，不覆盖已有。
/rui code --from-doc user-login📋
阶段: 文档反推
/rui doc --from-code
req 空时 pm 扫描推荐列表；req 有值时直接反推全文档基线。
/rui doc --from-code 用户登录📋
阶段: 本地补全
/rui doc --from-local
从已有本地文档补全缺失基线，只读已有，不覆盖。
/rui doc --from-local user-login📋
阶段: 端到端
/rui 需求
= /rui doc 需求 → /rui code name，无中断一气呵成。
/rui 添加用户登录功能📋
阶段: 只读
/rui
任务推荐：5 层链式管线评分排序，输出推荐列表。
/rui📋
