风险与缓解

#风险可能性影响缓解策略
R1模块间隐性依赖未被发现导致关系图不完整 MH Grep 全局搜索 + 源码交叉验证 + 多 Agent 独立绘制后合并
R2知识图谱节点膨胀导致可读性下降 MM 三层层次（domain/flow/step）控制粒度，每层独立可折叠
R3架构文档与源码不同步（渐进漂移） HH 每个断言附源码路径 + git hash；自改进阶段定时刷新
R4场景覆盖不全导致某些架构维度的参照缺失 MM 场景选择基于系统六大能力模块 + 八种角色 + 八组约束的全交叉矩阵

执行模式

Subagent-Driven（推荐）

独立 subagent 逐任务执行，每任务完成后触发 code-reviewer 审查。P0 清零再进下一任务。

适用：> 5 步骤 · 多文件并行

Inline 执行

当前会话逐步骤执行。每步验证后前进。

适用：≤ 5 步骤 · 简单场景
