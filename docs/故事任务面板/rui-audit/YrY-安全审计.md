<a id="sec2-threats"></a>
## §2 威胁建模

| # | 威胁 | 攻击面 | 可能性 | 影响 | STRIDE 分类 |
|---|------|--------|:--:|:--:|-----------|
| T1 | 伪造审计记录绕过越权检测 | CLI → 审计文件 | M | H | 伪装 (Spoofing) |
| T2 | AGENT_TOOLS 表被错误修改导致越权漏检 | 源码 → `AGENT_TOOLS` 对象 | L | H | 篡改 (Tampering) |
| T3 | 审计文件被删除销毁证据 | 文件系统 → 审计文件 | L | M | 否认 (Repudiation) |
| T4 | 审计记录中的 target 路径泄露内部结构 | 审计文件 → 外部读取者 | L | L | 信息泄露 (Information Disclosure) |
| T5 | 大量审计记录导致统计性能下降 | CLI → `readAuditRecords()` | L | L | 拒绝服务 (Denial of Service) |
| T6 | 未知 agent 利用警告机制绕过越权检测 | `check` → 未知 agent 仅警告 | M | H | 权限提升 (Elevation of Privilege) |

<a id="sec3-trust"></a>
## §3 信任边界

| 边界 | 跨越方向 | 数据流 | 校验点 | 当前状态 |
|------|---------|------|--------|:--:|
| CLI 参数 → audit 内部 | 入站 | process.argv → opts 对象 | 必填参数检查 + 命令白名单 | 已加固 |
| 审计文件 → audit 内部 | 入站 | JSONL → readAuditRecords → 记录数组 | 逐行 JSON.parse 容错 | 已加固 |
| audit 内部 → 审计文件 | 出站 | 记录对象 → JSON.stringify → 追加 | 结构化固定字段 | 已加固 |
| AGENT_TOOLS → check | 配置 | 源码硬编码 → Set.has() | Set 容器，不可运行时修改 | 已加固 |
| 审计文件 → 外部 | 出站 | 文件系统 → 任意进程 | 无（依赖 OS 权限） | 未加固 |
