语义化版本决策指南

🔧 PATCH — Bug 修复
修正 .card 阴影偏移、修复 toast 定时器竞态、修正格式化函数边界值。消费者自动获取。
例: 1.0.0 → 1.0.1 · npm ^1.0.0 自动包含
✨ MINOR — 新功能
新增 YrY.smoothScroll() API、新增 .modal 组件、新增 @keyframe 动画。向后兼容。
例: 1.0.0 → 1.1.0 · npm ^1.0.0 自动包含
💥 MAJOR — 不兼容变更
重命名 .card → .yry-card、移除废弃 API、改设计令牌命名。消费者需手动升级。
例: 1.0.0 → 2.0.0 · npm ^1.0.0 不包含

DevTools 5 步验证发布流程

查看 package.json 配置
检查 cdn/package.json 的 name/version/files 字段，理解 npm 包元数据和发布清单。
cat cdn/package.json | python3 -m json.tool📋
files 字段限制发布内容，确保 tests/、docs/ 等开发文件不会泄漏到 npm registry。
理解语义化版本决策树
学习 PATCH/MINOR/MAJOR 三级版本号的选择规则和影响范围。
PATCH: fix .card shadow → 1.0.0→1.0.1
MINOR: add YrY.smoothScroll → 1.0.0→1.1.0
MAJOR: rename .card→.yry-card → 1.0.0→2.0.0📋
语义化版本 (semver.org) 是 npm 生态的基础约定。^1.0.0 表示接受 1.x.x 范围内所有 PATCH+MINOR 更新。
模拟 npm publish 流程
执行 npm version → npm publish → git push --tags 完整的发布三步曲。
npm version patch                    # 1.0.0→1.0.1
npm publish --access public          # 推送 5 文件
git push origin main --tags          # 同步 tag📋
npm version 命令一步完成版本号更新→commit→tag 三个操作，确保版本号与 git tag 始终一致。
验证发布结果
使用 npm info 查看已发布包的元数据、版本列表和依赖信息。
npm info cdn version             # 查看最新版本
npm info cdn versions --json     # 全部版本列表📋
Git Tag 版本追溯
使用 git tag 查看版本标签链，git show 重现任意历史版本的完整代码。
git tag -l 'v*' --sort=-version:refname
git show v1.0.0 --stat📋
git tag 是"可回退"的保障——发现 v1.0.1 引入回归 bug 时，可以 git checkout v1.0.0 快速回退。
