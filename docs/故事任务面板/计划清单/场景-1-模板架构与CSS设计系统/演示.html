P0 必做FP1骨架
💡 说明: 7 区域是 FP1 的硬性要求 — 任何区域缺失都会触发 R1 业务规则失败。区域用注释明确标注语义,开发者接手时可快速理解结构。Token 占位符（{ {STORY_NAME} } 等）按需嵌入到具体区域中，由渲染层负责替换。
✅ 结果: 7 区域骨架 HTML 文件已就绪 — 面包屑 (1) · 场景导航 (2) · 跨文档导航 (3) · Header (4) · 统计卡片 (5) · Tab 系统 (6) · 7 个内容面板 (7)。每个区域有 HTML 注释说明语义和预期 Token。AC1 通过验证：开发者打开模板即可看到清晰的 7 区域结构。
设计 3 层 CSS 变量 Token ~20 min
建立 primitive → semantic → component 3 层 Token 体系，所有自定义属性以 --yry-* 前缀命名。Component 层级变量绑定到具体组件 class，semantic 层级绑定到设计语义，primitive 层级定义原始值。
@layer tokens {
  :root {
    /* primitive */
    --yry-color-accent: #ffc107;
    --yry-color-cyan: #22d3ee;
    /* semantic */
    --yry-color-text-1: #f5f5f5;
    --yry-color-bg: #0a0a0a;
    /* component */
    --yry-radius-md: 6px;
    --yry-space-4: 16px;
  }
}📋
P0 必做FP2设计系统
💡 说明: 3 层 Token 体系是 FP2 的核心 — 任意一层缺失都破坏可维护性。primitive 改变 (如换主色) → semantic 自动跟随 → component 同步更新。命名空间 --yry-* 前缀避免与宿主项目 CSS 冲突。light-mode 通过 @media (prefers-color-scheme: light) 覆盖 semantic 层变量实现，零主题切换 JS。
✅ 结果: 3 层 Token 体系建立完成 — primitive (15 项: 5 品牌色 + 字体 + 间距) · semantic (12 项: text1-3 + surface1-2 + border + 状态色) · component (8 项: 半径 + 阴影 + 动效曲线)。R2 验证通过：所有变量在 :root 内定义。AC2 通过：替换 Token 后页面正确展示新场景标题/版本/日期。
配置 CDN 加载链 ~10 min
在 <head> 中按正确顺序加载 4 个 CDN 资源：shared.css → theme.css → yry-checklist.css → shared.js。路径基于 { {CDN_DEPTH} } 计算相对路径，禁止硬编码层级数。
<head>
  <link rel="stylesheet" href="{ {CDN_DEPTH} }/shared.css">
  <link rel="stylesheet" href="{ {CDN_DEPTH} }/theme.css">
  <link rel="stylesheet" href="{ {CDN_DEPTH} }/yry-checklist.css">
</head>
<body>
  ...
  <script src="{ {CDN_DEPTH} }/shared.js"></script>
</body>📋
P0 必做FP3加载顺序
💡 说明: 加载顺序不可错乱 — yry-checklist.css 依赖 theme.css 的 CSS 变量和 shared.css 的基础 reset。shared.js 必须放在 body 末尾以确保 DOM 元素就绪。CDN 深度通过场景目录的相对路径计算 (docs/故事任务面板/计划清单/场景-1-.../ → 4 层)，由渲染层自动注入。SC3 验证要求浏览器 Network 面板 0 个 404。
✅ 结果: CDN 加载链配置完成 — 4 资源 (shared.css 6.0KB · theme.css 12.1KB · yry-checklist.css 4.5KB · shared.js 5.3KB) 全部 HTTP 200。R3 验证：路径中无硬编码 ../，深度由 { {CDN_DEPTH} } 动态计算。AC4 验证：刷新页面后 CDN 更新自动生效。
实现核心组件 CSS 类 ~25 min
定义 7 个核心组件类的完整样式：step · phase-strip · scorecard · risk-matrix · gantt · trend · checklist-head。每类含基础样式 + 响应式断点 (≤900px / ≤700px / ≤500px) + 可选状态 (pass/fail/warn)。
/* 组件 CSS — 内联或通过 CDN */
.step { display:flex; gap:14px; padding:16px 0; }
.phase-strip { display:flex; gap:6px; }
.scorecard { width:100%; border-collapse:collapse; }
.risk-matrix { display:grid; grid-template:repeat(5,1fr)/repeat(5,1fr); }
/* ... */📋
P0 必做FP2组件库
💡 说明: 7 个组件类覆盖计划清单页面的全部核心交互。样式优先放置在 CDN 资源中以便跨页面复用，页面专属样式限定在 .container 作用域内避免意外覆盖。状态颜色复用 semantic 层 Token (--yry-color-pass/fail/warn)。SC2 验证：模板使用的所有 CSS 类在 CDN 样式表中都有定义，覆盖度 100%。
✅ 结果: 7 个核心组件类全部实现 — step (步骤行) · phase-strip (阶段带) · scorecard (分数卡) · risk-matrix (5×5 风险热力图) · gantt (甘特条) · trend (趋势柱状图) · checklist-head (清单头)。SC2 通过：覆盖度 14/14 = 100%。R1 验证：template 中 class="panel" 数量 = 7。
实现 Token 替换机制 ~15 min
在 HTML 模板中嵌入 { {STORY_NAME} }、{ {SCENE_TITLE} }、{ {VERSION} }、{ {DATE} }、{ {CDN_DEPTH} } 等 20 个 Token。渲染层在生成 HTML 时按 Token 表替换，替换后页面不得残留 { {…} } 占位符。
// 渲染层伪代码
const tokens = {
  STORY_NAME: 'yry-checklist',
  SCENE_TITLE: '场景 1 · 模板架构与 CSS 设计系统',
  VERSION: 'v1.0.0',
  DATE: '2026-06-13',
  CDN_DEPTH: '../../../../'
};
Object.keys(tokens).forEach(k => {
  html = html.replaceAll(`{{${k}}}`, tokens[k]);
});📋
P0 必做FP4模板引擎
💡 说明: Token 系统是 FP4 的核心 — 替换后的页面必须 grep `{{` 零匹配 (TC-4 验证)。20 个 Token 按功能分为 3 类：① 标识类 (STORY_NAME · SCENE_TITLE · SCENE_ID) ② 时间类 (VERSION · DATE) ③ 路径类 (CDN_DEPTH · 各类链接前缀)。替换失败时打印警告并降级为原始占位符字符串，便于调试。
✅ 结果: 20 个 Token 全部替换完成 — 标识类 8 个 · 时间类 4 个 · 路径类 8 个。TC-4 验证通过：grep `{{` 匹配数 = 0。R4 通过：渲染后无残留占位符。AC2 通过：开发者替换 Token 即得新场景页面。

运行 TC-1 至 TC-5 全部测试用例：骨架完整性 · CSS 变量覆盖 · CDN 可达性 · Token 无残留 · 3 个响应式断点。验证 prefers-reduced-motion 和 Skip Link 等 a11y 特性。

P0 必做验证a11y
💡 说明: 验证是模板上线前的最后一道关卡。响应式 3 断点确保从桌面 (1100px) 到移动端 (≤500px) 都有合适布局 — 统计卡片从 6 列降到 3 列再降到 2 列。a11y 动效通过 prefers-reduced-motion 媒体查询禁用动画，照顾对动画敏感的用户。Skip Link 让键盘用户可跳过导航直达主内容。
✅ 结果: 5 项验证全部通过 — TC-1 骨架 7/7 · TC-2 CSS 14/14 · TC-3 CDN 4/4 · TC-4 Token 20/20 · TC-5 响应式 3/3。AC3 验证：浏览器宽度调整时统计卡片从 6 列 → 3 列 → 2 列正确重排。a11y 通过：prefers-reduced-motion 媒体查询生效，Skip Link 可达。

计划清单页面的 HTML 骨架必须包含几个区域？
A3 个
B5 个
C7 个
D12 个
正确答案: C 计划清单页面有 7 大 HTML 区域：① 面包屑 ② 场景导航 ③ 跨文档导航 ④ Header ⑤ 统计卡片 ⑥ Tab 系统 ⑦ 7 个内容面板。R1 业务规则要求任何区域缺失都视为骨架不完整（TC-1 验证）。
2
CSS 变量推荐使用什么命名空间前缀？
A无前缀
B--yry-*
C--css-*
D--app-*
正确答案: B 所有自定义 CSS 变量必须使用 --yry-* 命名空间前缀，避免与宿主项目 CSS 变量冲突。命名空间约定是 FP2 的一部分，FP2 还要求 3 层 Token 体系 (primitive → semantic → component) 清晰分层。
3
CDN 资源加载的正确顺序是？
Atheme.css → shared.css → yry-checklist.css → shared.js
Bshared.css → theme.css → yry-checklist.css → shared.js
Cshared.js → shared.css → theme.css → yry-checklist.css
Dyry-checklist.css → theme.css → shared.css → shared.js
正确答案: B 正确顺序是 shared.css → theme.css → yry-checklist.css → shared.js。shared.css 提供基础 reset，theme.css 定义 CSS 变量，yry-checklist.css 依赖前两者的变量定义组件样式。shared.js 放在 body 末尾以确保 DOM 元素就绪。R3 要求路径不硬编码 ../，由 { {CDN_DEPTH} } 动态计算。
3
Token 替换后页面不得残留什么？
ACDN 引用
BHTML 注释
C{{...}} 占位符
DCSS 类名
正确答案: C Token 替换后页面不得残留 {{...}} 占位符（R4 业务规则 + TC-4 验证）。20 个 Token 全部成功替换为实际值。grep `{{` 匹配数必须为 0。替换失败的 Token 应降级为原始占位符字符串或合理默认值，便于调试。

A≤1200px / ≤900px / ≤600px

模板架构常见踩坑

⚠ 6 大类 🔧 含修复方案
📁 P0: HTML 骨架缺失区域 HIGH

7 大区域（面包屑·场景导航·跨文档导航·Header·统计·Tab·7 面板）任一缺失即视为骨架不完整。R1 业务规则要求 class="panel" 数量 = 7，TC-1 验证会发现缺失即失败。

症状: /rui-html verify 报「Panel count mismatch: 6 != 7」。

修复: 1) 逐区域检查模板 HTML 2) 确认 7 个 panel 都存在 3) 缺失区域从其他场景的模板复制补全
🎨 P0: CDN 加载顺序错误 HIGH

CDN 资源加载顺序错乱会导致 yry-checklist.css 的组件样式失效（变量未定义）。正确顺序是 shared.css → theme.css → yry-checklist.css → shared.js。

症状: 组件颜色显示为默认黑白色，无 CSS 变量注入；Network 面板资源顺序与预期不符。

修复: 检查 <head> 中 link 标签顺序，将 shared.css 放在第一位，yry-checklist.css 放在 theme.css 之后。运行 /rui-html verify 验证加载顺序。
🔤 P1: 硬编码色值而非 Token MED

组件 CSS 中直接写 #ffc107 等原始色值，绕过 --yry-* 命名空间。导致主色变更时需逐文件改色，破坏可维护性。R2 业务规则要求所有变量在 :root 或 .theme-system 内定义。

症状: 主题切换无效，部分组件颜色不变；/rui-html lint --strict 报「Hardcoded color value」。

修复: 将硬编码色值替换为 var(--yry-color-*) 引用。primitive 层 (--yry-color-accent) 在 :root 定义，semantic 层 (--yry-color-text-1) 在 :root 引用 primitive。
🔄 P1: Token 替换残留占位符 MED

渲染层在替换 Token 时遗漏某些 Token（如新增 Token 未注册到替换表），导致页面出现 { {STORY_NAME} } 原始字符串。R4 业务规则要求 grep `{{` 0 匹配。

症状: 页面打开后看到 { {SCENE_TITLE} } 等原始占位符字符串；TC-4 验证 grep `{{` 匹配数 > 0。

修复: 1) 在替换表中添加缺失的 Token 2) 替换失败时降级为占位符字符串 + 警告日志 3) /rui-html verify 检查残留
📱 P1: 响应式断点不完整 MED

只实现 ≤900px 一个断点，遗漏 ≤700px 和 ≤500px。在小平板和手机上布局错乱，统计卡片 6 列直接挤压。TC-5 验证需要 3 个断点全部通过。

症状: 手机端文字溢出、按钮重叠、统计卡片水平滚动；浏览器宽度调整时布局不重排。

修复: 添加 3 个断点的媒体查询 — ≤900px 统计卡片 3 列、≤700px 隐藏次要元素、≤500px 2 列重排。使用 grid + auto-fit 自适应。
♿ P2: 缺少 a11y 动效克制 LOW

未实现 prefers-reduced-motion 媒体查询，对前庭功能障碍用户不友好。最佳实践是默认动画 ≤200ms，匹配用户偏好时完全禁用动画。

症状: macOS「减少动效」开启后页面仍有动画；屏幕阅读器跳过动画时内容布局抖动。

修复: 添加 @media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: .01ms !important; transition-duration: .01ms !important; } }。同时考虑添加 Skip Link 让键盘用户可达主内容。
⚠ 严重程度说明: HIGH = 触发 P0 阻断（骨架·CDN），需立即修复 · MED = 影响可维护性或响应式但不阻断 · LOW = 反模式（a11y 动效）但可后续改进。修复 HIGH 级错误后再继续后续步骤。
