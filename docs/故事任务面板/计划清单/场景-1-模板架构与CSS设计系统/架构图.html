加载顺序

铁律
  • CSS 1 shared.css — 浏览器 reset + 基础排版
  • CSS 2 theme.css — 主题变量(:root 自定义属性)
  • CSS 3 yry-checklist.css — 组件类(可被页面覆盖)
  • JS shared.js defer — 交互绑定(DOMContentLoaded 后)
顺序敏感不可并联

路径策略

CDN_DEPTH
  • 相对路径基于场景目录深度计算
  • 深度 4: ../../../../cdn/...
  • 深度 5: ../../../../../cdn/...
  • 禁止硬编码具体层数
Token 化不硬编码

降级策略

失败保护
  • CDN 不可达时回退到内联 <style>
  • JS 失败时页面仍可读(渐进增强)
  • 页面专属样式接管组件默认
可降级零阻塞

SC1 · 7 区域骨架完整 FP1 → AC1
度量:逐区域检查模板 HTML
目标:7/7 全部存在(CDN 头 · 面包屑 · 场景导航 · 统计卡 · 标签页 · 7 面板 · 脚本)。任一区域缺失即不通过 P0 阻断。grep -c 'class="panel' 计划清单.html ≥ 7。
SC2 · CSS 类覆盖 100% FP2 → AC2
度量:模板用类 vs CDN 选择器交叉比对
目标:覆盖度 100%。步骤 grep -oE 'class="[^"]+"' 计划清单.html | sort -u 与 cdn/yry-checklist.css 选择器集合做差集,差异 ≤ 0。页面专属类 允许独立定义,但需限定在 .container 作用域内。
SC3 · CDN 资源 200 OK FP3 → AC3
度量:浏览器 DevTools Network 面板
目标:全部 4 资源 HTTP 200。顺序为 shared.css → theme.css → yry-checklist.css → shared.js。加载顺序错误会导致组件类覆盖主题变量。自动化测试: curl -I -o /dev/null -w "%{http_code}" 4 资源全 200。
SC4 · Token 替换无残留 FP4 → AC4
度量:grep 检查占位符
目标:0 残留。命令 grep -c '{{' 渲染结果.html 必须为 0。Token 含 {{STORY_NAME}} · {{SCENE_TITLE}} · {{VERSION}} · {{DATE}} · {{CDN_DEPTH}} · {{SCENE_NUM}} 6 类,任一未替换即失败。
AC3 · 响应式断点 3 档
度量:调整窗口大小观察栅格重排
目标:3 断点自适应。≤ 900px 统计卡 6→3 列 · ≤ 700px 标签页横向滚动 · ≤ 500px 统计卡 3→2 列 · 暗色/亮色基于 prefers-color-scheme 自动切换。
AC4 · 主题切换无错 持续
度量:切换主题观察样式
目标:暗色/亮色切换无样式跳变。所有 token 必须基于 CSS 自定义属性,不允许硬编码 #xxx 色值于组件类中。新增主题通过追加 @media (prefers-color-scheme: ...) 块实现,不改组件定义。

验证命令

一键
  • 骨架: grep -c 'class="panel' 计划清单.html ≥ 7
  • CSS 覆盖: diff <(grep -oE '\.[a-z][a-z0-9-]+' 模板) <(grep -oE '^\.[a-z][a-z0-9-]+' cdn/*.css)
  • CDN: curl -sI -o /dev/null -w "%{http_code}\n" 4 资源
  • Token: grep -c '{{' 渲染结果.html = 0

常见失败

避坑
  • ❌ SC1 失败: div 缺 class · 或 7 面板被合并为 1
  • ❌ SC2 失败: 模板用 badge-warn 而 CDN 是 chip.warn
  • ❌ SC3 失败: 路径错(深度 5 写成了 4),404 报错
  • ❌ SC4 失败: 漏替换 {{SCENE_NUM}} · 残留 {{SCENE_NUM}}

下游约束

传递

  • 本场景的 Token 列表是后续场景的 最小集,可扩展不可删减
