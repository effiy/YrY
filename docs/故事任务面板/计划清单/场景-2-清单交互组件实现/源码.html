清单交互常见误区 避坑指南

⚠️
不要跳过 Data 层直接改 Renderer
清单交互的核心是 Data→Template→Renderer 三层协作。先理解 localStorage Schema（Data 层），再理解 HTML data-* 钩子（Template 层），最后才看 DOM 更新逻辑（Renderer 层）。
🚫
不要跨场景混用 localStorage key
localStorage key 含场景标识 s2（yry-arch-s2-checklist），避免与场景 1/3/4 的状态冲突。修改 key 时必须同步更新所有读写点。
✅
利用步骤状态自动推导
步骤状态（已完成✅/就绪🔶/待开始⏳）根据前置步骤完成情况自动推导，无需手动标记。如果状态显示异常，检查前置步骤的 checkbox 状态是否正确。

排错速查 清单交互高频问题

🔧
勾选后进度条不更新
症状：点击 checkbox 但进度条百分比不变。检查 updateProgress() 是否被正确绑定到 change 事件。
document.querySelectorAll('.chk-input').forEach(el=>console.log(el.onchange))
⏱
localStorage 状态未持久化
症状：刷新后勾选状态丢失。检查 localStorage key 是否与代码中一致（yry-arch-s2-checklist），以及是否在隐私模式下运行。
console.log(Object.keys(localStorage).filter(k=>k.includes('s2')))
⚠️
折叠面板动画卡顿
症状：点击交付物头部展开/收起时 CSS transition 不流畅。检查 max-height transition 是否设置了合理的时间（300ms ease-out）。
getComputedStyle(el).transition // 应包含 max-height 0.3s
📑
键盘快捷键 1-9 不生效
症状：按数字键无法切换标签页。检查当前焦点是否在 input/textarea 元素上——快捷键应跳过输入聚焦状态。
document.activeElement.tagName // 应为 BODY 而非 INPUT
📦
交付物类型过滤无效
症状：点击过滤按钮后列表未变化。检查 .deliv-link 元素是否已通过自动标注获得正确的 data-type 属性。
document.querySelectorAll('.deliv-link:not([data-type])') // 应返回空 NodeList
🔄
步骤状态推导错误
症状：步骤显示"就绪"但前置步骤未全部完成，或显示"待开始"但前置已全完成。检查 updateStepStatus() 中的前置步骤完成判断逻辑。
document.querySelectorAll('.step-row').forEach(el=>console.log(el.dataset.step,el.querySelector('.step-status')?.textContent))

关键术语对照 清单交互领域语言

  • → Data 层 — 三层架构最底层，负责 localStorage 读写（yry-arch-s2-checklist / yry-arch-s2-checklist-tab）
  • → Template 层 — 三层架构中间层，HTML data-* 属性作为 JS 钩子，连接 Data 与 Renderer
  • → Renderer 层 — 三层架构最顶层，内联 JS 读取 Data 层状态后更新 DOM（进度条/步骤状态/面板显隐）
  • → 步骤状态推导 — 根据前置步骤完成情况自动推导：已完成(✅ 当前步骤已勾选)、就绪(🔶 前置已全完成)、待开始(⏳ 前置未完成)
  • → 进度条双向同步 — checkbox 勾选 → updateProgress() → 进度条 width + overview 统计 + localStorage 三项同步更新
  • → 交付物自动标注 — 页面加载时扫描所有 .deliv-link，根据文件扩展名自动标注 data-type，无需手动维护
  • → 哈希定位 — 支持从交付物 tab 跳转到清单 tab 的特定步骤（#step-N-deliv），自动展开并高亮目标行
  • → 键盘劫持防护 — 数字键 1-9 切换标签页时，跳过 input/textarea 聚焦状态，避免误触发
