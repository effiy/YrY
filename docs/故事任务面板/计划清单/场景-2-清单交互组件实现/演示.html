A第 2 个步骤
B场景 2 标识，避免跨场景冲突
CSchema 版本号
D存储大小限制 2KB
正确答案: B s2 = scene-2，含场景标识避免跨场景 localStorage 冲突。场景 1 用 s1，场景 3 用 s3。每个场景独立管理自己的持久化状态。
3
步骤状态推导的三个状态是？
A未开始/进行中/已完成
B已完成(✅)/就绪(🔶)/待开始(⏳)
CP0/P1/P2
D待审核/已通过/已驳回
正确答案: B 已完成(✅ 当前步骤已勾选)、就绪(🔶 前置步骤已全部完成，当前可开始)、待开始(⏳ 前置步骤未完成，当前不可操作)。状态由 deriveStatus() 根据前置步骤完成情况自动推导，无需手动标记。
4
Data/Template/Renderer 三层架构中，Renderer 层的职责是？
A管理 localStorage 读写
B定义 data-* DOM 钩子
C视觉更新：CSS class 切换 + 动画 + 焦点环
D处理用户事件分发
正确答案: C 三层职责分明 — Data 层：状态 + localStorage；Template 层：data-* 钩子 + ARIA；Renderer 层：classList.toggle + CSS transition + 焦点环 2px。三层通过事件总线通信，不直接互相调用。

常见踩坑

⚠ 6 大类 🔧 含修复方案
🔑 P0: localStorage key 未含场景标识导致跨场景冲突 HIGH

直接使用 checklist 作为 key 而非 yry-arch-s2-checklist，导致场景 1、场景 2、场景 3 的勾选状态互相覆盖。

症状: 在场景 1 页面勾选步骤后，打开场景 2 页面发现步骤也被勾选了。

修复: 1) 统一使用 yry-arch-{scene}-{purpose} 命名规范 2) 在每个场景的 key 中加入场景标识 s2 3) 在 index.md 的 localStorage Schema 章节记录所有 key
🎨 P0: CSS class 命名不一致导致动画失效 HIGH

JS 使用 classList.toggle('collapsed') 但 CSS 定义了 .collapse，class 名不匹配导致折叠面板无动画。

症状: 点击折叠按钮无反应；DevTools 显示 class 已添加但元素未隐藏。

修复: 1) 统一 4 个 CSS class 命名: collapsed / open / dim / on 2) 在事件契约中明确列出每个组件使用的 class 名 3) TC-11 折叠测试会捕获此类不匹配
⌨️ P1: 键盘快捷键劫持 input 输入 MED

keydown 事件处理中未检查 e.target.tagName，导致用户在 input 中按数字键时触发标签页切换而非正常输入。

症状: 在搜索框中输入数字时页面突然切换到其他标签页。

修复: 在 keydown handler 开头加入 if (e.target.tagName === 'INPUT' || e.target.tagName === 'TEXTAREA' || e.target.tagName === 'SELECT') return; 跳过表单元素。
💾 P1: localStorage 未 try/catch 导致隐私模式下崩溃 MED

直接调用 localStorage.setItem/getItem 未包裹 try/catch。浏览器隐私模式或存储满时抛出 QuotaExceededError 导致整个组件初始化失败。

症状: 隐私模式下页面 JS 报错，勾选功能完全不可用。

修复: 所有 localStorage 操作包裹 try { ... } catch(e) {} 。读操作失败返回默认空对象 {}，写操作失败静默降级为内存模式。
📐 P1: 跳过 Data 层直接操作 DOM 导致状态不一致 MED

在事件 handler 中直接修改 DOM (el.classList.toggle)，而不是先更新 Data 层状态再触发 Renderer 重绘。多次操作后 DOM 状态与 localStorage 不一致。

症状: 刷新页面后勾选状态与页面显示不匹配；进度条百分比与实际勾选数不同。

修复: 严格遵守 Data → Renderer 单向数据流：事件 → 更新状态 → 写 localStorage → 调用 render() 更新 DOM。永远不直接从事件 handler 修改 DOM。
🧪 P2: 未定义事件契约就写实现代码 LOW

直接开始写 addEventListener 代码，未先定义事件类型、CSS class 映射、回调签名。导致后续组件间事件不一致，重构成本高。

症状: 勾选组件用 change 事件，折叠组件用 click 事件，标签组件混用两种事件 — 维护困难。

修复: 先在 index.md §0 中定义事件契约表格：组件 → 事件类型 → CSS class → 回调签名。6 组件统一为 click (5 个) + keydown (1 个) 两种事件类型。
⚠ 严重程度说明: HIGH = 触发功能阻断，需立即修复 · MED = 影响用户体验但不阻断核心功能 · LOW = 反模式但可后续改进。修复 HIGH 级错误后再继续后续步骤。
