### 2.2 六维评估

| 维度 | 评级 | 证据 | 说明 |
|------|------|------|------|
| 耦合 | 低 | rui-claude → import-docs（单向委托），rui-story sync 同模式 | 委托解耦 |
| 稳定性 | 高 | API_X_TOKEN 缺失降级；网络超时不阻断；pull 失败退出码 0 | 无单点崩溃 |
| 扩展性 | 中 | 新增 pull 目录类型需加 `resolvePullFilter` 分支 + `toLocal` 函数 | 策略模式已预留扩展点 |
| 可测试性 | 高 | `resolvePullFilter` 为纯函数，输入路径+root → 输出策略或 null | 可独立单测 |
| 安全边界 | 强 | Token 仅环境变量；一级标签硬约束；kebab-case 校验 | 纵深防御 |
| 依赖方向 | 正确 | rui-claude → rui（管线委托）→ import-docs（同步委托） | 逐层委托不反向 |
