### 主要价值

- 🔒 源码扫描安全 — 路由扫描仅读取源码，不执行代码，防止注入
- 🛡️ 路径遍历防护 — 扫描器限定在项目目录内，禁止访问系统路径
- 📋 自动生成文档安全 — 自动生成的文档不含密钥/Token/敏感信息
- 🔍 独立审计 — security agent 独立检查扫描器和文档生成器的安全边界

<a id="sec1-assets"></a>

## §1 资产识别

### 1.1 数据资产

| 资产 | 敏感级别 | 存储位置 | 访问路径 |
|------|---------|---------|---------|
| 目标项目路由/页面结构 | 低 — 项目内部结构信息 | 目标项目源码 | discover.mjs 只读访问 |
| 自动生成的故事文档 | 低 — 规约化技术文档 | `docs/故事任务面板/<name>/` | discover.mjs 只写 |
| 发现报告 | 低 — 页面/路由清单 | yry 终端输出 | stdout |

> discover.mjs 的资产均为只读源码 + 只写文档，不涉及密钥、用户数据或生产环境访问。

### 1.2 功能资产

| 资产 | 认证要求 | 授权级别 |
|------|---------|---------|
| discover.mjs 执行 | 本地 shell 执行权限 | 项目开发者 |
| 目标项目源码读取 | 文件系统读取权限 | 项目开发者 |
| 故事文档写入 | `feat/<name>` 分支 | 分支隔离门禁 |

<a id="sec2-threats"></a>

## §2 威胁建模

| # | 威胁 | 攻击面 | 可能性 | 影响 | STRIDE 分类 |
|---|------|--------|--------|------|------------|
| T1 | 恶意路由文件包含路径遍历，诱导扫描器读取项目外系统文件 | 路由配置文件内容 | L | M | Information Disclosure (信息泄露) |
| T2 | 路由扫描时通过精心构造的文件名实现命令注入 | 路由文件路径 | L | H | Tampering (篡改) + Elevation of Privilege (权限提升) |
| T3 | 自动生成的页面/路由文档中包含从源码提取的敏感信息（如内网地址、测试凭据） | 目标项目源码 | M | H | Information Disclosure (信息泄露) |
| T4 | 攻击者通过篡改目标项目的路由文件，诱导 yry 创建恶意内容的故事文档 | 目标项目路由文件 | L | M | Tampering (篡改) |
| T5 | discover.mjs 被未授权调用，暴露项目路由结构给外部 | 本地 shell 访问 | L | L | Information Disclosure (信息泄露) |
| T6 | 自动创建大量故事目录导致磁盘占满（资源耗尽） | 超大项目（100+ 路由） | L | L | Denial of Service (拒绝服务) |

<a id="sec3-trust"></a>

## §3 信任边界

| 边界 | 跨越方向 | 数据流 | 校验点 | 当前状态 |
|------|---------|--------|--------|---------|
| 项目源码 → discover.mjs | 受信 → 受信 | 文件系统读取 | 路径限定在项目根目录内，禁止 `..` 跳出 | 待实施 |
| discover.mjs → 故事文档 | 受信 → 受信 | Write `docs/故事任务面板/` | 分支隔离 (`feat/<name>`)，内容为 markdown | 待实施 |
| 路由文件路径 → 文件系统 | 非受信 → 受信 | `fs.readFile` | 路径解析后校验在项目根目录内 | 待实施 |
| 文档内容 → 远端同步 | 受信 → 非受信 | POST 到 API | API_X_TOKEN, HTTPS | 已加固 |

<a id="sec4-mitigation"></a>

## §4 缓解措施

| 威胁# | 缓解措施 | 实现位置 | 优先级 | 状态 |
|-------|---------|---------|--------|:---:|
| T1 | 所有文件读取路径解析为绝对路径后，校验前缀等于项目根目录 | `discover.mjs` readFile 封装 | P0 | 待实施 |
| T2 | 路由文件名不作为 shell 参数，全部通过 Node.js fs API 读取，不经由 `exec`/`spawn` | `discover.mjs` 扫描逻辑 | P0 | 待实施 |
| T3 | 自动生成文档内容仅含项目结构信息（路由路径、组件文件、handler 文件），不含文件正文内容 | 文档生成逻辑 | P0 | 待实施 |
| T4 | 文档内容为 markdown 格式，不包含可执行脚本；写入前通过 branch-check.mjs 验证 | 管线门禁 | P1 | 待实施 |
| T5 | discover.mjs 仅通过 `/rui yry` 管线调用，不暴露为独立 CLI | SKILL.md 命令面 | P1 | 待实施 |
| T6 | 首次发现限制最多创建 50 个故事目录，超出则分批处理并提示 | `discover.mjs` 批量控制 | P1 | 待实施 |

## §5 合规检查
