<a id="sec1-assets"></a>

## §1 资产识别

### 1.1 数据资产

| 资产 | 敏感级别 | 存储位置 | 访问路径 |
|------|---------|---------|---------|
| 模块地图内容（四层表格 + 依赖矩阵） | 低 — 项目内部文档 | `docs/故事任务面板/module-map/技术评审.md` | git clone 即可读取 |
| 模块入口文件路径 | 低 — 公开的项目结构信息 | 同上 | 同上 |
| 模块依赖关系 | 低 — 可从源码 import 推导 | 同上 | 同上 |

> 模块地图为纯文档资产，不含密钥、Token、用户数据。敏感级别为低。

### 1.2 功能资产

| 资产 | 认证要求 | 授权级别 |
|------|---------|---------|
| 模块地图读写（本地） | git 仓库访问权限 | 仓库协作者 |
| 模块地图读写（远端） | `API_X_TOKEN` 环境变量 | API Token 持有者 |
| 漂移检测执行 | 本地文件系统读取权限 | 项目开发者 |

<a id="sec2-threats"></a>

## §2 威胁建模

| # | 威胁 | 攻击面 | 可能性 | 影响 | STRIDE 分类 |
|---|------|--------|--------|------|------------|
| T1 | 模块地图被恶意篡改，插入虚假模块条目或误导性依赖关系 | git 仓库（供应链/内部威胁） | L | M | Tampering (篡改) |
| T2 | 模块地图内容变更无法追溯到修改者和修改原因 | git 操作绕过规范（直接 commit 到 main 无 PR 审查） | M | L | Repudiation (否认) |
| T3 | 远端同步时模块地图内容被中间人截获或篡改 | api.effiy.cn 网络传输 | L | L | Information Disclosure (信息泄露) + Tampering |
| T4 | 模块地图中意外包含敏感信息（如内网地址、Token） | 人工编写时的疏忽 | L | H | Information Disclosure (信息泄露) |
| T5 | 漂移检测被绕过或结果被伪造，导致过时模块地图被视为准确 | 检测流程被跳过或手动标记为"已检查" | L | M | Tampering (篡改) |
| T6 | 模块地图作为知识资产被未授权的外部访问者获取 | 远端 API 未授权访问（如 Token 泄露） | L | L | Information Disclosure (信息泄露) |

<a id="sec3-trust"></a>

## §3 信任边界

| 边界 | 跨越方向 | 数据流 | 校验点 | 当前状态 |
|------|---------|--------|--------|---------|
| 开发者 → 本地文件系统 | 受信 → 受信 | git clone / git checkout | git 仓库访问权限 | 不适用（本地操作） |
| 本地文件系统 → 远端 API | 受信 → 非受信 | `rui-import sync.mjs` POST | `API_X_TOKEN` Header, HTTPS | 已加固 |
| 远端 API → 本地文件系统 | 非受信 → 受信 | `rui-import sync.mjs` (pull mode) | 内容为 markdown 文本，非可执行格式 | 已加固 |
| 文档编写 → git commit | 受信 → 受信 | `git add + git commit` | 分支隔离 (`feat/<name>`), commit message 规范 | 已加固 |
| 模块地图 → 自改进 Agent | 受信 → 受信 | markdown 解析 | 仅读取，不执行 | 已加固 |

<a id="sec5-compliance"></a>

## §5 合规检查

| # | 检查项 | 要求 | 当前状态 | 偏差说明 |
|---|--------|------|---------|---------|
| 1 | 认证不可绕过 | Token/认证链路不可跳步 | ✅ 通过 | `API_X_TOKEN` 环境变量，无硬编码 |
| 2 | 密钥不落盘 | Token/密钥/凭据禁止出现在源码或配置文件 | ✅ 通过 | 模块地图不含任何密钥 |
| 3 | 输入必校验 | 用户输入必须经过验证/转义 | ✅ 通过 | 模块地图为静态文档，不处理用户输入 |
| 4 | 最小权限 | 每个模块仅访问其需要的资源 | ✅ 通过 | 模块地图仅读取文件系统，不写入源码 |
| 5 | 默认拒绝 | 未明确授权的操作默认禁止 | ✅ 通过 | 漂移检测仅报告，不自动修改 |
| 6 | 审计日志完整 | 关键操作有可追溯的审计记录 | ✅ 通过 | git commit 链 + 变更记录表 |
