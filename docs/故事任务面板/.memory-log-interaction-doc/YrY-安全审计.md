## §1 资产识别

### 1.1 数据资产

| 资产 | 敏感级别 | 存储位置 | 访问路径 |
|------|:--:|------|---------|
| 交互日志内容（用户输入全文、助手响应摘要） | 中 | `docs/故事任务面板/<story>/{project}-交互日志.md` | 文件系统读取 |
| 会话 ID（YYMMDD-HHmmss） | 低 | 日志文件会话头 | `generateSessionId()` 生成 |
| 项目名（从 CLAUDE.md 提取） | 低 | 日志文件名和内容 | `readProjectName()` 读取 |
| 关键决策文本 | 中 | 日志文件 turn 条目 | `--decisions` 参数传入 |

### 1.2 功能资产

| 端点/组件 | 认证要求 | 授权级别 |
|----------|:--:|:--:|
| `cmdAppend()` | 无 | 本地文件系统写入权限 |
| `hasTurnEntry()` | 无 | 本地文件系统读取权限 |
| `readProjectName()` | 无 | CLAUDE.md 读取权限 |

## §2 威胁建模

| # | 威胁 | 攻击面 | 可能性 | 影响 | STRIDE 分类 |
|---|------|--------|:--:|:--:|-----------|
| T1 | 恶意 Markdown 注入用户输入导致日志渲染 XSS | `--user-input` → 日志文件 → Markdown 渲染器 | L | M | 篡改 (Tampering) |
| T2 | 伪造 turn 编号覆盖或混淆交互记录 | `--turn` 参数 | L | L | 伪装 (Spoofing) |
| T3 | 伪造 agent 名称混淆责任归属 | `--agent` 参数 | L | L | 伪装 (Spoofing) |
| T4 | 敏感信息通过 `--user-input` 或 `--assistant-response` 泄露到日志 | 参数 → 日志文件 | M | M | 信息泄露 (Information Disclosure) |
| T5 | 交互日志被非授权进程读取 | 文件系统 → 任意进程 | L | M | 信息泄露 (Information Disclosure) |
| T6 | 决策分隔符注入导致列表错乱 | `--decisions` 含分号 → 误拆分 | L | L | 篡改 (Tampering) |
| T7 | 日志文件被非授权进程修改或删除 | 文件系统 → 任意进程 | L | M | 篡改 (Tampering) |

## §3 信任边界

| 边界 | 跨越方向 | 数据流 | 校验点 | 当前状态 |
|------|---------|------|--------|:--:|
| CLI 参数 → log-interaction 内部 | 入站 | `process.argv` → `opts` 对象 | 必填参数检查（story/turn/agent 非空） | 已加固 |
| CLAUDE.md → log-interaction 内部 | 入站 | 文件读取 → 正则匹配 → 项目名 | `readProjectName()` 正则提取 + 回退 | 已加固 |
| log-interaction 内部 → 交互日志 | 出站 | `buildTurnEntry()` → Markdown → 文件 | 格式模板固定，参数直接拼接 | 未加固 |
| 交互日志 → 外部读取者 | 出站 | 文件系统 → 任意进程 | 无（依赖操作系统权限） | 未加固 |
