🗺️ 六步理解自改进闭环 建议严格按序

每步约 5 分钟，全程 ~30 分钟。从规则到引擎到技能，逐层理解自改进闭环的完整运作机制。

STEP 1 · 5 min 阅读自改进规则
打开 rules/self-improve.md，理解 D0-D8 诊断维度、E1-E4 效果评估等级、五条经验技能化升级路径的触发条件与目标映射。
→ 重点看诊断决策表和升级条件阈值
✅ 完成标志：能列出 5 条升级路径及其触发条件
STEP 2 · 5 min 理解 Agent 行为定义
阅读 agents/self-improve.md，掌握四段闭环（数据采集→诊断→提案→评估）的完整行为定义、提案矩阵和跨会话记忆注入机制。
→ 重点看提案矩阵路由表和记忆压缩策略
✅ 完成标志：能画出四段闭环的数据流图
STEP 3 · 5 min 学习诊断引擎
打开 lib/engine/diagnostics.mjs，理解 D0-D7 诊断信号的生成逻辑、触发条件和评分计算。结合 skills/rui/concerns/diagnostics.md 了解诊断关注点注入。
→ 跟踪一个诊断信号从触发到输出的完整路径
✅ 完成标志：能解释 D0-D7 每个维度的诊断逻辑
STEP 4 · 5 min 掌握提案与推荐系统
依次阅读 lib/proposals.mjs → lib/recommend.mjs → lib/recommend-detect.mjs，理解从诊断信号到改进提案的生成、路由、排序全流程。
→ 重点看提案类型路由表和推荐排序算法
✅ 完成标志：能说出 4 种提案类型的路由规则
STEP 5 · 5 min 理解执行记录与记忆注入
阅读 lib/record.mjs，理解执行记忆的写入、压缩、检索和注入机制。了解 .memory/health-trend.jsonl 的数据结构和跨会话恢复流程。

✅ 完成标志：能解释记忆压缩策略和相似检索注入
STEP 6 · 5 min 运行自改进技能
阅读 skills/rui-yry/SKILL.md，理解自改进技能的入口、参数和输出。执行 node skills/rui-yry/help.mjs 验证帮助输出，然后运行一次完整的自改进巡检。

✅ 完成标志：成功生成一份自改进报告并理解其内容

💡 常见误区 避坑指南

⚠️
不要跳过规则直接看引擎代码
自改进系统是规约驱动的——先读 rules/self-improve.md 理解"为什么这样诊断"和"升级条件是什么"，再看 lib/engine/diagnostics.mjs 才能理解实现细节。
🚫
不要混淆 D0-D7 诊断与 E1-E4 评估
D0-D7 是诊断维度（发现问题），E1-E4 是效果评估（衡量改进）。诊断触发提案，评估验证提案效果——两者是闭环的前后两段，不可混用。
✅
经验技能化需要连续触发
五条升级路径有严格的触发条件：process/quality/refactor/security 类型需连续三个故事触发；skill 类型需连续两个故事触发。单次触发不会启动升级。

🐞 排错速查 高频问题

🔧
健康检查报错缺少模块
症状：Cannot find module。修复：
npm install && node skills/rui-bot/send.mjs health
⏱
健康趋势文件为空或损坏
症状：.memory/health-trend.jsonl 解析失败。修复：删除损坏行或重建趋势文件。
tail -1 .memory/health-trend.jsonl | node -e "JSON.parse(require('fs').readFileSync('/dev/stdin','utf8'))"
🔒
企微通知未触发
症状：WECHAT_WEBHOOK 环境变量缺失。检查项：
echo $WECHAT_WEBHOOK && ls .env*
🧭
诊断维度触发异常
症状：预期触发的诊断维度未触发。检查诊断规则定义和触发条件。
grep -A5 "D[0-7]" rules/self-improve.md | head -40
📐
架构合规检查失败
症状：arch-check 报告不合规。先用 --verbose 查看详情，再用 --fix 自动修复。
node lib/arch-check.mjs --verbose && node lib/arch-check.mjs --fix
🔄
升级路径未触发
症状：经验技能化升级条件满足但未触发。检查连续触发计数是否正确累加。
grep -n "连续" rules/self-improve.md agents/self-improve.md
