## §2 诊断

### 2.1 诊断决策表

| 规则 | 触发条件 | 当前值 | 触发 |
|------|---------|--------|------|
| D0 | 代码行数 > 500 | ~400 | 否 |
| D1 | 单文件多职责 > 3 | SKILL.md 规约 + help.mjs 输出 | 否（职责分离清晰） |
| D2 | P0 审查未清零 | 0 | 否 |
| D3 | 跨模块耦合 | 仅委托 import-docs | 否 |
| D4 | Gate B > 2 轮 | 1 轮 | 否 |

### 2.2 六维评估

| 维度 | 评级 | 说明 |
|------|------|------|
| 耦合 | 低 — 仅委托 import-docs，零其他跨模块调用 | 自包含 CLI 技能 |
| 稳定性 | 高 — API 不可达降级提示，不崩溃 | 优雅降级 |
| 扩展性 | 中 — SKILL.md 内追加命令定义 | 模块内扩展 |
| 可测试性 | 高 — 命令输入 → 输出可预测 | 输入输出确定 |
| 安全边界 | 强 — kebab-case + API_X_TOKEN + 子进程隔离 | 三道防线 |
| 依赖方向 | 正确 — rui-story → import-docs（子进程委托） | 单向依赖 |

## §4 经验沉淀
