## §2 诊断

### 2.1 诊断决策表

| 规则 | 触发条件 | 本故事值 | 触发? | 根因假设 |
|------|---------|---------|-------|---------|
| D1 上下文膨胀 | 单文件 > 500 行 | 730 行 | ✅ | 格式化输出函数可抽取为独立模块 |
| D2 远端依赖 | API 调用 > 3 处 | 2 处 (query + read-file) | ❌ | — |
| D3 硬编码 | 配置硬编码 | API_URL 有默认值 + 环境变量覆盖 | ❌ | — |
| D4 函数职责过重 | 函数 > 50 行 | printList 62 行, printShow 37 行 | ✅ | 格式化逻辑含 TTY 判断 + 列对齐计算 |
| D5 并发安全 | 共享可变状态 | queue 在 worker 间共享 | ❌ | 单线程事件循环下安全 |

### 2.2 六维评估

| 维度 | 评级 | 证据 | 说明 |
|------|------|------|------|
| 耦合 | 低 | 仅依赖 Node.js 内置模块 (fs, path, child_process) + fetch API | 无第三方依赖 |
| 稳定性 | 高 | 全部命令实际执行通过，错误处理全覆盖 | 异常路径均优雅退出 |
| 扩展性 | 中 | 新增命令需添加 case + cmd* 函数 + print* 函数 | 模式一致但需多处修改 |
| 可测试性 | 中 | CLI 可通过 node 命令直接测试，但无自动化测试框架 | 依赖人工执行验证 |
| 安全边界 | 高 | Token 不落盘、命令注入防护、路径遍历约束、错误截断 | 6 项威胁全部缓解 |
| 依赖方向 | 健康 | 出站依赖：远端 API；入站依赖：CLAUDE.md + rui-state.json | 依赖方向清晰 |

<a id="sec4-lessons"></a>

## §4 经验沉淀

| # | 经验 | 类别 | 来源阶段 | 适用范围 |
|---|------|------|---------|---------|
| 1 | SKILL.md 规约可作为 doc --from-code 的高质量文档生成源 — 规约中的命令族全景、操作边界、状态判定直接映射为文档基线 | 方法论 | 文档生成 | 所有有完整 SKILL.md 的 skill |
| 2 | 写入命令（sync/clear/remove）代理给规约+agent 而非脚本实现是合理的分界 — 破坏性操作需要智能确认流程 | 设计模式 | 实现 | 后续 skill 的写入操作 |
| 3 | 项目名解析需要防御性设计 — CLAUDE.md 格式可能变化，3 模式 + fallback 是合理的防御深度 | 实现模式 | 实现 | 所有依赖 CLAUDE.md 解析的模块 |
| 4 | TTY 颜色输出应保持 `isTTY` 检测 — 管道/重定向场景下 ANSI 转义序列会污染输出 | 实现模式 | 验证 | 所有 CLI 工具 |
| 5 | 类型推断的并发策略（4 worker）在单故事场景无实际收益，但在多故事场景（10+）有显著提升 | 性能模式 | 验证 | 并发数量可按故事数动态调整 |
