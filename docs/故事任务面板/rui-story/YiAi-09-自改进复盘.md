### 2.2 六维评估

| 维度 | 评级 | 证据 | 说明 |
|------|------|------|------|
| 耦合 | 低 | 零跨模块调用，仅依赖 core 层 | `story_panel.py` 自包含 |
| 稳定性 | 高 | 文件系统 I/O 有退化对策（目录不存在 → 空结果） | 无 500 错误路径 |
| 扩展性 | 中 | 新增端点需在 `story_panel.py` 追加 | 模块内扩展简单 |
| 可测试性 | 高 | 所有端点纯函数风格，输入→输出可预测 | curl 可覆盖 |
| 安全边界 | 强 | X-Token 中间件 + 路径校验 + 子进程超时 | 三道防线 |
| 依赖方向 | 正确 | core → api/routes（符合分层架构） | 不反向依赖 |
