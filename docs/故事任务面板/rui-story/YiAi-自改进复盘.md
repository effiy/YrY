## §2 诊断

### 2.1 诊断决策表

| 规则 | 触发条件 | 当前值 | 触发 |
|------|---------|--------|------|
| D0 | 代码行数 > 500 | 487 | 否 |

| D4 | Gate B > 2 轮 | 1 轮 | 否 |

### 2.2 六维评估

| 维度 | 评级 | 说明 |
|------|------|------|
| 耦合 | 低 — 零跨模块调用，仅导入 core 层 | 自包含模块 |
| 稳定性 | 高 — 目录不存在返回空列表；sync 失败透传不崩溃 | 优雅降级 |
| 扩展性 | 中 — 模块内追加端点成本低 | FastAPI APIRouter |
| 可测试性 | 高 — curl 可覆盖全部端点 | 无状态设计 |
| 安全边界 | 强 — X-Token 中间件 + kebab-case 校验 + 路径遍历防护 | 三道防线 |
| 依赖方向 | 正确 — routes → core（项目内部）；routes → import-docs（子进程委托） | 依赖方向单向 |
