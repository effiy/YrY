## §4 经验沉淀

| # | 经验 | 类别 | 来源阶段 | 适用范围 |
|---|------|------|---------|---------|
| 1 | CLI 技能应优先使用远端 API 作为数据源，本地文件系统仅作为同步副本 | 模式 | 架构设计 | 依赖远端知识库的 CLI 技能 |
| 2 | 命令精简比功能堆砌更有价值 — 每次移除不必要命令都降低了维护成本 | 原则 | 迭代演进 | 所有 CLI 工具设计 |
| 3 | 双基线模型（问题空间 + 用户空间）使文档独立于实现，便于独立演进 | 模式 | 文档生成 | 所有 story 文档 |
| 4 | 语义常量提取（如 LEFT_COLUMN_WIDTH）让"魔法数字"的意图显式化 | 实践 | 代码重构 | 所有 help/输出格式化脚本 |
| 5 | sync 委托模式 — 不自实现同步逻辑，完全委托 import-docs | 模式 | 架构设计 | 跨技能协作 |
| 6 | 同一故事目录下多项目文档应保持对称 — 内部一致性降低认知负担 | 原则 | 文档管理 | 多项目共享故事目录 |
