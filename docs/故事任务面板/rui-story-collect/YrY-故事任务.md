## §0 基线声明

> **问题空间基线 (Problem Space Baseline)**: 本文档定义"做什么(WHAT)"和"验收标准(AC)"。下游所有文档必须可溯源至本文档的功能点与业务规则。

### 主要价值

- 🎯 七维指标一次采集：阻断率/P0密度/工具错误率/Agent参与/回溯/T3占比/提案闭合率
- 📊 三命令闭环：单故事诊断 → 跨故事汇总 → D0-D7 异常检测
- 🔍 异常检测基于跨故事统计均值，自动标记偏离 2x 的故事
- 📈 阶段耗时追踪帮助定位管线瓶颈
- 📋 JSON/Table 双格式输出，管道和人工查看兼顾

## §1 Story

### Story 1: CRUD 指标采集与异常检测

| 字段 | 内容 |
|------|------|
| 作为 | 项目管理者 |
| 我想要 | 自动采集每个故事的执行指标并检测异常 |
| 以便 | 数据驱动地发现效率退化、质量下降、流程阻塞等问题 |
| 优先级 | P0 |
| 范围边界 | 只读本地文件系统（.memory/* 和 .improvement/*），不修改任何文件 |
| 依赖 | 故事目录下有 execution-memory.jsonl 等数据文件 |

#### §1.1 User Operations

| # | 操作 | 触发条件 | 操作步骤 | 预期结果 |
|---|------|---------|---------|---------|
| 1 | 单故事指标采集 | 执行 `collect story --story=<name>` | 读取故事目录下全部数据文件 → 计算七维指标 → 输出 JSON 或表格 | 指标值精确到小数点后两位 |
| 2 | 跨故事指标汇总 | 执行 `collect all` | 扫描全部故事目录 → 逐故事计算指标 → 聚合输出 | 含故事数和总记录数的汇总行 |
| 3 | 异常检测 | 执行 `collect anomalies` | 计算跨故事均值 → 逐故事对照阈值 → 标记 D0-D7 异常 | 异常故事红色标注诊断标签 |
| 4 | 查看帮助 | 执行 `collect --help` | 输出命令列表 + 指标说明 + 异常检测规则 | 七维指标全部说明 |

## §2 Requirements

### 功能点

| FP# | 描述 | 输入 | 输出 | 错误行为 | 优先级 |
|-----|------|------|------|---------|:--:|
| FP1 | 单故事指标采集 — 从数据文件计算七维指标 | story name | 结构化指标对象（JSON 或表格） | 故事目录不存在 → 提示错误 | P0 |
| FP2 | 跨故事汇总 — 扫描全部故事并聚合 | 项目根目录 | 全部故事的指标列表 + 汇总统计 | 无故事目录 → 输出空状态 | P0 |
| FP3 | 异常检测 — D0-D7 跨故事对比 | 全部故事指标 + 阈值 | 异常故事列表 + 诊断标签 | 数据不足（<2 故事）→ 提示 | P0 |
| FP4 | 数据容错 — 缺失数据文件不崩溃 | 部分数据文件不存在 | 对应指标为 0 或空 | 损坏 JSONL 行跳过 | P1 |

### 业务规则

| R# | 描述 | 校验方式 | 证据级别 |
|----|------|---------|:--:|
| R1 | 阻断率 > 20% 或 > 2x 均值触发 D1 效率退化 | `detectAnomalies()` D1 逻辑 | A |
| R2 | P0 密度 > 2x 均值触发 D2 质量退化 | `detectAnomalies()` D2 逻辑 | A |
| R3 | T3 占比 > 30% 触发 D3 复杂度增长 | `detectAnomalies()` D3 逻辑 | A |
| R4 | 阶段耗时 > 3x 均值触发 D5 依赖退化 | `detectAnomalies()` D5 逻辑 | A |
| R5 | 提案闭合率 < 50% 触发 D7 配置漂移 | `detectAnomalies()` D7 逻辑 | A |
| R6 | JSONL 无效行静默跳过，不阻断整体计算 | `readJsonl()` try-catch 容错 | A |

### 数据约束

| 约束 | 类型 | 范围/格式 | 来源 |
|------|------|----------|------|
| 故事名称 | string | kebab-case | 故事目录名 |
| 窗口大小 | number | 默认 12（小时） | --window 参数 |
| 异常阈值 | number | 默认 2.0（倍数） | --threshold 参数 |
| 输出格式 | enum | json / table | --format 参数 |

## §3 成功标准

| SC# | 描述 | 度量方式 | 目标值 | 优先级 | 关联 FP# |
|-----|------|---------|--------|:--:|---------|
| SC1 | story 命令输出七维指标完整 | 输出含全部 7 个关键指标 | 7/7 非 null | P0 | FP1 |
| SC2 | all 命令扫描全部故事目录 | 故事数 = 实际目录数 | 100% 覆盖 | P0 | FP2 |
| SC3 | anomalies 命令标记异常故事 | 与人工审计一致 | 无误报/漏报 | P0 | FP3 |
| SC4 | 缺失数据文件不导致崩溃 | 任意文件缺失仍正常输出 | exit(0) | P0 | FP4 |

## §4 范围边界

### 范围内

| # | 条目 | 关联 FP# | 边界说明 |
|---|------|---------|---------|
| 1 | 读取 .memory/ 和 .improvement/ 下的数据文件 | FP1–FP3 | execution-memory, tool-audit, delivery-tracking, status-history, proposals |

| 3 | D0-D7 异常检测 | FP3 | 跨故事对比，5 条检测规则 |

### 范围外

| # | 条目 | 排除原因 | 替代方案 |
|---|------|---------|---------|
| 1 | 修改数据文件 | 只读采集 | — |
| 2 | 自动修复异常 | 异常检测只报告不修复 | `/rui yry` 自改进闭环 |
| 3 | 实时监控 | 按需手动执行 | Cron 定时执行 |

## §5 AC

| AC# | Given | When | Then | 门禁 |
|-----|-------|------|------|:--:|
| AC1 | 故事 X 目录有 execution-memory.jsonl | 执行 `collect story --story=X` | 输出七维指标 JSON | Gate A |
| AC2 | 项目有 ≥2 个故事目录 | 执行 `collect all` | 输出全部故事指标 + 汇总 | Gate A |
| AC3 | 某故事阻断率 = 50%，均值 = 10% | 执行 `collect anomalies` | 该故事标记 D1 效率退化 | Gate A |
| AC4 | 故事目录不存在 | 执行 `collect story --story=missing` | 提示目录不存在，exit(0) | Gate A |
