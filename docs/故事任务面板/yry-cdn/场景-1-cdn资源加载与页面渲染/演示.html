查看 CDN 加载链路
打开任意 Cat B 页面（审查/测试/演示/计划），使用浏览器 DevTools Network 面板观察资源加载顺序。
✅ 结果: 加载顺序：① shared.css（5.9KB）→ ② theme.css（11.3KB）→ ③ shared.js（5.1KB）。所有资源返回 HTTP 200，Content-Type 正确。页面在 DOMContentLoaded 后 120ms 可交互。
💡 说明: Cat A 页面在 shared.css 之前额外加载 Google Fonts（fonts.googleapis.com/css2?family=JetBrains+Mono），字体文件为 woff2 格式。Cat B 使用系统字体栈，跳过外部字体请求，首屏渲染快约 80ms。
打开审查页面 → F12 → Network 标签 → 刷新 → 观察加载瀑布流📋
验证双主题系统
同时打开 Cat A 页面（架构图/知识图谱）和 Cat B 页面（审查/测试/演示/计划），对比样式差异。
✅ 结果: Cat A — 深蓝黑底(#020617) + JetBrains Mono 等宽字体 + 脉冲动画点 + 卡片使用半透明背景。Cat B — 深紫黑底(#161620) + 系统字体 + 渐变色卡片 + 更大圆角和阴影。

测试 JS API (YrY.*)
在浏览器 Console 中测试 shared.js 暴露的 9 个公共 API：YrY.toast / YrY.copyCmd / YrY.switchPanel 等。
✅ 结果: YrY.toast('Hello CDN!') 在底部居中显示通知，1.8s 后自动消失。YrY.copyCmd 向剪贴板写入文本并触发按钮图标切换。window.YrY 对象在 IIFE 执行后立即可用。
💡 说明: shared.js 使用 IIFE 模式 (function(){...})() 封装，仅暴露 window.YrY 单一命名空间。内部 3 个私有辅助函数（_getOrCreateToastEl / _resetToastTimer / _copyToClipboard）不污染全局。所有 API 在 DOMContentLoaded 之前即可调用（内部自动等待 DOM 就绪）。

验证降级策略
使用 DevTools Network 面板阻止外部资源，验证页面的降级行为。
✅ 结果: 阻止 fonts.googleapis.com 后，Cat A 页面回退为 monospace 字体栈，布局不受影响。阻止 shared.js 后，HTML/CSS 正常渲染，仅交互增强（toast/copyCmd/switchPanel）不可用——无 JS 异常，页面不白屏。
💡 说明: 降级策略分为两层：① 字体降级 — Cat A 页面 CSS font-family 指定了回退栈 'JetBrains Mono', monospace；② JS 降级 — 所有 YrY API 在调用前检查 DOM 元素存在性，不存在时静默返回而非抛异常。页面核心内容（HTML + CSS）独立于 JS 运行。
# DevTools → Network → 右键 fonts.googleapis.com → Block request domain

查看页面性能 (Lighthouse)
使用 Chrome DevTools Lighthouse 面板运行性能审计，检查 CDN 资源的加载性能。
✅ 结果: Cat B 页面 Performance 评分 96/100。首次内容绘制(FCP) 0.6s，最大内容绘制(LCP) 0.9s。CSS 文件大小：shared.css 5.9KB + theme.css 11.3KB = 17.2KB (gzip 后 ~3.5KB)。无渲染阻塞资源（CSS 为 <link> 方式异步加载）。
💡 说明: 所有 CDN 资源使用 <link rel="stylesheet"> 方式加载（不阻塞 HTML 解析），shared.js 放在 </body> 之前（不阻塞 DOM 构建）。Google Fonts 使用 <link> 而非 @import（避免 CSS 链式加载延迟）。整体加载策略遵循 Web Vitals 最佳实践。
