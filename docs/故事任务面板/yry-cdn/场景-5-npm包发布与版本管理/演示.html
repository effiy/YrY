查看 package.json 配置
检查 cdn/package.json 的 name/version/files 字段，理解 npm 包元数据和发布清单
cat cdn/package.json | jq .📋
✅ 结果: name: cdn, version: 1.0.0, files: [shared.css, shared.js, theme.css, theme-mono.css, package.json] — 5 文件精准发布清单，npm 包总体积约 29KB。
💡 说明: files 字段限制发布内容，确保 tests/、docs/ 等开发文件不会泄漏到 npm registry。消费者 npm install cdn 仅下载生产文件。
理解语义化版本决策树
学习 PATCH/MINOR/MAJOR 三级版本号的选择规则和影响范围
✅ 结果: PATCH(1.0.0→1.0.1): Bug 修复。MINOR(1.0.0→1.1.0): 新增 API/组件。MAJOR(1.0.0→2.0.0): 不兼容变更。npm ^1.0.0 自动获取 PATCH+MINOR 更新。
💡 说明: 语义化版本 (semver.org) 是 npm 生态的基础约定。消费者在 package.json 中写 "^1.0.0" 表示接受 1.x.x 范围内的所有 PATCH 和 MINOR 更新。
PATCH: fix .card shadow → 1.0.0→1.0.1
MINOR: add YrY.smoothScroll → 1.0.0→1.1.0
MAJOR: rename .card→.card → 1.0.0→2.0.0📋
模拟 npm publish 流程
执行 npm version → npm publish → git push --tags 完整的发布三步曲
✅ 结果: npm version patch: 更新 package.json 版本 + git commit + git tag (三合一)。npm publish: 将 5 文件包推送到 npm registry。git push --tags: 同步 tag 到 GitHub。
💡 说明: npm version 命令一步完成版本号更新→commit→tag 三个操作，是发布前最关键的命令。它确保版本号与 git tag 始终一致。
npm version patch                    # 1.0.0→1.0.1 (自动 commit+tag)
npm publish --access public          # 发布到 npm registry
git push origin main --tags          # 推送 tag 到远程📋
验证发布结果
使用 npm info 查看已发布包的元数据、版本列表和依赖信息
✅ 结果: npm info cdn: 返回 name/version/description/keywords。npm info cdn versions: 返回 ["1.0.0", "1.0.1"] 版本列表。确认目标版本已在 npm registry。
💡 说明: npm info 读取的是 npm registry 的实时数据（非本地），是验证发布成功的权威命令。如果版本列表包含目标版本号，说明发布已生效。
npm info cdn version             # 查看最新版本
npm info cdn versions --json     # 查看全部版本列表📋
Git Tag 版本追溯
使用 git tag 查看版本标签链，git show 重现任意历史版本的完整代码
✅ 结果: git tag -l 'v*' → v1.0.0, v1.0.1。git show v1.0.0 --stat → 显示该版本发布时的 5 个文件列表和提交信息。支持任意历史版本的可重现构建。
💡 说明: git tag 是"可回退"的保障——发现 v1.0.1 引入回归 bug 时，可以 git checkout v1.0.0 快速回退到前一个稳定版本进行 hotfix。
git tag -l 'v*' --sort=-version:refname  # 版本标签历史
git show v1.0.0 --stat                    # 查看发布时的文件快照📋
