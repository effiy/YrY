✅ 好案例 语义化版本策略 (Semver)

📁 cdn/package.json · version 字段 + git tag 管理
采用标准的 PATCH/MINOR/MAJOR 三级语义化版本策略：PATCH（1.0.0→1.0.1）用于 CSS bug 修复和样式微调，MINOR（1.0.0→1.1.0）用于新增 JS API 或组件，MAJOR（1.0.0→2.0.0）用于重命名 API 或不兼容的 CSS class 变更。
💡 为什么好：语义化版本使得 55 个消费者页面可以精确控制依赖版本——"^1.0.0" 自动获取 PATCH 和 MINOR 更新但不会意外引入 MAJOR 破坏性变更。这是 npm 生态的约定俗成，降低了版本升级的沟通成本。
PATCH (1.0.0→1.0.1): 修复 .card 阴影偏移 1px
MINOR (1.0.0→1.1.0): 新增 YrY.smoothScroll API
MAJOR (1.0.0→2.0.0): 重命名 .card → .card (破坏性)

✅ 好案例 files 字段精准控制发布内容

📁 cdn/package.json · "files": [...] 字段
package.json 的 files 字段仅列出 5 个文件：shared.css (5.9KB), shared.js (5.1KB), theme.css (11.3KB), theme-mono.css (6.1KB), package.json。测试文件、开发文档、构建脚本等不会被发布到 npm，确保了 npm 包的最小化和安全性。
💡 为什么好：精准的 files 配置有两个好处：① 减小 npm 包体积——用户 npm install cdn 只下载需要的 5 个文件（共约 29KB）；② 安全——开发环境的配置文件（如 .env 等）不会意外泄漏到公开的 npm registry。
// package.json
{ "files": [
    "shared.css",
    "shared.js",
    "theme.css",
    "theme-mono.css",
    "package.json"
  ]
}

✅ 好案例 git tag 版本追踪

📁 git tag v1.0.0 → 对应 package.json version 1.0.0
每次 npm publish 前创建对应的 git tag（如 v1.0.0 / v1.0.1），tag message 包含版本变更摘要。git tag -l 'v*' 可列出全部历史版本，git show v1.0.0 可查看该版本的完整代码状态。
💡 为什么好：git tag 将 npm 发布与代码仓库的特定提交点绑定，实现了"任意历史版本可重现"——git checkout v1.0.0 即可获得该版本发布时的完整代码，用于回归测试或紧急 hotfix 发布。
# 发布流程
npm version patch            # 1.0.0 → 1.0.1 (自动创建 git tag)
npm publish --access public  # 发布到 npm
git push origin main --tags  # 推送 tag

npm 包发布流程整体规范：语义化版本、精准 files 控制、git tag 追溯均已到位。

以下改进建议为 P2/P3 级别，旨在增强自动化程度和可追溯性。

P1 · 立即

添加 .npmignore 排除开发文件

创建 .npmignore 文件排除 tests/、docs/、scripts/ 等开发文件。虽然 files 字段已限制发布范围，但 .npmignore 提供了额外的安全网，防止未来 files 字段错误配置导致开发文件泄漏。

P1 · 立即

配置 prepublishOnly 自动校验脚本

在 package.json scripts 中添加 "prepublishOnly": "node scripts/validate-package.mjs"，自动校验：① files 字段列出的文件全部存在；② 版本号与 git tag 一致；③ 所有 CSS/JS 文件语法有效。

P2 · 本迭代

增加 publish 后的自动 git tag 一致性校验

在 CI 流程中添加 post-publish 校验：npm info cdn version 与 git describe --tags 对比，确保发布版本与 git tag 一一对应。如有不一致自动告警。

P3 · 未来

配置 npm deprecation 策略

定义版本弃用规则：① 发现安全漏洞的版本立即标记 deprecated；② MAJOR 版本发布后 6 个月，前一个 MAJOR 版本标记为 deprecated。使用 npm deprecate 命令执行。
