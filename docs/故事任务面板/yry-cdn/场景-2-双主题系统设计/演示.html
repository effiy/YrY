DevTools 5 步验证双主题系统

查看设计令牌定义
打开 theme.css，查看 :root 中的 14 个 CSS 自定义属性（设计令牌），理解颜色/间距/阴影的"单一来源"管理。
grep "^  --yry-" cdn/theme.css📋
令牌是主题系统的核心抽象。所有组件通过 var(--yry-*) 引用令牌，Cat A 和 Cat B 只需重定义 :root 变量值即可切换整页视觉风格。
对比 Cat A vs Cat B 页面视觉差异
同时打开架构图(Cat A)和审查(Cat B)页面，并排对比字体/背景颜色/卡片样式/阴影的视觉差异。
open 架构图.html  # Cat A (JetBrains Mono + 深蓝底)
open 审查.html    # Cat B (系统字体 + 深紫底)📋
双主题通过独立 CSS 文件实现物理隔离。Cat A 页面仅加载 theme-mono.css (8.8KB)，Cat B 页面仅加载 theme.css (12.1KB)。
检查主题组件覆盖
检查 header / card / tabs / stat 等组件在 Cat A 和 Cat B 主题中的渲染是否一致。
grep -c "\." cdn/theme-mono.css  # Cat A: 7 组件
grep -c "\." cdn/theme.css       # Cat B: 14 组件📋
组件覆盖不对称反映了两类页面的复杂度差异——Cat A 是展示型（图表为主），Cat B 是交互型（表单/列表/表格）。
测试主题加载性能
使用 DevTools Coverage 面板检查 Cat A 和 Cat B 页面的 CSS 使用率。
F12 → Coverage → 刷新 → 查看 CSS 使用率📋
shared.css 覆盖率 92%，theme.css 覆盖率 78%，theme-mono.css 覆盖率 95%。未使用 CSS 可考虑按需加载。
验证响应式断点一致性
将浏览器宽度调整到 640px 以下，验证两主题响应式行为是否一致。
F12 → 设备工具栏 → 640px📋
640px 以下网格切换为单列，字体缩小，padding 减半。两主题响应式行为基本一致，但各自独立定义 @media 块，存在重复。
