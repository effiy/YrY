查看设计令牌定义
打开 theme.css，查看 :root 中的 14 个 CSS 自定义属性（设计令牌），理解颜色/间距/阴影的"单一来源"管理。
✅ 结果: 14 令牌：7 颜色(--yry-bg / --yry-bg-card / --yry-text 等) + 4 功能色(--yry-accent / --yry-cyan / --yry-pass / --yry-fail) + 3 间距(--yry-shadow / --yry-shadow-lg / --yry-radius)。
💡 说明: 令牌是主题系统的核心抽象。所有组件通过 var(--yry-*) 引用令牌，Cat A 和 Cat B 只需重定义 :root 变量值即可切换整页视觉风格。
cat cdn/theme.css | grep "^  --yry-"  # 查看所有设计令牌📋
对比 Cat A vs Cat B 页面视觉差异
同时打开架构图(Cat A)和审查(Cat B)页面，并排对比字体/背景颜色/卡片样式/阴影的视觉差异。
✅ 结果: Cat A — 深蓝黑底(#020617) + JetBrains Mono 等宽字体 + 脉冲动画点。Cat B — 深紫黑底(#161620) + 系统字体 + 渐变卡片 + 更大阴影圆角。
💡 说明: 双主题通过独立 CSS 文件实现物理隔离。Cat A 页面仅加载 theme-mono.css (6.1KB)，Cat B 页面仅加载 theme.css (11.3KB)，两者共享 shared.css 的全局样式。
# 并排对比
open 架构图.html  # Cat A (JetBrains Mono + 深蓝底)
open 审查.html    # Cat B (系统字体 + 深紫底)📋
检查主题组件覆盖
检查 header / card / tabs / stat 等组件在 Cat A 和 Cat B 主题中的渲染是否一致。
✅ 结果: Cat A 覆盖 7 组件（架构图/知识图谱专用），Cat B 覆盖 14 组件（审查/测试/演示/计划清单通用）。7 个公共组件在两主题中均有定义，但结构属性存在重复。
💡 说明: 组件覆盖不对称反映了两类页面的复杂度差异——Cat A 页面是展示型（图表为主），Cat B 页面是交互型（表单/列表/表格）。
# 统计各主题的组件定义
grep -c "\." cdn/theme-mono.css  # Cat A: 7 组件
grep -c "\." cdn/theme.css       # Cat B: 14 组件📋
测试主题加载性能
使用 DevTools Coverage 面板检查 Cat A 和 Cat B 页面的 CSS 使用率，验证无冗余样式加载。
✅ 结果: shared.css 覆盖率 92%（面包屑/导航/动画全部使用），theme.css 覆盖率 78%（14 组件中 11 个在典型页面上使用）。Cat A 的 theme-mono.css 覆盖率 95%（7 组件全部使用）。
💡 说明: CSS 覆盖率数据可用于指导按需加载策略——将低频组件（如 modalIn 动画）拆分为独立 CSS 文件，仅在需要时加载。
F12 → Coverage → 刷新页面 → 查看 CSS 使用率📋
验证响应式断点
将浏览器宽度调整到 640px 以下，验证 Cat A 和 Cat B 页面的响应式布局正确切换。
✅ 结果: 640px 以下：网格切换为单列，字体缩小（h1: 1.8rem→1.4rem），stats 卡片 min-width 从 80px 降至 65px，padding 减半。两主题响应式行为一致。
💡 说明: 响应式规则在 theme.css 和 theme-mono.css 中独立定义（各含 @media(max-width:640px) 块），应抽取到 shared.css 以消除重复。
# Chrome DevTools → 切换设备工具栏 → 设置宽度 640px📋
