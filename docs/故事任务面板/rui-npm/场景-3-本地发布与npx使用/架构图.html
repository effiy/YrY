场景-3 · 本地发布与npx使用 — 架构图

publish 管道：文件/目录 → 登录校验 → package.json → registry 冲突检测 → npm publish

发布管道

  • 文件/目录输入 → 登录校验
  • package.json 自动生成或读取
  • npm publish → registry
  • 发布确认 + CDN URL 返回

冲突与错误处理

  • 同名冲突 → --name 改名发布
  • 未登录 → 提示 npm login
  • 无 package.json → 自动生成

npx 使用

  • npx 即发即用
  • --yes 自动确认不提示
  • 流式输出执行结果
