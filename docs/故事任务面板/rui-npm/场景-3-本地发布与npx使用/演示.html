发布本地脚本为 npm 包
自动生成 package.json，验证 npm 登录状态，检测 registry 同名冲突后发布。
/rui-npm publish ./my-cli.mjs --name my-util📋
💡 说明: publish 自动完成 5 步: 验证登录 → 生成 package.json → 冲突检测 → 打包上传 → 输出 CDN URL。
✅ 结果: 📦 发布成功 — 包名: my-util@1.0.0 / 入口: my-cli.mjs (3.2KB) / CDN URL: unpkg.com/my-util@1.0.0/my-cli.mjs。
通过 npx 直接运行
无需安装，npx 自动拉取远程包并执行。
/rui-npm npx my-util📋
💡 说明: npx 自动从 registry 拉取并执行。首次需下载，后续 24h 从缓存读取。
✅ 结果: ⚡ npx 执行 — 拉取 my-util@1.0.0 (3.2KB, 0.6s) → 输出: "Hello from my-util! v1.0.0"。
传递参数给 npx 包
使用 -- 分隔符将参数透传给被执行的包。
/rui-npm npx my-util@1.0.0 -- --flag value📋
💡 说明: -- 是 npx 的分隔符——左边是 npx 的参数，右边是传给被执行的包的参数。
✅ 结果: ⚡ npx 执行 (带参数) — --flag value → 输出: "Flag set to: value" → 参数透传成功。
模拟发布预览
--dry-run 模拟发布过程，预览上传内容，不实际上传。
/rui-npm publish ./my-lib --dry-run📋
💡 说明: --dry-run 模拟发布全过程但不实际上传。强烈建议首次发布或大版本升级前先 dry-run。
✅ 结果: 🔍 Dry Run 预览 — 将打包: 3 个文件 / 包名: my-lib / 总大小: 4.4KB / [模拟] 未实际上传。
