查看账号下所有 npm 包
一条命令列出自己发布的所有包——名称、描述、最新版本、周下载量，按下载量降序排列。
/rui-npm my-packages📋
💡 说明: my-packages 优先调用 registry search API (GET /-/v1/search?text=maintainer:<user>)，返回含描述/版本/下载量的完整元数据；API 不可达时自动降级为 npm access ls-packages。
✅ 结果: 📦 账号下 3 个包 — my-util (v2.1.0, 1.2K/周) / my-cli (v1.5.0, 340/周) / my-lib (v0.9.0, 86/周)。
安全废弃旧版本
标记废弃版本并附替代方案引导——不影响已安装用户，比直接删除更安全。
/rui-npm deprecate my-util@1.0.0 "Use 2.0.0 — API redesigned"📋
💡 说明: deprecate 执行前验证当前用户是包所有者（npm whoami + npm view maintainers 交叉检查）。非所有者拒绝操作并展示维护者列表。废弃消息会显示在 npm 网站包页面。
✅ 结果: ⚠️ my-util@1.0.0 已标记为 deprecated — 替代版本: 2.0.0 / 查看: npmjs.com/package/my-util/v/1.0.0。
删除误发版本（不可逆操作）
安全警告前置展示后，精确删除指定版本。不可逆操作，慎用。
/rui-npm unpublish my-util@1.0.0📋
💡 说明: unpublish 执行前展示三重安全警告：① 操作不可逆 ② 72h 内可通过 npm support 恢复 ③ 包名可被他人注册。所有权验证 + 安全警告双重确认后才执行。
⚠️ 安全提示: 优先使用 deprecate 而非 unpublish。deprecate 对已安装用户无影响；unpublish 是破坏性操作，npm 官方建议仅在误发密钥等紧急场景使用。
✅ 结果: 🗑️ my-util@1.0.0 已从 registry 删除 — ⚠️ 不可恢复 / 72h 内可联系 npm support / 包名 my-util 仍被占用（其他版本存在）。
完整生命周期：从本地发布到账号清理
发布本地文件/文件夹 → 查看账号资产 → 废弃旧版本 → 清理误发版本，四步闭环。
/rui-npm publish ./src/my-helper.mjs --name my-helper
/rui-npm my-packages
/rui-npm deprecate my-helper@0.1.0 "Merged into my-util@2.0.0"
/rui-npm unpublish my-helper@0.0.1 --force📋
💡 说明: 完整闭环演示账号级包管理的四个阶段：
① publish — 将本地 .mjs 文件或目录发布为 npm 包（自动生成 package.json，支持 --name/--version/--dry-run）
② my-packages — 确认新包出现在账号资产列表中
③ deprecate — 标记不再维护的早期版本，引导用户升级
④ unpublish --force — 删除误发的占位版本（--force 跳过 72h 限制）
✅ 结果: 🔄 完整闭环验证通过 — publish ✅ / my-packages 确认可见 ✅ / deprecate 标记成功 ✅ / unpublish 删除完成 ✅。
