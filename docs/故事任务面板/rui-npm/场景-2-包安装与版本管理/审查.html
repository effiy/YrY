💡 为什么好：参数命名遵循 npm 惯例而非发明新术语——用户已经知道 @version 和 --dev 的含义，零学习成本。--depth 默认值为 0（仅直接依赖），符合 80% 使用场景。

💡 为什么好：错误信息的最佳实践是"告诉用户发生了什么 + 为什么会发生 + 下一步怎么做"。这里做到了三点中的最关键一步——下一步可操作。

增强 install 幂等性检查

install 已安装的包时应提示"已安装，将更新 package.json"而非静默重装。当前 npm install 行为是不幂等的（可能触发 postinstall 钩子），明确提示可让用户知情决策。
