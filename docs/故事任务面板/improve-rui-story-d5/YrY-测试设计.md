### §0 基线声明

> **测试设计基线 (Test Design Baseline)**: 本文档为 `improve-rui-story-d5` 的测试设计，Gate A 交接信令。

---

### 主要价值

- 🧪 **测试策略**: 验证 D5 诊断准确性和工具调用可靠性
- 🎯 **覆盖目标**: 工具调用正常/边界/异常三类场景全覆盖
- 📋 **Gate A 信令**: 测试设计完成 → 允许进入 code 阶段
- 🔄 **回归保护**: 确保修复不引入新的工具调用失败模式

<a id="sec2-cases"></a>

## §2 测试用例

### TC-N-01: D5 诊断正确检测工具调用失败

| 字段 | 内容 |
|------|------|
| 类型 | 正常 |
| 优先级 | P1 |
| Given | execution-memory.jsonl 包含 ≥3 条工具调用记录，其中 1 条 status=failure |
| When | yry §2 执行 D5 诊断扫描 |
| Then | D5 检测到失败率 33.3%，生成改进提案写入 proposals.jsonl |

### TC-N-02: 工具调用全部成功时 D5 不触发

| 字段 | 内容 |
|------|------|
| 类型 | 正常 |
| 优先级 | P1 |
| Given | execution-memory.jsonl 中所有工具调用 status=success |
| When | yry §2 执行 D5 诊断扫描 |
| Then | D5 不触发，无提案生成 |

### TC-N-03: 修复后失败率降低

| 字段 | 内容 |
|------|------|
| 类型 | 正常 |
| 优先级 | P1 |
| Given | 已应用工具路由修复，execution-memory 含修复后 ≥3 条 success 记录 |
| When | yry §2 重跑 D5 诊断 |
| Then | D5 失败率 < 5%，不触发新提案 |

### TC-E-01: 执行记忆数据不足

| 字段 | 内容 |
|------|------|
| 类型 | 边界 |
| 优先级 | P2 |
| Given | execution-memory.jsonl 为空或 < 3 条记录 |
| When | yry §2 执行 D5 诊断 |
| Then | D5 降低置信度，标注 Level C 证据，不生成故事实例 |

### TC-E-02: 修复后诊断仍触发

| 字段 | 内容 |
|------|------|
| 类型 | 异常 |
| 优先级 | P1 |
| Given | 已应用修复但 execution-memory 中仍有 failure 记录 |
| When | yry §2 重跑 D5 诊断 |
| Then | ≤2 轮自修复；超过则标记 skip，记录到 proposals.jsonl |

### TC-R-01: 回归 — 现有工具调用不受影响

| 字段 | 内容 |
|------|------|
| 类型 | 回归 |
| 优先级 | P0 |
| Given | 应用工具路由修复后 |
| When | 执行 /rui doc / /rui code / /rui update 等标准命令 |
| Then | 所有命令正常完成，无新增工具调用失败 |
