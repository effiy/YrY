初始化项目基线
运行 /rui init 建立项目骨架。detect → explore → generate → arch → setup → verify，自动识别技能、Agent 和规则的模块结构。
/rui init📋
💡 说明: 初始化建立了完整的项目骨架基线。CLAUDE.md / README.md 中的 marker 段全量重生，版本号 4.3.0 已同步到 plugin.json / CLAUDE.md / README.md 三处。后续所有管线操作依赖此基线。
✅ 结果: 六阶段全部通过 — detect(6 技能) → explore(9 Agent + 8 规则) → generate(12 文件) → arch(SVG 三图层) → setup(.claude/ 配置) → verify(PASS) — 耗时约 45 秒，所有 rui 标记段已生成。
查看模块全景
阅读 CLAUDE.md 中的项目画像和角色拓扑。AGENT.md 定义了 8 个 Agent 的职责分配和协作关系。
cat CLAUDE.md agents/AGENT.md📋
💡 说明: 两个文件共同构成项目的「操作宪法」和「组织架构图」。CLAUDE.md 定义不可妥协底线（认证不可绕过/密钥不落盘/输入必校验/规约完整性/自托管一致性），AGENT.md 规定 Agent 间的信号协议和 P0 清零纪律。新人只需阅读这两个文件即可理解项目运行规则。
✅ 结果: CLAUDE.md: 3 基础信念 + 4 铁律 + 4 退化对策 + 项目约束 + 6 引导入口；AGENT.md: 9 Agent 拓扑 + 6 行为纪律 + 4 设计原则 + 3 执行准则 + ADR 记录。
探查技能模块入口
每个技能目录下的 SKILL.md 是模块规约入口。rui 管线串行调用 6 个技能形成完整 SDLC。
ls skills/*/SKILL.md📋
💡 说明: 每个 SKILL.md 是技能的独立规约入口，包含命令参考、管线一览、核心约束、mermaid 图和阻断标识。规约内联了关键模式摘要，不依赖外部 URL 可达性（退化对策 L1 自包含）。技能间通过 plugin.json 注册表声明依赖关系。
✅ 结果: 6 个 SKILL.md 规约文件 — rui/SKILL.md (SDLC 编排) / rui-story/SKILL.md (故事面板) / rui-claude/SKILL.md (配置管理) / rui-import/SKILL.md (文档同步) / rui-bot/SKILL.md (企微通知) / rui-npm/SKILL.md (npm 包管理)。
验证 Agent 职责边界
阅读每个 Agent 的 .md 文件确认职责范围。pm 决策 → architect 设计 → coder 实现 → tester 验证 → security 审计 → reporter 报告。
ls agents/*.md📋
💡 说明: pm → architect → planner → coder 是正向流水线；security → code-reviewer 是旁路审计，结果注入 coder 的任务列表；tester 做 Gate A/B 验证；reporter → self-improve 形成反馈闭环。每个 Agent 有明确的输入信号、输出信号和交接验证方法。
✅ 结果: 9 个 Agent 定义文件 — AGENT.md (总纲) / pm.md / architect.md / planner.md / coder.md / tester.md / security.md / code-reviewer.md / reporter.md / self-improve.md。
检查规则覆盖
rules/ 目录定义了管线纪律、安全护栏、文档约束和自改进规范。每条规则对应一个治理维度。
ls rules/*.md📋
💡 说明: 每条规则包含触发条件、验证方法和阻断标识。规则之间有关联 — code-pipeline 引用 delivery-gate 的收口步骤，security-guardrails 的约束注入 code-pipeline 的 Gate A 检查。规则文件不依赖外部链接（退化对策 L1）。
✅ 结果: 8 条治理规则 — code-pipeline.md (管线纪律) / delivery-gate.md (交付门禁) / doc-generation.md (文档生成约束) / knowledge-graph.md (知识图谱规范) / security-guardrails.md (安全护栏) / self-improve.md (自改进闭环) / context-engineering.md (上下文工程) / expression-priority.md (表达优先)。
生成架构全景图
架构图以 SVG 渲染三层拓扑（Skills → Agents → Rules），标注每个模块的核心职责。
open 架构图.html📋
💡 说明: 架构图是「表达优先」铁律的落地实现 — 用可缩放 SVG 替代大段文字描述，三层图层清晰展示模块边界和交互关系。每个节点可点击跳转，svg-pan-zoom 支持拖拽和缩放，适配从 640px 移动端到 4K 显示器的全部尺寸。
✅ 结果: 浏览器打开架构图.html — SVG 三层拓扑: Layer1(6 Skills 带管线连接) → Layer2(9 Agents 带协作边) → Layer3(8 Rules 带治理维度标注)，节点间边标注数据流方向和信号类型，支持点击节点跳转到对应规约文件。

🧪

验证关卡。Gate A 准入检查，Gate B 交付验证，每轮 ≤ 2 修复。
输入: 代码变更 → 输出: 测试报告
🛡️
security角色Agent
安全审计。S1-S5 防线扫描，密钥/XSS/注入 P0 阻断。

📊
reporter角色Agent
报告生成。汇总各阶段产出，生成交付总结和变更日志。

交付收口。三步 Hook（通知日志 → 同步 → 企微），任一失败回滚。
