YrY Self-Test — 场景 2: 提交前增量自检

Commit Hook 触发 · 5 步增量门禁 · 分支隔离验证 · 变更范围约束 · P0 回归检测 · 文档同步检查

🚫 阻断门禁 (G1–G3)

  • • G1: 分支隔离 — 在 main 上直接阻断
  • • G2: 变更范围 — 越界文件列出所属故事
  • • G3: P0 回归 — 新增阻断标识则阻断
  • • 阻断即停：不继续读取文件系统
  • • 每项含: 名称 + 证据 + 修复路径

⚠ 警告门禁 (G4)

  • • G4: 文档同步 — 代码变而文档未动则警告
  • • 纯文档变更自动跳过此步
  • • 警告不阻断提交，但建议修复
  • • 列出需同步的场景文档章节
  • • 可配置升级为阻断

⚡ 增量特性

  • • 仅检查受影响关卡，非全量
  • • ≤ 10 秒完成 (≤ 20 文件)
  • • 每次实时查询版本控制系统
  • • P0 对比基于变更前状态快照
  • • 只读，不修改任何文件
