分支隔离检查
验证当前工作在 feat/<name> 分支上，禁止直接在 main 分支修改。分支隔离不可绕过。
✅ 结果: branch-check.mjs 输出 — 当前分支: feat/yry-self-test ✅ / 分支前缀验证: feat/ ✓ / main 分支保护: 非 main ✓ / 跨分支文件检查: .memory/ 和 .claude/ 缓存未跨分支共享 ✅ → 通过。
💡 说明: 分支隔离是不可妥协底线 #7。branch-check.mjs 做三件事: (1) 验证当前在 feat/ 分支，(2) 确认不在 main，(3) 扫描 .memory/ .claude/ 等缓存目录确认未跨分支共享状态。违反任一项 = P0 阻断 commit。
node skills/rui/branch-check.mjs📋
铁律合规检查
对照四大铁律逐条验证。验先于称（有证据）· 溯先于修（有根因）· 清先于进（P0清零）· 表达优先（图→文→表）。
✅ 结果: 铁律合规扫描 — 验先于称: diff 中无"应该没问题"/"上次通过了"等违反信号词 ✅ / 溯先于修: 所有修复前有 grep/Read 证据 ✅ / 清先于进: 当前模块 P0=0 ✅ / 表达优先: 变更中架构描述使用 mermaid 图非纯文本 ✅ → 四大铁律全部合规。
💡 说明: 铁律合规检查不是 grep 关键词，而是对照四种违反信号扫描 diff 内容。验先于称的信号词: "should work"/"上次通过"/"应该没问题"; 溯先于修的信号: 无根因分析的修复方案; 清先于进的信号: 跳过 P0 标 P1; 表达优先的信号: 架构用大段文字无图。
cat CLAUDE.md | grep "NO COMPLETION"📋
变更文件范围验证
git diff 检查变更范围是否与故事计划一致。超出范围的变更标记为异常需解释。
✅ 结果: git diff --stat origin/main — 变更文件 12 个 (8 演示.html 修改 + 2 SKILL.md 更新 + 1 知识图谱.json + 1 plugin.json) / 故事计划预期: 12 文件 (yry-self-test 场景 1-4 演示补全) → 范围匹配 ✅ / 0 超范围文件。
💡 说明: 变更范围验证防止「顺手改」问题 — 开发者在修一个 bug 时顺手改了无关文件。超出故事计划范围的变更必须: (1) 在 commit message 中明确说明原因，(2) 创建对应的故事卡片记录，(3) 或者拆分到独立 commit。未解释的超范围变更 = 阻断 commit。
git diff --stat origin/main📋
P0 清零确认
确认所有已知 P0 已清零，无未处理的阻断问题。P0 未清不得进入下一模块。
✅ 结果: grep P0/TODO 扫描 — P0 标记: 0 个未解决（所有 P0 已标 resolved 或 closed）/ TODO 项: 2 个（"TODO: 补充场景-4 的 rui-npm audit 测试用例" + "TODO: lib/ 共享函数文档补全"）→ 2 个 TODO 均为 P2 级优化，非阻断。
💡 说明: P0 清零确认是「清先于进」铁律的执行检查。grep 扫描后人工确认每个 P0/TODO 的状态 — P0 必须已解决；P1 可有未解决但需在 commit message 中说明；P2 可延后。关键判断: P0 是阻断级（安全漏洞/数据丢失/规约不一致），不是所有 TODO 都是 P0。
grep -r "P0\|TODO" --include="*.md" .📋
测试增量执行
运行受变更影响的测试，验证增量改动不破坏已有功能。仅执行变更相关测试。
✅ 结果: audit.mjs --incremental — 变更文件 12 个 → 裁剪测试范围: rui.test.mjs(12 项) + rui-npm.test.mjs(0 项，无 npm 相关变更) → 执行 12/12 通过 → 对比基线 results.json: 0 回归，12 项结果与基线一致。
💡 说明: 增量测试只执行受变更影响的测试项（通过分析 diff 中的文件路径与测试文件的 import/grep 关系裁剪），避免全量运行浪费时间。对比基线 results.json 检测退化 — 之前通过现在失败的测试 = 回归 = P0。
node skills/rui/audit.mjs --incremental📋
文档同步检查
确认文档变更与代码变更一致。如有代码无文档（缺口）或文档无代码（过时），标记 P1。
✅ 结果: rui-import mode=list — 本地文件 47 个 → 远端文件 47 个 → 待同步: 12 个 (8 演示.html + 2 SKILL.md + 1 知识图谱 + 1 测试面板) → 版本差: 本地 4.3.0 vs 远端 4.2.0 (本地领先) → 0 代码无文档缺口 / 0 文档无代码过时。
💡 说明: 文档同步检查验证代码变更与文档变更的一致性。两大致命问题: (1) 缺口 — 只改代码不改文档，后来的开发者无文档可读；(2) 过时 — 文档描述了已被删除的代码。两者都是 P1（不阻断但必须在交付前修复）。mode=list 仅列出差异不执行同步。
/rui-import mode=list📋
