步骤 1: 验证 vitest 框架安装与配置
安装依赖后执行 framework-presence 测试，验证 package.json、vitest.config.mjs、vitest 模块、适配层、node_modules gitignore、遗留保留等 19 项检查。
npm install
npx vitest run tests/infrastructure/framework-presence.test.mjs📋
💡 说明: framework-presence.test.mjs 检查 6 个维度：package.json (5项) + vitest.config.mjs (3项) + vitest 模块 (2项) + node_modules gitignore (1项) + 适配层 (4项) + 遗留保留 (4项)。全部为阻断级 —— 任一项失败则 vitest 不可用。
✅ 结果: 19/19 通过 — package.json 存在且合法，vitest.config.mjs 含 include/exclude 配置，vitest 模块可 import()，适配层 9 个断言方法可用，node_modules 已 gitignored，遗留测试基础设施完整。
步骤 2: 验证 CDN 三路源及 npm Registry 可达性
执行 third-party-reachability 测试，对 npm Registry（HEAD + vitest 包存在性）、CDN 三路源（unpkg/jsDelivr/esm.sh HEAD）、npm Search API 逐一做 HTTP HEAD 检查。
npx vitest run tests/infrastructure/third-party-reachability.test.mjs📋
💡 说明: 单项服务不可达标记为警告（不影响 core flow），全部服务不可达标记为阻断。每项含备选 CDN 源提示（如 jsDelivr 不可达时提示使用 unpkg 或 esm.sh）。当前三路源全部可达。
✅ 结果: 6/6 通过 — npm Registry (800ms)、vitest 包存在 (550ms)、unpkg (630ms)、jsDelivr (1585ms)、esm.sh (800ms)、npm Search API (710ms)。三路源响应时间在可接受范围。
步骤 3: 双轨独立运行验证
分别执行 vitest 工程化轨道和遗留自定义 harness 轨道，验证两个轨道可独立运行且互不影响。使用 npm run test:all 触发全量。
npm test && npm run test:legacy
# 或一次性全量:
npm run test:all📋
💡 说明: 双轨架构保证零风险迁移：vitest 仅管理 tests/infrastructure/ 测试（25 项），遗留运行器保持零依赖兜底（171+ 断言）。一个轨道失效不影响另一个。vitest 提供了更好的错误 diff、watch 模式、并行执行等工程化能力。
✅ 结果: vitest 轨道: 25/25 通过 (~7s) · 遗留轨道: node tests/run.mjs 正常执行 · 双轨兼容验证: ✅ · 零风险迁移: 遗留测试零变更。
