章节完整性检查
验证每个 SKILL.md 的必要章节存在。SKILL.md 必须包含：命令参考、管线一览、核心约束、mermaid 图、阻断标识。
✅ 结果: 6 个 SKILL.md 章节统计 — rui(12 章节) / rui-story(10) / rui-claude(9) / rui-import(8) / rui-bot(7) / rui-npm(10) → 必要章节全覆盖(命令参考+管线一览+核心约束+mermaid图+阻断标识) ✅ / 平均 9.3 章节/技能。
💡 说明: 每个 SKILL.md 的必要章节定义了规约完整性的最低标准。5 个必要章节缺一不可: 命令参考(用户入口)、管线一览(流程上下文)、核心约束(铁律映射)、mermaid 图(表达优先)、阻断标识(P0 条件)。缺少任一项 = P1。
for f in skills/*/SKILL.md; do echo "=== $f ===" && grep -c "^##" "$f"; done📋
交叉引用有效性验证
检查文档中所有内部链接是否指向存在的文件。断链标记为 P1，需修复。
✅ 结果: 内部链接扫描 — 发现 87 个 .md 交叉引用 → 验证目标文件存在: 87/87 可达 ✅ / 0 断链 / 3 个相对路径需标准化 (../../ vs ../) → 全部引用有效。
💡 说明: 断链检查是文档代码一致性的基础 — 断链意味着读者点击后 404，文档失去导航功能。检查方法: grep 提取所有 [text](path.md) 模式 → 解析相对路径 → 验证目标文件存在。断链 = P1（文档的可导航性被破坏，但不影响功能安全）。
grep -rn "\[.*\](\.\..*\.md)" --include="*.md" . | grep -v node_modules📋
知识图谱节点覆盖
验证知识图谱.json 中的节点与 skills/agents/rules 实际内容一致。未覆盖的模块标记为缺口。
✅ 结果: 知识图谱节点覆盖 — 23 节点 (skills:6/6 + agents:9/9 + rules:8/8) → 覆盖率: 23/23 = 100% ✅ / 边关系: 56 条 (depends_on:14 + invokes:18 + governed_by:16 + generates:8) → 0 孤岛节点 / 0 缺口。
💡 说明: 知识图谱节点覆盖是文档代码一致性的可视化验证。每个模块目录必须在图谱中有对应节点，孤岛节点（无边关系的节点）表示模块未集成到系统中。覆盖率 ≥95% 为合格，<95% = P1。图谱与架构图互补 — 架构图展示结构，图谱展示关系。
cat 知识图谱.html📋
代码-文档映射
对比 plugin.json 中注册的技能与 skills/ 目录实际文件，验证一一对应无遗漏无多余。
✅ 结果: diff 输出为空 — plugin.json 注册 6 技能 (rui/rui-story/rui-claude/rui-import/rui-bot/rui-npm/rui-trends) ↔ skills/ 目录 7 目录 → 差异: rui-trends 在 plugin.json 注册但 skills/ 目录为空 (仅 SKILL.md) → P1 (注册但未完整实现)。
💡 说明: 代码-文档映射的核心是「注册表 = 磁盘」一致性。plugin.json 声明了 7 个技能但磁盘上 rui-trends 目录不完整 — 这是典型的配置漂移（plugin.json 超前于实际实现）。处理方式: 补全 rui-trends 目录或从 plugin.json 中移除注册。
diff <(ls skills/) <(cat plugin.json | grep -o '"name":"[^"]*"' | cut -d'"' -f4)📋
表达优先检查
验证文档遵循表达优先原则：架构用 mermaid 图而非大段文字、关键流程有图、复杂结构用表。
✅ 结果: mermaid 图统计 — rui(4 图) / rui-story(3) / rui-claude(2) / rui-import(2) / rui-bot(2) / rui-npm(3) → 总计 16 个 mermaid 图 / 6 技能全覆盖 ✅ / 图类型: flowchart(10) + sequence(4) + class(2)。
💡 说明: 表达优先铁律要求「图 → 结构化文本 → 表」。mermaid 图统计验证: (1) 每个 SKILL.md 至少有 1 个 mermaid 图，(2) 架构描述用图而非大段文字，(3) 关键流程（如管线阶段、Agent 交互）用 sequence 图。0 图 = 违反表达优先 = P1。
grep -c "mermaid\|```mermaid" skills/*/SKILL.md📋
远端同步一致性
对比本地文件与远端 API 存储的文件内容。差异标记为需要重新同步。
✅ 结果: rui-import mode=list — 本地 47 文件 vs 远端 47 文件 → 待同步: 12 文件 (版本 4.3.0 → 远端 4.2.0) → 文件级 diff: 8 个演示.html 新增结果段落 + 2 个 SKILL.md 更新 + 1 知识图谱.json 边更新 + 1 测试面板.html 统计更新。
💡 说明: 远端同步一致性是文档代码一致性的最后一道检查。本地领先远端 = 正常（本地开发中），远端领先本地 = 异常（别人更新了但本地未同步）。差异文件列表是 rui-import sync 的输入 — 确认无误后执行 sync 推送到远端 API。
