计划概述

目标：建立 YrY 免疫系统——管线健康自检 + 文档基线完整性校验，在问题影响交付前发出警报。

架构：两大维度并行推进。Story 1 定义管线 6 项自检规则（分支隔离·关卡门禁·交接信号·证据链·P0 清零·交付收口）；Story 2 定义文档 6 项校验规则（目录·章节·引用·回溯链·知识图谱·版本一致性）。

技术栈：node.js 校验脚本 + Markdown + Mermaid + HTML/SVG 架构图 + JSON 知识图谱

风险与缓解

#风险可能性影响缓解策略
R1自检项定义与实际管线行为存在偏差导致漏检或误报 MH 每自检项从规约文档直接映射；security 独立审计自检逻辑
R2增量自检边界判定不准确导致全量重跑丧失性能优势 MM git diff 范围 + 依赖图传播分析确定最小变更面
R3自检脚本自身存在静默失败导致管线问题未被发现 MH 自检脚本的测试覆盖 + 定期人工抽查 + 自检结果交叉验证
R4自检开销过大导致开发者跳过自检流程 HH 增量模式优先；单次自检控制在合理时间内；渐进式修复不阻断

执行模式

Subagent-Driven（推荐）

独立 subagent 逐任务执行，每任务完成后触发 code-reviewer 审查。P0 清零再进下一任务。

适用：> 5 步骤 · 多文件并行

Inline 执行

当前会话逐步骤执行。每步验证后前进。

适用：≤ 5 步骤 · 简单场景
