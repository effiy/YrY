## §1 资产识别

### 1.1 数据资产

| 资产 | 敏感级别 | 存储位置 | 访问路径 |
|------|---------|---------|---------|
| API_X_TOKEN | 高 — 远端 API 访问凭据 | 环境变量 | process.env → sync.mjs |
| 文档文件路径 | 低 — 项目内部路径 | argv 参数 | CLI 参数 → parseArgs() |
| 文档文件内容 | 中 — 项目文档数据 | 文件系统 | existsSync 验证 → sync.mjs 读取 |

| import-doc.mjs CLI | 无（由管线调用） | 文件系统读取 |
| sync.mjs (间接调用) | API_X_TOKEN 环境变量 | 远端 API 写入 |
| stdout/stderr 输出 | 无 | 管线日志捕获 |

## §3 信任边界

| 边界 | 跨越方向 | 数据流 | 校验点 | 当前状态 |
|------|---------|--------|--------|---------|
| CLI 参数 → 文件系统 | 输入 | 文件路径字符串 | 管线传入（非用户直输），resolve() 规范化 | 已加固 |
| 文件系统 → sync.mjs | 输入 | 文件内容 | sync.mjs 内部验证文件格式 | 已加固 |
| 环境变量 → sync.mjs | 输入 | API_X_TOKEN | 环境变量由系统管理，不落盘 | 已加固 |
| sync.mjs → 远端 API | 输出 | HTTP 请求 | HTTPS 传输，API_X_TOKEN 认证 | 已加固 |
| sync.mjs → import-doc | 进程间 | exit code + stderr | exit code 检查 + stderr 模式匹配 | 已加固 |
| import-doc → 管线日志 | 输出 | stdout/stderr | 错误信息不包含 token | 已加固 |
