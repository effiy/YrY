## §2 测试用例

### 2.1 正常路径 (TC-N*)

| ID | Given | When | Then | 关联 FP | 优先级 |
|----|-------|------|------|---------|--------|
| TC-N1 | 有效文档文件存在于磁盘 | 执行 `node skills/rui/import-doc.mjs <valid-file>` | exit 0，stderr 含 "✓ imported"，远端可见该文档 | FP1, FP2 | P0 |
| TC-N2 | 无参数执行 | 执行 `node skills/rui/import-doc.mjs` | exit 0，stdout 含帮助文本（用法+选项+行为） | FP4 | P0 |

### 2.2 边界用例 (TC-B*)

| ID | Given | When | Then | 关联 FP | 优先级 |
|----|-------|------|------|---------|--------|
| TC-B1 | 文件名含空格 | 执行 `node skills/rui/import-doc.mjs "path/with spaces/file.md"` | 路径正确解析，导入成功或文件不存在错误 | FP1 | P1 |
| TC-B2 | 传入 --help 参数 | 执行 `node skills/rui/import-doc.mjs --help` | exit 0，输出帮助文本 | FP4 | P1 |

### 2.3 异常用例 (TC-E*)

| ID | Given | When | Then | 关联 FP | 优先级 |
|----|-------|------|------|---------|--------|
| TC-E1 | 不存在的文件路径 | 执行 `node skills/rui/import-doc.mjs /nonexistent/file.md` | exit 0，stderr 含 "file not found" | FP1, AC2 | P0 |
| TC-E2 | API_X_TOKEN 未设置 | 设置有效文件，清除 API_X_TOKEN 环境变量 | exit 0，stderr 含 "no-token — skipped" | FP2, AC3 | P0 |
| TC-E3 | --json 模式下文件不存在 | 执行 `node skills/rui/import-doc.mjs /nonexistent --json` | exit 0，stdout JSON `{"status":"error","reason":"file-not-found"}` | FP5 | P1 |

### 2.4 回归用例 (TC-R*)

| ID | Given | When | Then | 关联 FP | 优先级 |
|----|-------|------|------|---------|--------|
| TC-R1 | 有效文档文件 | 正常导入流程 | 不修改源文件内容，不产生副作用文件，exit code 始终为 0 | FP1, FP2, FP3 | P0 |
