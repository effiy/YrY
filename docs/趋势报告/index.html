扫描机制：Cron 每周一早 9:00 触发 rui-trends all → 并行抓取四源 → 分类打标 (AI/LLM · Web/前端 · 后端/基础设施 · DevOps · 编程语言 · 安全) → 生成独立 HTML 报告 + 聚合视图 → 更新趋势清单
诊断关联：过时趋势信号触发 D5 外部趋势诊断 · 连续 2 周无新趋势信号 → P1 严重级 · 3 周以上信号缺失 → 自动升级 P0 · 与 D4 (依赖过期) 联动评估替代方案
趋势判定：对比连续两次扫描条目数的变化率 — ↑ 上升 (增长 >10%) · ↓ 下降 (减少 >10%) · → 持平 (变化 ≤10%) · 单个源偏差 >25% 触发异常检测
聚合权重：GitHub Trending 35% (实时焦点) · TrendShift 25% (迁移信号) · OSS Insight 20% (深度分析) · Top-Starred 20% (社区投票) — 加权合成整体趋势方向
投影机制：API 不可达时，基于前 3 周期历史数据外推投影，偏差范围 ±2~±11 条目。投影数据在报告中以 📡 标记，下次可达后以实际数据替换。
