💡 演示 1 · 最小页面骨架加载

在空 HTML 中按规范引入 5 类资源，观察 Network 面板的加载时序：

<link rel="preconnect" href="https://unpkg.com" crossorigin>
<link rel="dns-prefetch" href="https://unpkg.com">
<link rel="preload" href="https://unpkg.com/vue@3/dist/vue.global.prod.js"
      as="script" crossorigin fetchpriority="high">
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/yry-cdn@1.2.0/shared/index.css">
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/yry-cdn@1.2.0/theme/index.css">

预期：5 步顺序加载，FCP ≤ 310ms，TTI ≤ 480ms。

💡 演示 2 · 字体预加载与 FOUT 消除

对比有无 preload 时字体的渲染表现：

/* 有 preload — FOUT 不可见 */
<link rel="preload" as="font" type="font/woff2" crossorigin
      href="https://cdn.jsdelivr.net/npm/yry-cdn@1.2.0/fonts/JetBrainsMono-Regular.woff2">

/* 无 preload — 先系统字体回退，woff2 到达后切换 (FOUT 可见) */

验证：用 Chrome DevTools → Performance → filmstrip 检查首 5 帧字体渲染。

💡 演示 3 · CDN 回退与降级

在 DevTools → Network → Block request URL 中屏蔽 cdn.jsdelivr.net，观察回退：

  1. 主 CDN 不可达 → <script onerror> 触发 → 切换到 unpkg
  2. unpkg 也不可达 → 自托管 /cdn-fallback/ 兜底
  3. 全部失败 → 静态 HTML 渲染 (内容可读，交互不可用)
