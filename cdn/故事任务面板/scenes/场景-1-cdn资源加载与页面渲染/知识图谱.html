🔗 知识图谱 · 资源提示与优先级层级

preconnect (建立 TLS 提前) ──┐
                           │ 高优先级
preload (as=script) ───────┤
                           │
modulepreload (ESM 模块) ──┘

dns-prefetch (仅 DNS) ─────── 低优先级
prefetch (空闲加载下一页) ──── 低优先级

资源提示节点构成一棵优先级树: 高优先级分支 (preconnect/preload/modulepreload) 覆盖当前页关键资源，低优先级分支 (dns-prefetch/prefetch) 处理下页或非关键源。

🌐 资源提示语义表

提示时机优先级浏览器支持
preconnect关键第三方源最高95%+
dns-prefetch非关键第三方源中95%+
preload当前页关键资源最高95%+
prefetch下一页可能资源低90%+
modulepreloadESM 模块最高90%+

🛡️ SRI 与 CDN 回退链

<script src="https://cdn.jsdelivr.net/npm/yry-cdn@1.2.0/shared/index.js"
        integrity="sha384-<base64-hash>"
        crossorigin="anonymous"></script>
<script>
  if (!window.YrY) {
    document.write('<script src="/cdn-fallback/shared/index.js"><\/script>');
  }
</script>
  • SRI 哈希生成 — openssl dgst -sha384 -binary file.js | openssl base64 -A
  • 双 CDN 回退 — jsDelivr → unpkg → 自托管 (通过 <script onerror>)
  • CSP 规则 — script-src 'self' jsdelivr.net unpkg.com
  • 供应链监控 — Dependabot · npm audit · Socket (每周自动 PR)
