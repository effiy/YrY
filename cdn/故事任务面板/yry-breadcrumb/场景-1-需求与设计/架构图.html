架构洞察: 这是一个 fan-out / fan-in 模式 — 设计决策 (DEC) 作为扇出点分叉为三条并行设计路径，三条路径各自独立推进互不阻塞，最终在 Props API 草案 (API) 处汇聚收敛。这种模式体现了关注点分离原则：物理结构 (3 文件拆分)、加载策略 (异步加载)、可访问性 (a11y) 在设计阶段即可并行决策，无需相互等待。

说明

上图展示场景 1 · 需求与设计 的关键决策流：从需求来源出发，经过设计决策阶段，分叉为三条并行路径（3 文件拆分、异步加载、a11y），最终汇聚到 Props API 草案。

其中三个分支各自独立推进，互不阻塞：3 文件拆分确定组件的物理组织结构（HTML/JS/CSS），异步加载确定模板获取策略（fetch + DOMParser），a11y确定可访问性标注方案（aria-current + aria-hidden）。三条路径在 Props API 处汇合，共同定义组件的对外接口。

颜色 #22d3ee 为本场景主色，与场景 tag 和侧边栏指示一致。
