关键步骤

梳理需求来源 ~15 min
从项目结构和使用场景出发，分析面包屑组件的实际需求：3 种渲染模式（含链接、纯文本、长路径）、异步加载、键盘可达性。
✅ 产出: 需求来源文档，明确 3 种 items 配置模式
确定 3 文件拆分方案 ~10 min
将组件拆分为 index.html（Vue 模板）、index.css（样式 + 设计令牌）、index.js（Loader 自举）三个独立文件，支持 CDN 直接引用。
✅ 产出: 3 文件架构图，明确各文件职责
设计 Props API ~20 min
定义 items 数组结构（label 必填、href/icon 可选）、ariaLabel 可访问性属性，确保 API 简洁且覆盖所有使用场景。
✅ 产出: Props API 草案，含类型定义与示例
设计评审通过 ~10 min
与团队评审 Props API 设计，确认 3 文件拆分方案可行，异步加载策略合理，a11y 覆盖完整。
✅ 产出: 评审通过的 API 草案，进入场景 2 模板与样式
🚀
下一步: 场景 2 · 模板与样式

自测

1 YryBreadcrumb 采用 3 文件拆分方案，这 3 个文件分别是？
Aindex.html + index.css + index.js
Btemplate.html + style.css + main.js
Capp.vue + app.css + app.js
✓ 正确! 组件拆分为 index.html（Vue 模板）、index.css（样式+设计令牌）、index.js（Loader IIFE 自举）三个独立文件，支持 CDN 直接引用。
2 Props API 中 items 数组的每个元素包含哪些字段？
Aname / url / emoji
Blabel(必填) + href(可选) + icon(可选)
Ctitle / link / image
✓ 正确! label 为必填字段显示文本，href 可选提供链接跳转，icon 可选提供 emoji 图标装饰。
3 面包屑组件的 a11y 无障碍标记使用了哪些 ARIA 属性？
Arole="link" + aria-hidden="true"
Btabindex="0" + role="navigation"
Caria-label + aria-current="page" + aria-hidden
✓ 正确! nav 元素使用 aria-label 描述导航用途，当前页使用 aria-current="page" 标记，分隔符使用 aria-hidden="true" 对屏幕阅读器隐藏。
