图: Loader 完整时序 — 从 index.js 加载到 Vue 挂载的 11 步消息交换
架构洞察: 这是一个 异步编排 (Async Orchestrator) 模式 — index.js 作为中央编排器，依次执行：自定位 (currentScript) → 超时保护 (setTimeout 5s) → 网络请求 (fetch) → HTML 解析 (DOMParser) → 组件注册 (window.YryBreadcrumb) → 事件派发 (dispatchEvent) → Vue 挂载。关键设计特征：5s 超时熔断防止无限等待、IIFE 自执行保证加载即注册、自定义事件解耦（页面与 Loader 通过 yry-breadcrumb-ready 事件通信）、fetch same-origin 安全策略。

概念视图

🌐
浏览器
入口参与者
通过 <script src="index.js"> 触发整个加载链。IIFE 格式保证脚本加载后立即自执行，无需外部调用。
→ index.js
⚙️
index.js 编排器
中央编排者
Loader 核心：document.currentScript 自定位 → setTimeout 5s 超时保护 → fetch 模板 → DOMParser 解析 → getElementById 提取模板 → window.YryBreadcrumb 注册 → dispatchEvent 派发 ready。
← 浏览器 → Network · DOMParser · Vue
📡
Network + DOMParser
数据获取与解析
fetch(index.html) 以 same-origin 凭证获取模板 HTML 文本；DOMParser.parseFromString 将文本转为 DOM Document；getElementById('yry-breadcrumb-tpl') 提取模板节点。每步均有错误捕获。
← index.js → 组件注册
🖥️
Vue 挂载出口
最终参与者
接收 yry-breadcrumb-ready 事件后，Vue.createApp(YryBreadcrumb, props).mount('#breadcrumb-app') 将组件渲染到 DOM。props 包含 ariaLabel 和 items[{label,href,icon}]。
← index.js (ready event)
架构完整性: 场景 3 的 Loader 时序展现了 单一编排者 模式：index.js 是唯一的主动参与者，其他参与者 (Network/DOMParser/Vue) 均为被动响应。每个步骤都有明确的错误处理路径（超时 → console.error / fetch 失败 → catch 不注册 / 解析失败 → 异常捕获 / id 缺失 → null 检查）。时序图中的双向消息 (实线箭头 = 同步调用，虚线箭头 = 异步返回) 清晰区分了阻塞与非阻塞操作。
