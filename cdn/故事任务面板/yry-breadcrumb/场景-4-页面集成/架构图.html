架构洞察: 这是一个 多阶段管道 (Multi-Stage Pipeline) 模式 — 页面集成分 4 个顺序阶段：Stage 1 (CSS 加载防 FOUC) → Stage 2 (Vue 3 运行时就绪) → Stage 3 (Loader 注册组件 + 派发 ready 事件) → Stage 4 (mount 脚本创建 Vue app 并挂载)。关键设计特征：4 文件固定加载顺序（CSS → Vue → Loader → mount 不可重排）、事件驱动解耦（yry-breadcrumb-ready 自定义事件异步通知页面挂载时机）、color-coded 节点分类（7 种类别通过 classDef 着色：page/css/js/dom/comp/mount/ui）。

概念视图

📄
Page HTML
入口节点
所有场景页面的 HTML 入口，通过 <head> 中的 link/script 标签声明对 CSS、Vue 运行时的依赖，通过 <body> 中的 #breadcrumb-app 占位符预留挂载点。
→ CSS · Vue · Loader
📦
三文件依赖链
加载阶段
CSS (index.css) 先加载防 FOUC；Vue 3 (vue.global.prod.js) 提供运行时；Loader (index.js) 注册组件并派发 ready 事件。三者按序加载，CSS 和 Vue 在 head 中，Loader 在 body 底部。
← Page HTML → DOM Ready · Ready Event
🔗
事件驱动挂载
编排层
yry-breadcrumb-ready 自定义事件作为 Loader 与页面的解耦信号。页面监听此事件 (once: true)，收到后执行 mountBreadcrumb() → Vue.createApp(props).mount('#breadcrumb-app')。
← Loader (ready event) → nav.breadcrumb
🍞
nav.breadcrumb 渲染
出口节点
Vue 3 模板引擎将 props.items 渲染为可交互的 nav.breadcrumb DOM，包含正确的 href 链接、aria-current 标注、CSS 过渡动画，支持 4 种 items 模式 (href/纯文本/长路径/图标)。
← Vue.createApp mount
架构完整性: 场景 4 的页面集成流程展现了 声明式依赖 + 事件驱动挂载 的双重解耦：文件依赖通过 HTML 标签声明（无 JS 模块系统依赖），组件挂载通过自定义事件触发（无轮询检查）。4 文件固定顺序保证 CSS 先于 DOM 渲染、Vue 先于组件注册、Loader 先于挂载脚本，形成不可逆的单向管道。7 种节点颜色分类 (page/css/js/dom/comp/mount/ui) 使架构图中的关注点一目了然。

📐 4 文件加载顺序

  1. <link rel="stylesheet" href="yry-breadcrumb/index.css"> — CSS 先加载防 FOUC
  2. <script src="vue.global.prod.js"> — Vue 3 运行时
  3. <script src="yry-breadcrumb/index.js"> — Loader 注册组件 + 派发 ready
  4. mountBreadcrumb() 内联脚本 — 创建 Vue app + 挂载
