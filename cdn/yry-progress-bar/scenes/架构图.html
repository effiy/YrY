双模式判定架构

flowchart TB
    MOUNT["mounted()"] --> CHECK{"el.hasAttribute('done')
|| el.hasAttribute('total')"}
    CHECK -->|"是 → 显式模式"| EXPLICIT["_scrollMode = false
computed pct = done/total*100
无 listener 绑定"]
    CHECK -->|"否 → 自动模式"| SCROLL["_scrollMode = true
绑定 scroll + resize
passive: true"]
    SCROLL --> INIT["$nextTick → _updateScroll()"]
    INIT --> RAF["RAF 节流
_ticking flag 防重入"]
    RAF --> CALC["计算: scrollY / (scrollH - innerH)"]
    CALC --> COMPUTED["computed pct 响应式更新"]
    EXPLICIT --> COMPUTED
    COMPUTED --> RENDER["width + class(is-done/is-empty) 渲染"]

加载时序架构

sequenceDiagram
    participant Page as 页面
    participant Vue3 as Vue 3 运行时
    participant Loader as vue-ce-loader.js
    participant JS as index.js
    participant Network as 网络
    participant CE as CustomElements

    Page->>Vue3: <script src="vue.global.prod.js">
    Vue3-->>Page: window.Vue 就绪
    Page->>Loader: <script src="vue-ce-loader.js">
    Loader-->>Page: window.YrYVueCE 就绪 (幂等)
    Page->>JS: <script src="index.js">
    JS->>Loader: YrYVueCE.define({componentName, templateId, buildComponent})
    Loader->>Network: fetch('index.html')
    Network-->>Loader: HTML 文本
    Loader->>Loader: DOMParser → getElementById(templateId)
    Loader->>Loader: buildComponent(templateHTML) → 组件定义
    Loader->>Vue3: Vue.defineCustomElement(component)
    Vue3-->>Loader: Custom Element 构造器
    Loader->>CE: customElements.define('yry-progress-bar', ctor)
    Loader->>Page: dispatchEvent('yry-progress-bar-ready')
    Loader->>Page: window.YryProgressBar = component

降级架构

层级失败点降级行为用户影响
L1vue-ce-loader 未加载console.warn + return组件不注册 · 无阻塞
L2Vue 3 未加载console.error (loader 层)组件不注册
L3fetch 超时 (5s)console.error + 不注册组件不渲染
L4模板 id 缺失throw → catch → console.error组件不注册
L5网络失败 (4xx/5xx).catch → console.error组件不注册
CSSbackdrop-filter 不支持不透明深色背景无磨砂效果
CSSprefers-reduced-motion动画/发光停止降低视觉负担
