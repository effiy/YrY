YryProgressBar · 场景 4/5 · 判定流程 · RAF 节流

双模式判定与执行

flowchart TB
    MOUNT["mounted()"] --> CHECK{"el.hasAttribute('done')
|| el.hasAttribute('total')"}
    CHECK -->|"是"| STATIC["显式模式: _scrollMode=false
pct = done/total*100"]
    CHECK -->|"否"| SCROLL["自动模式: _scrollMode=true
绑定 scroll+resize (passive)"]
    SCROLL --> INIT["_updateScroll() 初始调用"]
    INIT --> RAF["RAF 节流: _ticking flag
同一帧合并多次事件"]
    RAF --> CALC["计算: done=scrollY
total=max(1, scrollH-innerH)"]
    CALC --> COMPUTED["computed pct 重新计算"]
    STATIC --> COMPUTED
    COMPUTED --> RENDER["渲染: width=pct+'%'"]

RAF 节流时序

sequenceDiagram
    participant Scroll as scroll 事件
    participant Handler as _onScroll
    participant RAF as requestAnimationFrame
    participant Update as _updateScroll
    Scroll->>Handler: scroll (60+ Hz)
    Handler->>Handler: _ticking? → true → return
    Handler->>RAF: _ticking=true → rAF(cb)
    Scroll->>Handler: scroll (合并)
    Handler->>Handler: _ticking? → true → return
    RAF->>Update: 下一帧回调
    Update->>Update: 计算 scrollY / docH
    Update->>Update: _ticking = false
    Scroll->>Handler: 下次 scroll → 重复
