批量生成引擎连接前四层的所有能力——模板→提取→组件→可视化，形成从 markdown 源到 HTML 产出的完整自动化管线。自循环机制确保 markdown 变更后 HTML 自动更新。

sequenceDiagram
    participant A as 实例 A
    participant FS as 文件系统
    participant B as 实例 B
    A->>FS: 检查 .lock
    FS-->>A: 不存在
    A->>FS: 写入 .lock (PID)
    A->>FS: 写入 HTML.tmp
    A->>FS: rename HTML.tmp → HTML
    A->>FS: 删除 .lock
    B->>FS: 检查 .lock
    FS-->>B: 存在 (PID=A)
    B->>B: 等待 1s 重试
    B->>FS: 检查 .lock
    FS-->>B: 不存在
    B->>FS: 写入 .lock (PID=B)

锁策略实现优劣超时
文件锁 .lockflock 或 PID 写入简单 · 跨进程30s
内存锁 Mutex单进程极快 · 不跨进程—
数据库锁事务隔离强一致 · 重—
无锁 + CAS原子 rename并发高 · 复杂—

推荐组合：文件锁 + 原子 rename，生成完成后立即释放锁。
