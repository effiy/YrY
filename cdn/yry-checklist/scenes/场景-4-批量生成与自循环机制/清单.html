① 5 个管线阶段清单

1扫描 — 输入故事目录路径 · 递归扫描 场景-N-*/ 子目录 · 输出场景列表 · < 50ms
2提取 — 输入每场景 index.md · 解析 §0-§4 章节 + 表格 + mermaid · 输出 Context 对象 · < 100ms/场景
3渲染 — 输入 Context + 模板 · Token 替换 + 模板渲染 · 输出 HTML 字符串 · < 50ms/文档
4写入 — 输入 HTML 字符串 · 写入文件系统 + .bak 备份 · 输出 7 个 HTML 文件 · < 100ms/场景
5报告 — 输入生成结果 · 统计 created/updated/skipped · 输出摘要报告 · < 10ms

② 自循环检测逻辑

每 30 分钟:
  1. 扫描所有故事目录的场景-N-*/index.md
  2. 对比 .md 与 .html 的 mtime
  3. .md > .html → 重新渲染该场景
  4. 记录 created/updated/skipped 统计
  5. 输出日志 + 企微通知 (可选)
1mtime 对比 — 仅 .md > .html 时重渲染 · 避免无变更时的空跑
2.bak 备份 — 覆盖前先备份原 HTML · 异常时可回滚
3失败隔离 — 单场景渲染失败不阻塞其他场景
4日志 + 通知 — 每次自循环输出统计 · 可选企微 Webhook

③ 命令接口

命令用途触发

node scripts/yry-checklist-loop.mjs自循环守护进程 (30min)cron/systemd
/rui-html --scene 场景-N单场景生成手动

④ 角色链与门禁

1决策链 — architect (管线设计) · reviewer (方案评审) · architect (门禁策略)
2实现链 — coder (5 阶段实现) · tester (TC 覆盖) · coder (自循环守护)
3闭环链 — deliverer (签收) · self-improve (E1-E4 评估)
4门禁 — html-gen-gate · 7 文档全部生成 · 0 错误 · .bak 备份齐全
