加载时序说明

Loader 在页面中的加载时序如下。以下为静态文档说明，非实时模拟。

时间点事件控制台输出状态
T0浏览器解析到 <script src="index.js">—开始加载
T0 + ~10msIIFE 自执行，检测 window.Vue[YryBreadcrumb] Vue 3 未加载 (若缺失)检测中
T0 + ~50msfetch index.html 发出请求—网络请求中
T0 + ~200msindex.html 返回，DOMParser 解析—解析中
T0 + ~210mswindow.YryBreadcrumb 被赋值—组件就绪
T0 + ~211msdispatchEvent('yry-breadcrumb-ready')—事件派发
T0 + ~215msVue.createApp + mount—渲染完成
T0 + 5000ms超时触发 (仅网络失败时)[YryBreadcrumb] 模板加载超时（5s）降级

事件流说明

Loader 使用 CustomEvent 实现页面与组件的解耦通信。

JS
// Loader 侧 — 组件就绪后派发
document.dispatchEvent(new CustomEvent('yry-breadcrumb-ready'));

// 页面侧 — 使用 {once:true} 确保只挂载一次
if (window.YryBreadcrumb) {
  mountBreadcrumb();                      // 路径 A：已就绪，直接挂载
} else {
  document.addEventListener(              // 路径 B：等待 ready 事件
    'yry-breadcrumb-ready',
    mountBreadcrumb,
    { once: true }                        // 防止重复触发
  );
}

关键步骤

确定异步加载策略 ~15 min
分析 CDN 场景下的加载需求：组件必须异步自举，不阻塞页面渲染。选择 IIFE + fetch 方案，确保在 Vue 3 CDN 加载后自动注册。
✅ 产出: 异步加载策略文档，IIFE 自举框架
实现 fetch + DOMParser 模板提取 ~25 min
通过 document.currentScript 解析组件路径，fetch index.html 获取模板，DOMParser 解析并提取 #yry-breadcrumb-tpl 模板内容。
✅ 产出: Loader 核心代码，模板提取逻辑
设计 CustomEvent 事件通信 ~20 min
定义 yry-breadcrumb-ready 自定义事件，组件就绪后 dispatchEvent 通知页面。页面使用 {once:true} 监听，确保单次挂载，防止并发冲突。
✅ 产出: 事件通信协议，解耦的页面挂载模式
5s 超时保护与错误降级 ~15 min
实现 setTimeout 5s 超时保护，catch 网络错误后 console.warn/error 诊断输出。所有错误场景下页面正常渲染，不抛未捕获异常。
✅ 产出: 超时 + 降级机制，5 种错误场景覆盖
