① 5 步 Loader 生命周期

1初始化 — 检测 Vue 3 全局变量 · 失败时 console.warn 降级退出
2异步加载 — fetch index.html · 5s 超时保护 (setTimeout) · 失败时 console.error 降级
3解析模板 — DOMParser().parseFromString + getElementById('yry-breadcrumb-tpl') · 模板不存在抛错
4注册组件 — window.YryBreadcrumb 构造器 · customElements.define('yry-breadcrumb', ctor)
5派发事件 — document.dispatchEvent(new CustomEvent('yry-breadcrumb-ready'))

② 3 个代码块结构

1代码块 1 — IIFE 加载器框架 · 'use strict' · 检测 window.Vue · 解析 script.src + templateUrl
2代码块 2 — fetch + DOMParser 核心 · TIMEOUT_MS = 5000 · setTimeout 超时 · fetch → text() → DOMParser → getElementById
3代码块 3 — CustomEvent 派发 + 错误处理 · onError(err) · clearTimeout(timer) · console.error · 成功路径赋值 window.YryBreadcrumb

③ 4 个降级场景清单

1Vue 缺失 — window.Vue 未定义 · 降级：console.warn('[YryBreadcrumb] Vue 3 未加载') · return 退出
2网络失败 — fetch reject · 降级：onError(err) · console.error · 组件不注册
3模板 id 缺失 — getElementById 返回 null · 降级：throw new Error('template #yry-breadcrumb-tpl not found')
4超时 — 5s 内无响应 · 降级：onError(new Error('timeout')) · fetch 未取消但逻辑上视为失败

④ 就绪事件消费方

// 消费方监听 yry-breadcrumb-ready 事件
document.addEventListener('yry-breadcrumb-ready', function () {
  var bc = document.getElementById('breadcrumb-app');
  if (bc) {
    bc.items = [
      { label: '首页', href: '/' },
      { label: '当前页' }   // 末项无 href, 渲染为 aria-current="page"
    ];
  }
});

双设策略：消费方既可在 DOMContentLoaded 时立即赋值 (元素已在 DOM)，也可监听 ready 事件二次赋值 (覆盖组件晚于元素出现的场景)。
