② 6 项设计决策 (ADR 风格)

1框架 → Vue 3 (全局变量) · 替代：Vanilla JS / Web Component · 理由：项目已用 Vue 3，生态一致 · R1
2文件结构 → 3 文件拆分 (index.html/js/css) · 替代：单 .js 混合 · 理由：设计师/前端/样式各自修改不干扰 · R2
3加载方式 → 异步 fetch + DOMParser · 替代：同步 XHR / 打包 · 理由：零打包，模板源可在浏览器预览 · R4
4样式 → 设计令牌 var(--yry-color-*, #fallback) · 替代：hardcoded · 理由：跨主题可移植，无主题时降级 · R5
5a11y → aria-current="page" + aria-hidden · 替代：仅视觉 · 理由：屏幕阅读器友好 · R3
6自定义元素 → defineCustomElement · 替代：全局组件 · 理由：可在任何页面直接使用 <yry-breadcrumb> · R1
