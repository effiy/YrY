# YrY

> 插件/配置系统开发，规则完整性与集成契约。

- **项目类型** — 元项目(插件/配置)（single）
- **技术栈** — 元项目(插件/配置)
- **生态** — 未识别

## 快速开始

```bash
/rui init                    # 建立项目基线（README.md + 故事面板目录）
/rui doc "需求描述"           # 拆需求为故事
/rui code <story-name>       # 实现故事
/rui                         # 任务推荐
```

## 项目结构

| 目录/文件 | 职责 | 生成方式 |
|-----------|------|---------|
| `README.md` | 系统视图 + 项目画像 | rui init 全量重生 |
| `docs/故事任务面板/` | 故事产出（每故事独立目录） | rui doc/code 生成 |

### 故事目录骨架（fullstack）

```
docs/故事任务面板/<Project>/<name>/
├── 01-故事任务.md              ← 唯一真相源（pm）
├── 02-后端技术评审.md          ← coder + security
├── 04-测试用例评审.md          ← tester
├── 05-后端实施报告.md          ← 验证阶段
├── 07-测试用例报告.md          ← tester
├── 08-自改进复盘.md            ← pm + reporter
├── 00-消息通知列表.md          ← 自动（hook）
├── .memory/                    ← 管线状态 + 执行记忆
└── .improvement/               ← 自改进提案
```
