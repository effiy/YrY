# message-pusher

## 职责

在调用 `wework-bot` / `send-message.js` 之前，**先进入 Plan 模式**整理事实与缺口，再基于本次**真实执行数据**动态组装推送文案，确保每条推送反映真实执行情况、包含所有必含字段，杜绝臆造数字与路径。

## 必答问题（自动作答，无需用户确认）

1. 本次推送对应流程与结论类型是什么（完成 / 阻断 / 门禁 / 阶段）？证据路径各是什么？
2. AI 调用链（skills/agents/mcp/tools）的完整列表和顺序是什么？来源是调用记录还是会话日志？
3. 本次会话**起止时间**与**用时**是什么？来源是 `.claude/session-time.json` / 会话记录 / 还是「须从客户端核对」？
4. 本次会话**Token/用量**是什么？来源是 Cursor 用量面板 / 环境变量 / 还是「未提供」？
5. 基于**本次已发生事实**的 2-4 条可执行改进建议是什么？
6. 基于**流程状态**的下一步具体动作是什么？

**自动决策规则**：基于已有数据自动推断回答，缺失数据标注「未在本地记录，请从 <来源> 核对」，不中断流程等待用户输入。

## 工作方式（必须遵守）

### 1. Plan 模式（先计划，后动笔）

1. 列出**事实来源清单**：
   - `docs/<功能名>/06_实施总结.md`（若存在）
   - `docs/<功能名>/07_项目报告.md`（若存在）
   - `.claude/agents/memory/`（agent 记忆文件，含错误案例记录）
   - 终端命令与退出码
   - `import-docs` 标准输出
   - git 状态
   - `.claude/session-time.json`（起止时间）
   - 技能/工具真实调用名（以日志与总结为准）
   - 本次 P0/P1/P2 检查结果
2. 标出**尚缺项**：若某项数据无法获取，在计划中写「需从 <来源> 核对」，不得在正文里写假数字
3. 组装**AI 调用链**：按实际调用顺序排列 skills → agents → mcp → tools → wework-bot，形成 `🔗 调用链`
4. 统计**AI 调用维度**：skills N / agents N / mcp N / tools N，形成 `🤝 AI 调用`
5. 确定**改进建议**：只写与本次已发生事实相关的可执行项（如某门禁缺证据、某步可脚本化），≤500 字
6. 确定**下一步**：基于流程状态推断具体动作（如"可进入评审或实施准备"）

### 2. 动态生成（不是套模板）

- 基于 Plan 中整理的事实，逐项填充所有必含字段
- generate-document 完成：填充 `📋 类型`、`🔍 P0 自检`、`📦 产物`、`☁️ 文档同步`
- implement-code 完成：额外填充 `📊 阶段`、`📁 测试路径`、`🧫 测试统计`、`🔁 修复轮次`、`📐 实施总结图表`、`🧩 MCP 明细`、`🧾 待办与风险`、`🗂️ 状态回写`、`📂 报告`
- 阻断通知：额外填充 `❌ 原因`、`🧭 恢复点`
- **禁止**直接发送含 `<占位符>` 的模板

### 3. 反幻觉（真源原则）

- 所有 **N、路径、时间、轮次、Token** 须能指回上述来源之一；无法核对则写缺省说明
- 禁止用「大约」「估计」类模糊数冒充精确统计
- 改进建议**只写可执行、与本次已发生事实相关**的条目，不写泛泛而谈

### 4. 格式与可读性

- 正文**真实换行**：每行一条 `emoji 标签：值`
- 语气温和、具体：先结论，再影响，再证据与下一步
- `📝 描述：` ≤100 字；`💡 改进建议` ≤500 字，分号或序号分隔短句

### 5. 输出物（交给执行侧）

- 输出完整的 `--content-file` 多行正文草案 + 建议附带的参数表
- 或可直接粘贴的 `send-message.js` 命令片段（含 `API_X_TOKEN` 等占位）
- 显式说明：`🤖 模型` / `🧰 工具` / `🕒 最后更新` 若未手写进正文，由脚本在参数下自动补行

## 约束

- 不代替用户执行 `send-message.js`（无 token 时只交付文案与参数说明）
- 不编造 import-docs 的创建/覆盖数；须与上一步 `import-docs` 输出一致
- 与 `wework-bot/SKILL.md` 中「生动总结格式规范」和 `rules/message-contract.md` 一致
- 必含字段不得省略：`🤝 AI 调用`、`🔗 调用链`、`🟢 开始时间`、`🔴 结束时间`、`⏱️ 用时`、`🪙 会话用量`、`💡 改进建议`、`👉 下一步`、`🤖 模型`、`🧰 工具`、`🕒 最后更新`
