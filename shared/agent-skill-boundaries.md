# Skill 与 Agent 边界

本文档用于统一 `.claude/skills/` 与 `.claude/agents/` 的职责边界，避免同一能力在两个入口里重复承载完整流程。

## 总原则

- **Skill** 负责方法、流程、输入输出格式和调用规则。
- **Agent** 负责角色定位、并行分工、必答问题和候选结论。
- **Skill 决定怎么做**，**Agent 负责从某个专业视角补充判断**。

## 常见配对

| 能力主题 | Skill 职责 | Agent 职责 |
|----------|-----------|-----------|
| `code-review` / `code-reviewer` | 定义代码审查输入、维度与输出格式 | 作为代码审查专家给出候选问题列表 |
| `e2e-testing` / `e2e-tester` | 定义 E2E 测试方案的生成方法和产物结构 | 作为 E2E 专家回答场景级测试问题 |
| `find-skills` | 发现当前仓库内真实存在的 skill | 不适用 |
| `find-agents` | 发现当前仓库内真实存在的 agent，并给出组合建议 | 不适用 |
| `generate-document` | 负责文档生成主流程与规则装配 | 可按场景调用多个 agent 作为候选输入 |
| `implement-code` | 负责代码实施主流程与门禁规则 | 可按场景调用多个 agent 作为候选输入 |

## generate-document 代理清单

| Agent | 阶段 | 职责 |
|-------|------|------|
| `docs-retriever` | 1 | 规范检索：根据文档类型和关键词返回适用的 rules/、shared/、checklists/ 规范集合 |
| `doc-impact-analyzer` | 2 | 影响链分析：全项目搜索与依赖闭合分析，输出四表（搜索词清单、影响链、闭合摘要、未覆盖风险），覆盖代码+文档双面影响 |
| `doc-architect` | 3 | 文档架构设计：回答 5 个必答问题，输出与项目约定一致的模块划分、接口规范、数据流方案 |
| `codes-builder` | 3 | 代码架构与构建规划：输出可验证、可回滚、可构建的决策记录，含文件清单+构建顺序+验收标准 |
| `docs-lookup` | 2-3 | 文档歧义定位：按需检索历史文档结构与表达 |
| `doc-mermaid-expert` | 4 | Mermaid 语法审查与修复：对含图文档做语法校验和修复建议 |
| `doc-reviewer` | 4 | 文档质量审查：审查所有文档类型的结构完整性、规范符合性、可读性、跨文档一致性 |
| `doc-quality-tracker` | 4 | 质量统计与分析：统计 P0/P1/P2 数据，识别质量趋势、薄弱维度，输出可操作建议 |
| `docs-builder` | 5 | 知识策展：整理本次任务的可复用知识，消费 reporter agents 输出，沉淀跨工作流模式 |

## implement-code 代理清单

| Agent | 阶段 | 职责 |
|-------|------|------|
| `codes-retriever` | 1 | 代码上下文检索：提供准确、完整的代码信息供下游 agent 消费 |
| `doc-impact-analyzer` | 1/3/4 | 文档影响分析：追踪代码变更对关联文档的影响（反向依赖、交叉引用、示例代码时效） |
| `code-impact-analyzer` | 1/3/4 | 代码影响分析：追踪代码变更的完整影响链（类型、测试、构建配置） |
| `codes-builder` | 1 | 架构确认：验证实施策略与项目架构约定一致 |
| `code-reviewer` | 2/3 | 代码审查：拦截会变成线上事故的缺陷，P0 必须修复 |
| `doc-quality-tracker` | 4 | 质量统计与分析：统计代码审查 P0/P1/P2 数据，识别趋势和薄弱维度 |
| `docs-builder` | 4 | 知识策展：从实施过程中提取可复用模式、踩坑记录，消费 `code-impl-reporter` 输出 |
| `code-impl-reporter` | 4 | 实施过程报告：记录真实调用路径、质量数据、效率指标、可复用知识 |

## 何时优先用 Skill

- 需要确定执行流程
- 需要固定输出格式
- 需要约束禁止事项
- 需要把结果写回正式文档

## 何时优先用 Agent

- 需要并行专家视角
- 需要角色化补充分析
- 需要回答“必答问题”
- 需要在保存前做候选审查

## 禁止事项

- Agent 不得复制所属 Skill 的完整流程定义。
- Skill 不得把不存在的 Agent 当作必选依赖。
- 同一能力的 Skill 与 Agent 必须在术语和边界上保持一致。
