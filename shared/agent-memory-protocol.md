# Agent 记忆协议（共享规范）

> 本文件为 `generate-document` 与 `implement-code` 所涉及的全部 agent 提供统一的记忆读写协议。所有 agent 必须遵守。

## 1. 总原则

- **每个 agent 维护独立的记忆文件**：`.claude/agents/memory/<agent-name>.md`
- **调用前读取**：agent 被调用时，先读取自身记忆文件，将历史发现作为上下文参考
- **调用后追加**：agent 产出结果后，将关键发现（1-3 条）追加到自身记忆文件
- **跨查阅**：agent 可读取 `knowledge.md` 获取跨 agent 共性知识，但只能写入自己的记忆文件
- **知识策展**：`knowledge-curator` 有权读取所有记忆文件并提取跨 agent 共性知识到 `knowledge.md`

## 2. 记忆文件格式

```markdown
---
agent: <agent-name>
last_updated: <ISO datetime>
entry_count: <N>
---

## 记忆条目

### [<YYYY-MM-DD>] <简要标题>
- **上下文**：功能名 / 任务类型 / 阶段
- **发现/决策**：<具体内容>
- **可复用性**：高/中/低
- **证据**：<产物路径或代码位置>

---
```

## 3. 容量控制

- 每个记忆文件最多保留 **50 条**条目
- 超出时按以下优先级淘汰：
  1. `可复用性：低` 的条目优先淘汰
  2. 时间最久远的条目优先淘汰
  3. 不得删除 `可复用性：高` 的条目（除非已超过 30 条）
- 淘汰操作由 `knowledge-curator` 在每次策展时统一执行

## 4. 写入规则

1. **写入时机**：每次 agent 被调用并产出结果后，必须将关键发现追加到自己的记忆文件
2. **写入数量**：每次调用追加 1-3 条条目，禁止追加超过 5 条
3. **写入质量**：
   - 条目标题必须具体（禁止"一般性经验"等模糊标题）
   - 发现/决策必须是可验证的（含文件路径、行号或产物路径）
   - 可复用性标注必须基于实际重复出现概率，禁止全部标"高"
4. **禁止事项**：
   - 禁止写入虚构的发现（无证据支撑的结论）
   - 禁止写入与当前任务无关的泛泛观察
   - 禁止一次性清空或覆写记忆文件（必须追加）

## 5. 读取规则

1. **读取时机**：agent 被调用时，先读取自身记忆文件
2. **读取深度**：优先读取最近 10 条条目；若任务类型与历史条目上下文匹配，可读取更多
3. **读取应用**：
   - 将历史发现作为上下文参考（如"上次审查发现 Store 未注册"）
   - 不将历史发现作为当前任务的必选约束（仅作参考）
   - 历史发现与当前任务冲突时，以当前任务的实际证据为准
4. **跨查阅**：
   - 任何 agent 可读取 `knowledge.md` 获取跨 agent 共性知识
   - 读取其他 agent 的记忆文件时，只作为补充参考，不作为必选依赖

## 6. 知识策展（knowledge-curator 专属）

- `knowledge-curator` 在每次策展时：
  1. 读取所有 agent 的记忆文件
  2. 提取跨 agent 共性知识（至少 2 个 agent 独立发现的相同模式）
  3. 将共性知识写入 `knowledge.md`
  4. 对超过 50 条的记忆文件执行淘汰
  5. 更新所有记忆文件的 `last_updated` 和 `entry_count` 字段

## 7. 与 evidence-and-uncertainty 的衔接

- 记忆条目中的"发现/决策"必须遵守 `../../shared/evidence-and-uncertainty.md` 的真值层级：
  - A 类（已证）发现可直接写入
  - B 类（可推导）发现须标注推导链
  - C 类（未证）发现须标注"待验证"
  - D 类（禁止）发现不得写入记忆
- 记忆条目仅作为**参考上下文**，不作为当前任务的证据来源；当前任务的事实必须基于本次实际工具调用结果
