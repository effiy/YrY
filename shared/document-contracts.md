# 文档契约总览

本文档定义项目文档生成体系中的核心契约，合并了文档类型矩阵、影响分析契约、路径约定、Skill-Agent 边界和 MCP 降级原则。

## 文档类型矩阵

| 类型 | 规范 | 模板 | 单独生成路径 | 全文档路径 | 说明 |
|------|------|------|-------------|-----------|------|
| 需求文档 | `skills/generate-document/rules/需求文档.md` | 可用 | `docs/01_需求文档/` | `docs/<功能名>/01_需求文档.md` | 模板骨架 + 规范约束 |
| 需求任务 | `skills/generate-document/rules/需求任务.md` | 可用 | `docs/02_需求任务/` | `docs/<功能名>/02_需求任务.md` | 模板骨架 + 规范约束 |
| 设计文档 | `skills/generate-document/rules/设计文档.md` | 禁用 | `docs/03_设计文档/` | `docs/<功能名>/03_设计文档.md` | 只读规范与上游事实 |
| 使用文档 | `skills/generate-document/rules/使用文档.md` | 不提供 | `docs/04_使用文档/` | `docs/<功能名>/04_使用文档.md` | 规范驱动 |
| 动态检查清单 | `skills/generate-document/rules/动态检查清单.md` | 禁用 | 仅全文档模式 | `docs/<功能名>/05_动态检查清单.md` | 从上游文档动态抽取 |
| 实施总结 | `implement-code` 技能输出（init 例外：由 init 写入） | 不适用 | 不单独生成 | `docs/<功能名>/06_实施总结.md` | 不由 `generate-document` 生成（init 命令除外） |
| 项目报告 | `skills/generate-document/rules/项目报告.md` | 不提供 | `docs/05_项目报告/` | `docs/<功能名>/07_项目报告.md` | 基于真实变更数据 |
| 通用文档 | `skills/generate-document/rules/通用文档.md` | 可用 | `docs/` | 不适用 | 模板骨架 + 规范约束 |
| 项目基础文件 | `skills/generate-document/rules/项目基础文件.md` | 禁用 | 根目录 + `docs/` | 不适用 | 仅 `/generate-document init` 使用 |
| 周报 | `skills/generate-document/rules/周报.md` | 禁用 | `docs/周报/` | `docs/周报/<YYYY>-W<NN>.md` | 仅 `/generate-document weekly` 使用 |

## 术语定义

- **规范**：强约束，定义章节结构、必填项和禁止事项。
- **模板**：弱约束，只提供起手骨架。
- **检查清单**：验收维度，按 P0 / P1 / P2 划分。
- **全文档**：同一功能在 `docs/<功能名>/` 下的阶段化文档集合。
- **Grounding**：所有技术事实必须能追溯到上游文档或代码。
- **全项目影响链分析**：按下方影响分析契约搜索上游依赖、反向依赖、传递依赖、导出链、注册链、测试、文档、配置与外部依赖影响，作为需求任务、设计文档和代码实施的 P0 前置条件。
- **证据与不确定性**：可采纳、可核对的陈述与待补充的边界，见 `evidence-and-uncertainty.md`；禁止用叙事代替证据。

## 影响分析契约

凡涉及需求拆解、设计生成、代码实施、修复回归或交付总结，都必须执行全项目影响分析。

### 核心原则

1. **全项目范围**：默认搜索整个仓库，而不是只搜索当前模块目录
2. **依赖图闭合**：每个拟改动点都必须追踪"它依赖谁、谁依赖它"，直到影响链闭合或明确停止原因
3. **反向依赖优先**：删除、重命名、修改公共接口前，必须先证明所有调用方已覆盖
4. **事实来自工具**：引用关系必须来自实际搜索或文件读取，不得凭记忆填写
5. **未命中也要记录**：每个搜索词都必须记录结果；没有命中时写"未找到引用"

### 适用阶段

| 阶段 | 使用方 | 产物位置 |
|------|--------|----------|
| 规划期 | `generate-document` | 写入需求任务第 6 章或设计文档第 5 章 |
| 实施前 | `implement-code` 阶段 1 | 写入 `06_实施总结.md` 阶段 1 记录 |
| 实施后 | `implement-code` 阶段 3 | 写入逐模块记录 |

### 必查维度

| 维度 | 闭合标准 |
|------|----------|
| 直接引用 | 每个搜索词都有命中记录或"未找到引用"记录 |
| 上游依赖 | 变更风险已标注 |
| 反向依赖 | 所有调用方处置方式明确 |
| 传递依赖 | 二级及以上影响已闭合或写明停止原因 |
| 导出链 | 导出入口和所有 import 方一致 |
| 注册链 | 注册点、消费点、注册顺序已核对 |
| 数据流 | payload 与所有消费方一致 |
| 类型与契约 | 类型、文档、mock 与实现一致 |
| 样式影响 | 模板、`:class`、文档示例均已覆盖 |
| 测试与文档 | 需更新项已列入处置 |
| 配置与构建 | 本地运行、构建、CI 无断链 |
| 外部依赖 | 新增依赖有来源、用途、风险 |

### P0 门禁

- 未完成全项目搜索，不得生成设计结论或开始实施
- 未输出搜索词与改动点清单，不得声称影响分析完整
- 未分析反向依赖和传递依赖，不得删除或修改公共接口
- 实施后未用真实 diff 回归搜索，不得进入最终总结

## 路径约定

### 稳定入口

- 技能目录：`.claude/skills/<skill-name>/`
- 技能真源：`.claude/skills/<skill-name>/SKILL.md`
- 代理目录：`.claude/agents/<agent-name>.md`
- 代理记忆：`.claude/agents/memory/<agent-name>.md`

### 禁止使用的旧路径

- `.claude/skills/generate-document.md`、`.claude/rules/...`、`.claude/templates/...`、`.claude/skills/checklist/...`

### 链接治理

- 只链接到仓库里真实存在的文件
- 若某能力没有对应 skill/agent，写"未提供专用入口"
- 目录结构调整后，做全仓 `.claude/` 链接回归搜索

## Skill 与 Agent 边界

- **Skill** 决定怎么做（方法、流程、输入输出格式）
- **Agent** 负责从某个专业视角补充判断（角色定位、必答问题、候选结论）

| 能力主题 | Skill 职责 | Agent 职责 |
|----------|-----------|-----------|
| `code-review` / `code-reviewer` | 定义审查输入、维度与输出格式 | 作为审查专家给出 P0/P1/P2 问题列表 |
| `e2e-testing` / `e2e-tester` | 定义测试方案生成方法和产物结构 | 作为 E2E 专家回答场景级测试问题 |
| `generate-document` | 文档生成主流程与规则装配 | 可按场景调用多个 agent 作为候选输入 |

禁止事项：Agent 不得复制所属 Skill 的完整流程；Skill 不得把不存在的 Agent 当作必选依赖。

## MCP 降级原则

1. MCP 不可用时不得阻断流程，使用 Grep/Read/Glob 替代
2. 降级须记录到产物（`06_实施总结.md` 或项目日志）
3. 禁止静默降级

## 真源优先级

1. `SKILL.md`
2. `shared/evidence-and-uncertainty.md`（与 `rules` 同遇冲突时，以本矩阵未覆盖的禁止性条款为准）
3. `rules/*.md`
4. `checklists/*.md`
5. `templates/*.md`
6. `README.md`

若不同文件之间存在冲突，以上述优先级解决，不得反向以模板覆盖规范。
