# Document Contracts Overview

This file defines the core contracts of this repository's document generation system, serving as the shared interpretation layer for `generate-document` and related rule files.

## Document Type Matrix

|------|------|----------|-----------------|---------------|-------|

| Requirement Tasks | `skills/generate-document/rules/requirement-tasks.md` | Allowed | `docs/02_requirement-tasks/` | `docs/<feature-name>/02_requirement-tasks.md` | Skeleton + rule constraints |

## Terminology

- **Spec**: Strong constraint defining chapter structure, required fields, and prohibitions.
- **Template**: Weak constraint providing only a starting skeleton.
- **Checklist**: Acceptance dimensions graded by P0 / P1 / P2.
- **Full-set**: The staged document collection under `docs/<feature-name>/` for the same feature.
- **Grounding**: All technical facts must be traceable to upstream documents or code.
- **Run/Block Log**: When a skill cannot write the target artifact due to missing parameters, undeterminable target directory, or missing prerequisite materials, it must still write a Markdown record to `docs/99_agent-runs/`.
- **Full-project impact chain analysis**: Per `impact-analysis-contract.md`, search upstream dependencies, reverse dependencies, transitive dependencies, export chains, registration chains, tests, docs, configuration, and external-dependency impacts as a P0 prerequisite for requirement tasks, design documents, and code implementation.
- **Evidence and uncertainty**: Admissible, verifiable statements and boundaries to be supplemented; see `evidence-and-uncertainty.md`. Both `generate-document` and `implement-code` body text and summaries must comply. Narrative must not replace evidence.

## Source-of-Truth Priority

1. `SKILL.md`
2. `shared/evidence-and-uncertainty.md` (when conflicting with `rules`, prohibitive clauses not covered by this matrix take precedence; otherwise `SKILL` / `rules` take precedence)
3. `rules/*.md`
4. `checklists/*.md`
5. `templates/*.md`
6. `README.md`

If conflicts exist between files, resolve using the above priority. Templates must not override specs.
